<span data-ttu-id="055e0-103">Функции обработки исключений в языке C# помогают вам справиться с непредвиденными или исключительными проблемами, которые возникают при выполнении программы.</span><span class="sxs-lookup"><span data-stu-id="055e0-103">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="055e0-104">Обработка исключений использует ключевые слова `try`, `catch` и `finally` для действий, которые могут оказаться неудачными. Это позволяет обрабатывать ошибки так, как кажется разумным, а также правильно высвобождать ресурсы.</span><span class="sxs-lookup"><span data-stu-id="055e0-104">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it is reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="055e0-105">Исключения могут создаваться средой выполнения (CLR), платформой .NET Framework, библиотеками сторонних поставщиков или кодом самого приложения.</span><span class="sxs-lookup"><span data-stu-id="055e0-105">Exceptions can be generated by the common language runtime (CLR), by the .NET Framework or any third-party libraries, or by application code.</span></span> <span data-ttu-id="055e0-106">Чтобы создать исключение, используйте ключевое слово `throw`.</span><span class="sxs-lookup"><span data-stu-id="055e0-106">Exceptions are created by using the `throw` keyword.</span></span>  
  
 <span data-ttu-id="055e0-107">Во многих случаях исключение может создаваться не тем методом, который вызывается в вашем коде, а одним из последующих методов в стеке вызовов.</span><span class="sxs-lookup"><span data-stu-id="055e0-107">In many cases, an exception may be thrown not by a method that your code has called directly, but by another method further down in the call stack.</span></span> <span data-ttu-id="055e0-108">В этом случае среда CLR разворачивает весь стек в поисках метода с блоком `catch` для исключений соответствующего типа, и выполняет первый обнаруженный блок `catch` подходящего вида.</span><span class="sxs-lookup"><span data-stu-id="055e0-108">When this happens, the CLR will unwind the stack, looking for a method with a `catch` block for the specific exception type, and it will execute the first such `catch` block that if finds.</span></span> <span data-ttu-id="055e0-109">Если подходящий блок `catch` не будет обнаружен во всем стеке вызовов, среда CLR завершает процесс и выводит сообщение для пользователя.</span><span class="sxs-lookup"><span data-stu-id="055e0-109">If it finds no appropriate `catch` block anywhere in the call stack, it will terminate the process and display a message to the user.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="055e0-112">Общие сведения об исключениях</span><span class="sxs-lookup"><span data-stu-id="055e0-112">Exceptions Overview</span></span>  
 <span data-ttu-id="055e0-113">Исключения имеют следующие свойства.</span><span class="sxs-lookup"><span data-stu-id="055e0-113">Exceptions have the following properties:</span></span>  
  
-   <span data-ttu-id="055e0-114">Исключения представляют собой типы, производные в конечном счете от `System.Exception`.</span><span class="sxs-lookup"><span data-stu-id="055e0-114">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>  
  
-   <span data-ttu-id="055e0-115">Используйте блок `try` для выполнения таких инструкций, которые могут создавать исключения.</span><span class="sxs-lookup"><span data-stu-id="055e0-115">Use a `try` block around the statements that might throw exceptions.</span></span>  
  
-   <span data-ttu-id="055e0-116">Когда внутри такого блока `try` возникает исключение, поток управления переходит к первому подходящему обработчику исключений в стеке вызовов.</span><span class="sxs-lookup"><span data-stu-id="055e0-116">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="055e0-117">В C# ключевое слово `catch` обозначает обработчик исключений.</span><span class="sxs-lookup"><span data-stu-id="055e0-117">In C#, the `catch` keyword is used to define an exception handler.</span></span>  
  
-   <span data-ttu-id="055e0-118">Если для созданного исключения не существует обработчиков, выполнение программы прекращается с сообщением об ошибке.</span><span class="sxs-lookup"><span data-stu-id="055e0-118">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>  
  
-   <span data-ttu-id="055e0-119">Не перехватывайте исключение, если вы не намерены его обрабатывать с сохранением известного состояния приложения.</span><span class="sxs-lookup"><span data-stu-id="055e0-119">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="055e0-120">Если вы перехватываете `System.Exception`, создайте его заново в конце блока `catch`, используя ключевое слово `throw`.</span><span class="sxs-lookup"><span data-stu-id="055e0-120">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>  
  
-   <span data-ttu-id="055e0-121">Если блок `catch` определяет переменную исключения, ее можно использовать для получения дополнительных сведений о типе созданного исключения.</span><span class="sxs-lookup"><span data-stu-id="055e0-121">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>  
  
-   <span data-ttu-id="055e0-122">Программа может явным образом создавать исключения с помощью ключевого слова `throw`.</span><span class="sxs-lookup"><span data-stu-id="055e0-122">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>  
  
-   <span data-ttu-id="055e0-123">Объекты исключения содержат подробные сведения об ошибке, например состояние стека вызовов и текстовое описание ошибки.</span><span class="sxs-lookup"><span data-stu-id="055e0-123">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>  
  
-   <span data-ttu-id="055e0-124">Код в блоке `finally` выполняется даже в том случае, если создано исключение.</span><span class="sxs-lookup"><span data-stu-id="055e0-124">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="055e0-125">Используйте блок `finally`, чтобы высвободить ресурсы, например закрыть потоки и файлы, которые были открыты внутри блока `try`.</span><span class="sxs-lookup"><span data-stu-id="055e0-125">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>  
  
-   <span data-ttu-id="055e0-126">Управляемые исключения реализованы в платформе .NET Framework на основе структурированного механизма обработки исключений Win32.</span><span class="sxs-lookup"><span data-stu-id="055e0-126">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="055e0-127">Дополнительные сведения см. в статьях [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) (Структурированная обработка исключений в C и C++) и [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm) (Интенсивное погружение в структурированную обработку исключений на платформе Win32).</span><span class="sxs-lookup"><span data-stu-id="055e0-127">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
