## <a name="class-and-struct-accessibility"></a><span data-ttu-id="40936-118">Доступность классов и структур</span><span class="sxs-lookup"><span data-stu-id="40936-118">Class and Struct Accessibility</span></span>  
 <span data-ttu-id="40936-119">Классы и структуры, объявленные непосредственно в пространстве имен (другими словами, не вложенные в другие классы или структуры), могут быть открытыми или внутренними.</span><span class="sxs-lookup"><span data-stu-id="40936-119">Classes and structs that are declared directly within a namespace (in other words, that are not nested within other classes or structs) can be either public or internal.</span></span> <span data-ttu-id="40936-120">Если модификатор доступа не указан, по умолчанию используется внутренний тип.</span><span class="sxs-lookup"><span data-stu-id="40936-120">Internal is the default if no access modifier is specified.</span></span>  
  
 <span data-ttu-id="40936-121">Члены структуры, включая вложенные классы и структуры, могут объявляться как открытые, внутренние или закрытые.</span><span class="sxs-lookup"><span data-stu-id="40936-121">Struct members, including nested classes and structs, can be declared as public, internal, or private.</span></span> <span data-ttu-id="40936-122">Члены класса, включая вложенные классы и структуры, могут объявляться как открытые, защищенные внутренние, защищенные, внутренние, защищенные закрытые или закрытые.</span><span class="sxs-lookup"><span data-stu-id="40936-122">Class members, including nested classes and structs, can be public, protected internal, protected, internal, private protected or private.</span></span> <span data-ttu-id="40936-123">По умолчанию уровень доступа к членам класса и членам структуры, включая вложенные классы и структуры, является закрытым.</span><span class="sxs-lookup"><span data-stu-id="40936-123">The access level for class members and struct members, including nested classes and structs, is private by default.</span></span> <span data-ttu-id="40936-124">Закрытые вложенные типы недоступны за пределами типа, в котором содержатся.</span><span class="sxs-lookup"><span data-stu-id="40936-124">Private nested types are not accessible from outside the containing type.</span></span>  
  
 <span data-ttu-id="40936-125">Производные классы не могут быть более доступны, чем соответствующие базовые типы.</span><span class="sxs-lookup"><span data-stu-id="40936-125">Derived classes cannot have greater accessibility than their base types.</span></span> <span data-ttu-id="40936-126">Другими словами, нельзя иметь открытый класс `B`, производный от внутреннего класса `A`.</span><span class="sxs-lookup"><span data-stu-id="40936-126">In other words, you cannot have a public class `B` that derives from an internal class `A`.</span></span> <span data-ttu-id="40936-127">Если бы это было возможно, класс `A` стал бы открытым, поскольку все защищенные или внутренние члены класса `A` были бы доступны из производного класса.</span><span class="sxs-lookup"><span data-stu-id="40936-127">If this were allowed, it would have the effect of making `A` public, because all protected or internal members of `A` are accessible from the derived class.</span></span>  
  
 <span data-ttu-id="40936-128">Доступ к внутренним типам можно предоставить некоторым другим сборкам с помощью класса InternalsVisibleToAttribute.</span><span class="sxs-lookup"><span data-stu-id="40936-128">You can enable specific other assemblies to access your internal types by using the InternalsVisibleToAttribute.</span></span> <span data-ttu-id="40936-129">Дополнительные сведения см. в разделе [Дружественные сборки](../../../standard/assembly/friend.md).</span><span class="sxs-lookup"><span data-stu-id="40936-129">For more information, see [Friend Assemblies](../../../standard/assembly/friend.md).</span></span>  
  
## <a name="class-and-struct-member-accessibility"></a><span data-ttu-id="40936-130">Доступность членов классов и структур</span><span class="sxs-lookup"><span data-stu-id="40936-130">Class and Struct Member Accessibility</span></span>  
 <span data-ttu-id="40936-131">Члены класса (включая вложенные классы и структуры) можно объявлять с любым из шести типов доступа.</span><span class="sxs-lookup"><span data-stu-id="40936-131">Class members (including nested classes and structs) can be declared with any of the six types of access.</span></span> <span data-ttu-id="40936-132">Члены структуры нельзя объявлять как защищенные, поскольку структуры не поддерживают наследование.</span><span class="sxs-lookup"><span data-stu-id="40936-132">Struct members cannot be declared as protected because structs do not support inheritance.</span></span>  
  
 <span data-ttu-id="40936-133">Как правило, уровень доступности члена не может быть выше уровня доступности типа, в который он входит.</span><span class="sxs-lookup"><span data-stu-id="40936-133">Normally, the accessibility of a member is not greater than the accessibility of the type that contains it.</span></span> <span data-ttu-id="40936-134">При этом открытый член внутреннего класса может быть доступен за пределами сборки, если он реализует методы интерфейса или переопределяет виртуальные методы, определенные в открытом базовом классе.</span><span class="sxs-lookup"><span data-stu-id="40936-134">However, a public member of an internal class might be accessible from outside the assembly if the member implements interface methods or overrides virtual methods that are defined in a public base class.</span></span>  
  
 <span data-ttu-id="40936-135">Тип любого члена, который является полем, свойством или событием, должен иметь, как минимум, такой же уровень доступности, как у самого члена.</span><span class="sxs-lookup"><span data-stu-id="40936-135">The type of any member that is a field, property, or event must be at least as accessible as the member itself.</span></span> <span data-ttu-id="40936-136">Точно так же тип возвращаемого значения и типы параметров любого члена, который является методом, индексатором или делегатом, должен иметь, как минимум, такой же уровень доступности, как у самого члена.</span><span class="sxs-lookup"><span data-stu-id="40936-136">Similarly, the return type and the parameter types of any member that is a method, indexer, or delegate must be at least as accessible as the member itself.</span></span> <span data-ttu-id="40936-137">Например, нельзя иметь открытый метод `M`, возвращающий класс `C`, если `C` не является также открытым.</span><span class="sxs-lookup"><span data-stu-id="40936-137">For example, you cannot have a public method `M` that returns a class `C` unless `C` is also public.</span></span> <span data-ttu-id="40936-138">Аналогичным образом нельзя иметь защищенное свойство типа `A`, если `A` объявлен как закрытый.</span><span class="sxs-lookup"><span data-stu-id="40936-138">Likewise, you cannot have a protected property of type `A` if `A` is declared as private.</span></span>  
  
 <span data-ttu-id="40936-139">Пользовательские операторы всегда должны объявляться как открытые и статические.</span><span class="sxs-lookup"><span data-stu-id="40936-139">User-defined operators must always be declared as public and static.</span></span> <span data-ttu-id="40936-140">Для получения дополнительной информации см. раздел [Перегрузка операторов](../../language-reference/operators/operator-overloading.md).</span><span class="sxs-lookup"><span data-stu-id="40936-140">For more information, see [Operator overloading](../../language-reference/operators/operator-overloading.md).</span></span>

 <span data-ttu-id="40936-148">Интерфейсы, объявляемые непосредственно в пространстве имен, могут быть объявлены как открытые или внутренние.
