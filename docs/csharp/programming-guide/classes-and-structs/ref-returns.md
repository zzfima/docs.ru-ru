<span data-ttu-id="a1051-104">Начиная с версии 7.0 язык C# поддерживает значения, возвращаемые по ссылке (возвращаемые ссылочные значения).</span><span class="sxs-lookup"><span data-stu-id="a1051-104">Starting with C# 7.0, C# supports reference return values (ref returns).</span></span> <span data-ttu-id="a1051-105">Возвращаемое ссылочное значение позволяет методу вернуть вызывающей стороне ссылку на переменную, а не фиксированное значение.</span><span class="sxs-lookup"><span data-stu-id="a1051-105">A reference return value allows a method to return a reference to a variable, rather than a value, back to a caller.</span></span> <span data-ttu-id="a1051-106">После этого вызывающий может самостоятельно решить, как обрабатывать полученную переменную: по значению или по ссылке.</span><span class="sxs-lookup"><span data-stu-id="a1051-106">The caller can then choose to treat the returned variable as if it were returned by value or by reference.</span></span> <span data-ttu-id="a1051-107">Вызывающий может создать новую переменную и присвоить ей ссылку на полученное значение (локальная ссылочная переменная).</span><span class="sxs-lookup"><span data-stu-id="a1051-107">The caller can create a new variable that is itself a reference to the returned value, called a ref local.</span></span>

<span data-ttu-id="a1051-109">Большинство разработчиков прекрасно знакомы с передачей аргумента в вызываемый метод *по ссылке*.</span><span class="sxs-lookup"><span data-stu-id="a1051-109">Most developers are familiar with passing an argument to a called method *by reference*.</span></span> <span data-ttu-id="a1051-110">Список аргументов вызываемого метода включает переменную, переданную по ссылке.</span><span class="sxs-lookup"><span data-stu-id="a1051-110">A called method's argument list includes a variable passed by reference.</span></span> <span data-ttu-id="a1051-111">Вызывающий может отследить любые изменения этого значения в вызываемом методе.</span><span class="sxs-lookup"><span data-stu-id="a1051-111">Any changes made to its value by the called method are observed by the caller.</span></span> <span data-ttu-id="a1051-112">*Возвращаемое ссылочное значение* означает, что метод возвращает *ссылку* на некоторую переменную (или ее псевдоним).</span><span class="sxs-lookup"><span data-stu-id="a1051-112">A *reference return value* means that a method returns a *reference* (or an alias) to some variable.</span></span> <span data-ttu-id="a1051-113">В область действия переменной должен входить этот метод.</span><span class="sxs-lookup"><span data-stu-id="a1051-113">That variable's scope must include the method.</span></span> <span data-ttu-id="a1051-114">Время существования переменной должно продолжаться после того, как метод возвращает управление.</span><span class="sxs-lookup"><span data-stu-id="a1051-114">That variable's lifetime must extend beyond the return of the method.</span></span> <span data-ttu-id="a1051-115">Все изменения, которые вызывающий производит с возвращаемым значением метода, применяются к возвращенной переменной.</span><span class="sxs-lookup"><span data-stu-id="a1051-115">Modifications to the method's return value by the caller are made to the variable that is returned by the method.</span></span>

<span data-ttu-id="a1051-116">Если для метода объявлено *возвращаемое ссылочное значение*, значит он возвращает псевдоним переменной.</span><span class="sxs-lookup"><span data-stu-id="a1051-116">Declaring that a method returns a *reference return value* indicates that the method returns an alias to a variable.</span></span> <span data-ttu-id="a1051-117">Чаще всего это нужно для того, чтобы вызывающий код получил псевдоним для доступа к этой переменной, в том числе для ее изменения.</span><span class="sxs-lookup"><span data-stu-id="a1051-117">The design intent is often that the calling code should have access to that variable through the alias, including to modify it.</span></span> <span data-ttu-id="a1051-118">Следовательно, методы с возвращаемыми ссылочными значениями не могут иметь тип возвращаемого значения `void`.</span><span class="sxs-lookup"><span data-stu-id="a1051-118">It follows that methods returning by reference can't have the return type `void`.</span></span>

- <span data-ttu-id="a1051-128">Возвращаемое значение не может быть константой, элементом перечисления, полученным по значению, возвращаемым значением свойства, а также методом `class` или `struct`.</span><span class="sxs-lookup"><span data-stu-id="a1051-128">The return value cannot be a constant, an enumeration member, the by-value return value from a property, or a method of a `class` or `struct`.</span></span> <span data-ttu-id="a1051-129">При нарушении этого правила возникает ошибка компилятора CS8156 "Выражение невозможно использовать в данном контексте, так как его невозможно вернуть по ссылке".</span><span class="sxs-lookup"><span data-stu-id="a1051-129">Violating this rule generates compiler error CS8156, "An expression cannot be used in this context because it may not be returned by reference."</span></span>

<span data-ttu-id="a1051-133">Метод, который возвращает *возвращаемое значение ссылки*, должен удовлетворять следующим двум условиям:</span><span class="sxs-lookup"><span data-stu-id="a1051-133">A method that returns a *reference return value* must satisfy the following two conditions:</span></span>
