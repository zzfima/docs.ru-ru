Как определить равенства значений для типа</span><span class="sxs-lookup"><span data-stu-id="628a4-102">How to define value equality for a type (C# Programming Guide)</span></span>

<span data-ttu-id="628a4-103">При определении класса или структуры необходимо решить, имеет ли смысл создавать пользовательское определение равенства значений (или эквивалентности) для этого типа.</span><span class="sxs-lookup"><span data-stu-id="628a4-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="628a4-104">Обычно правила определения равенства реализуются, если объекты этого типа будут добавляться в коллекции или если они в первую очередь предназначены для хранения набора полей или свойств.</span><span class="sxs-lookup"><span data-stu-id="628a4-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="628a4-105">В основу определения равенства значений можно положить сравнение всех полей и свойств в типе или только их части.</span><span class="sxs-lookup"><span data-stu-id="628a4-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span> <span data-ttu-id="628a4-106">Но в любом случае (как для классов, так и для структур) реализация должна соответствовать следующим пяти гарантиям равенства.</span><span class="sxs-lookup"><span data-stu-id="628a4-106">But in either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence:</span></span>  
  
1. <span data-ttu-id="628a4-107">`x.Equals(x)` возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="628a4-107">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="628a4-108">Это называется свойством рефлексивности.</span><span class="sxs-lookup"><span data-stu-id="628a4-108">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="628a4-109">`x.Equals(y)` возвращает то же значение, что и `y.Equals(x)`.</span><span class="sxs-lookup"><span data-stu-id="628a4-109">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="628a4-110">Это называется свойством симметрии.</span><span class="sxs-lookup"><span data-stu-id="628a4-110">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="628a4-111">Если `(x.Equals(y) && y.Equals(z))` возвращает `true`, `x.Equals(z)` возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="628a4-111">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="628a4-112">Это называется свойством транзитивности.</span><span class="sxs-lookup"><span data-stu-id="628a4-112">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="628a4-113">Последовательные вызовы `x.Equals(y)` возвращают одно и то же значение до тех пор, пока объекты, на которые ссылаются x и y, не будут изменены.</span><span class="sxs-lookup"><span data-stu-id="628a4-113">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="628a4-114">`x.Equals(null)` возвращает `false`.</span><span class="sxs-lookup"><span data-stu-id="628a4-114">`x.Equals(null)` returns `false`.</span></span> <span data-ttu-id="628a4-115">Однако `null.Equals(null)` вызывает исключение. Эта инструкция не удовлетворяет приведенному выше правилу номер два.</span><span class="sxs-lookup"><span data-stu-id="628a4-115">However, `null.Equals(null)` throws an exception; it does not obey rule number two above.</span></span>  
  
 <span data-ttu-id="628a4-116">Любая определяемая вами структура имеет заданную по умолчанию реализацию равенства значений, которая наследуется от переопределения <xref:System.ValueType?displayProperty=nameWithType> метода <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="628a4-116">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="628a4-117">Эта реализация использует отражение для проверки всех полей и свойств в типе.</span><span class="sxs-lookup"><span data-stu-id="628a4-117">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="628a4-118">Хотя эта реализация возвращает верный результат, она отличается невысокой скоростью по сравнению с пользовательской реализацией, которую можно написать специально для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="628a4-118">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="628a4-119">Детали реализации равенства значений для классов и структур различаются.</span><span class="sxs-lookup"><span data-stu-id="628a4-119">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="628a4-120">Однако для реализации равенства как для классов, так и для структур, необходимо выполнить одни и те же базовые действия.</span><span class="sxs-lookup"><span data-stu-id="628a4-120">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1. <span data-ttu-id="628a4-121">Переопределите [виртуальный](../../language-reference/keywords/virtual.md) метод <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="628a4-121">Override the [virtual](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="628a4-122">В большинстве случаев пользовательская реализация `bool Equals( object obj )` должна вызывать относящийся к конкретному типу метод `Equals`, который является реализацией интерфейса <xref:System.IEquatable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="628a4-122">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="628a4-123">(См. шаг 2.)</span><span class="sxs-lookup"><span data-stu-id="628a4-123">(See step 2.)</span></span>  
  
2. <span data-ttu-id="628a4-124">Реализуйте интерфейс <xref:System.IEquatable%601?displayProperty=nameWithType>, предоставив метод `Equals` для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="628a4-124">Implement the <xref:System.IEquatable%601?displayProperty=nameWithType> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="628a4-125">Именно на этом этапе происходит фактическое сравнение значений.</span><span class="sxs-lookup"><span data-stu-id="628a4-125">This is where the actual equivalence comparison is 3. <span data-ttu-id="628a4-131">Необязательно, но рекомендуется: перегрузите операторы [==](../../language-reference/operators/equality-operators.md#equality-operator-) и [!=](../../language-reference/operators/equality-operators.md#inequality-operator-).</span><span class="sxs-lookup"><span data-stu-id="628a4-131">Optional but recommended: Overload the [==](../../language-reference/operators/equality-operators.md#equality-operator-) and [!=](../../language-reference/operators/equality-operators.md#inequality-operator-) operators.</span></span>  
  
4. <span data-ttu-id="628a4-132">Переопределите <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> таким образом, чтобы два объекта с равными значениями создавали одинаковый хэш-код.</span><span class="sxs-lookup"><span data-stu-id="628a4-132">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>  
  
5. <span data-ttu-id="628a4-133">Необязательные: Для поддержки определений "больше" и "меньше" реализуйте для типа интерфейс <xref:System.IComparable%601>, а также перегрузите операторы [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) и [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-).</span><span class="sxs-lookup"><span data-stu-id="628a4-133">Optional: To support definitions for "greater than" or "less than," implement the <xref:System.IComparable%601> interface for your type, and also overload the [<=](../../language-reference/operators/comparison-operators.md#less-than-or-equal-operator-) and [>=](../../language-reference/operators/comparison-operators.md#greater-than-or-equal-operator-) operators.</span></span>
