<span data-ttu-id="ce976-117">Компиляция является представлением всего, что необходимо для компиляции программы C# или Visual Basic, куда входят все ссылки на сборки, параметры компилятора и исходные файлы.</span><span class="sxs-lookup"><span data-stu-id="ce976-117">A compilation is a representation of everything needed to compile a C# or Visual Basic program, which includes all the assembly references, compiler options, and source files.</span></span>

## <a name="symbols"></a><span data-ttu-id="ce976-125">Символы</span><span class="sxs-lookup"><span data-stu-id="ce976-125">Symbols</span></span>

<span data-ttu-id="ce976-126">Символ представляет отдельный элемент, объявленный исходным кодом или импортированный из сборки в виде метаданных.</span><span class="sxs-lookup"><span data-stu-id="ce976-126">A symbol represents a distinct element declared by the source code or imported from an assembly as metadata.</span></span> <span data-ttu-id="ce976-127">Символом представлены все пространства имен, типы, методы, свойства, поля, события, параметры или локальные переменные.</span><span class="sxs-lookup"><span data-stu-id="ce976-127">Every namespace, type, method, property, field, event, parameter, or local variable is represented by a symbol.</span></span>

<span data-ttu-id="ce976-137">По своей сути символы похожи на систему типов среды CLR, представленную API <xref:System.Reflection>, однако их возможности шире, так как они моделируют не только типы.</span><span class="sxs-lookup"><span data-stu-id="ce976-137">Symbols are similar in concept to the CLR type system as represented by the <xref:System.Reflection> API, yet they are richer in that they model more than just types.</span></span> <span data-ttu-id="ce976-138">Пространства имен, локальные переменные и метки — все это символы.</span><span class="sxs-lookup"><span data-stu-id="ce976-138">Namespaces, local variables, and labels are all symbols.</span></span> <span data-ttu-id="ce976-139">Кроме того, символы являются представлением концепций языка, а не концепций среды CLR.</span><span class="sxs-lookup"><span data-stu-id="ce976-139">In addition, symbols are a representation of language concepts, not CLR concepts.</span></span> <span data-ttu-id="ce976-140">Есть много похожего, но имеется и большое количество значимых различий.</span><span class="sxs-lookup"><span data-stu-id="ce976-140">There is a lot of overlap, but there are many meaningful distinctions as well.</span></span> <span data-ttu-id="ce976-141">Например, метод итератора в C# или Visual Basic является отдельным символом.</span><span class="sxs-lookup"><span data-stu-id="ce976-141">For instance, an iterator method in C# or Visual Basic is a single symbol.</span></span> <span data-ttu-id="ce976-142">Однако когда метод итератора преобразуется в метаданные среды CLR, он превращается в тип и несколько методов.</span><span class="sxs-lookup"><span data-stu-id="ce976-142">However, when the iterator method is translated to CLR metadata, it is a type and multiple methods.</span></span>
