# <a name="compiler-error-cs0151"></a><span data-ttu-id="54027-102">Ошибка компилятора CS0151</span><span class="sxs-lookup"><span data-stu-id="54027-102">Compiler Error CS0151</span></span>
<span data-ttu-id="54027-103">Требуется значение целочисленного типа</span><span class="sxs-lookup"><span data-stu-id="54027-103">A value of an integral type expected</span></span>  
  
 <span data-ttu-id="54027-104">В ситуации, когда требуется целочисленный тип, была использована переменная.</span><span class="sxs-lookup"><span data-stu-id="54027-104">A variable was used in a situation where an integral data type was required.</span></span> <span data-ttu-id="54027-105">Дополнительные сведения см. в разделе [Типы](../../../csharp/programming-guide/types/index.md).</span><span class="sxs-lookup"><span data-stu-id="54027-105">For more information, see [Types](../../../csharp/programming-guide/types/index.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="54027-106">Пример</span><span class="sxs-lookup"><span data-stu-id="54027-106">Example</span></span>  
 <span data-ttu-id="54027-107">Эта ошибка может возникнуть при отсутствии преобразования или в том случае, если доступные неявные преобразования приводят к возникновению неоднозначности.</span><span class="sxs-lookup"><span data-stu-id="54027-107">This error can occur when there is no conversion or if the available implicit conversions result in an ambiguous situation.</span></span> <span data-ttu-id="54027-108">В следующем примере возникает ошибка CS0151.</span><span class="sxs-lookup"><span data-stu-id="54027-108">The following sample generates CS0151.</span></span>  
  
```csharp  
// CS0151.cs  
public class MyClass  
{  
   public static implicit operator int (MyClass aa)  
   {  
      return 0;  
   }  
  
   public static implicit operator long (MyClass aa)  
   {  
      return 0;  
   }  
  
   public static void Main()  
   {  
      MyClass a = new MyClass();  
  
      // Compiler cannot choose between int and long  
      switch (a)   // CS0151  
      // try the following line instead  
      // switch ((int)a)  
      {  
         case 1:  
            break;  
      }  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="54027-109">Пример</span><span class="sxs-lookup"><span data-stu-id="54027-109">Example</span></span>  
 <span data-ttu-id="54027-110">В Visual Studio 2008 и более поздних версий ошибка CS0151 возникает при вызове метода [void](../../../csharp/language-reference/keywords/void.md).</span><span class="sxs-lookup"><span data-stu-id="54027-110">In Visual Studio 2008 and later, a [void](../../../csharp/language-reference/keywords/void.md) method invocation generates CS0151.</span></span> <span data-ttu-id="54027-111">Чтобы исправить эту ошибку, вызовите метод, возвращающий целочисленное значение, например [int](../../../csharp/language-reference/keywords/int.md) или [long](../../../csharp/language-reference/keywords/long.md).</span><span class="sxs-lookup"><span data-stu-id="54027-111">You can fix the error by calling a method that returns an integral type such as [int](../../../csharp/language-reference/keywords/int.md) or [long](../../../csharp/language-reference/keywords/long.md).</span></span>  
  
```csharp  
class C  
{  
    static void Main()  
    {  
  
        switch (M()) // CS0151  
        {  
            default:  
                break;  
        }  
    }  
  
    static void M()  
    {  
    }  
}  
```
