<span data-ttu-id="9891b-103">Неоднозначность между методом и другим атрибутом, имя которого совпадает с методом.</span><span class="sxs-lookup"><span data-stu-id="9891b-103">Ambiguity between method 'method' and non-method 'non-method'.</span></span> <span data-ttu-id="9891b-104">Использование группы методов.</span><span class="sxs-lookup"><span data-stu-id="9891b-104">Using method group.</span></span>  
  
 <span data-ttu-id="9891b-105">Унаследованные члены из разных интерфейсов, которые имеют одинаковые сигнатуры, вызывают ошибку неоднозначности.</span><span class="sxs-lookup"><span data-stu-id="9891b-105">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="9891b-106">Пример</span><span class="sxs-lookup"><span data-stu-id="9891b-106">Example</span></span>  
 <span data-ttu-id="9891b-107">Следующий пример приводит к возникновению ошибки CS0467.</span><span class="sxs-lookup"><span data-stu-id="9891b-107">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);
        // To resolve the warning, you can change the name of the method or
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()
    {  
    }  
}  
```
