 <span data-ttu-id="ca8ad-106">Это предупреждение может возникать, если два метода интерфейса отличаются только тем, как помечен определенный параметр: `ref` или `out`.</span><span class="sxs-lookup"><span data-stu-id="ca8ad-106">This warning can be generated when two interface methods are differentiated only by whether a particular parameter is marked with `ref` or with `out`.</span></span> <span data-ttu-id="ca8ad-107">Рекомендуется изменить код, чтобы предотвратить появление этого предупреждения, поскольку нельзя однозначно определить, какой метод будет вызван во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="ca8ad-107">It is best to change your code to avoid this warning because it is not obvious or guaranteed which method is called at runtime.</span></span>

## <a name="example"></a><span data-ttu-id="ca8ad-113">Пример</span><span class="sxs-lookup"><span data-stu-id="ca8ad-113">Example</span></span>  
 <span data-ttu-id="ca8ad-114">В следующем коде создается предупреждение CS1956, поскольку два метода `Test` класса `Base` отличаются только модификаторами `ref`/`out` первого параметра.</span><span class="sxs-lookup"><span data-stu-id="ca8ad-114">The following code generates CS1956 because the two `Test` methods in `Base` differ only by the `ref`/`out` modifier on the first parameter.</span></span>  
  
```csharp  
// cs1956.cs  
class Base<T, S>  
{  
    // This is the method that should be called.  
    public virtual int Test(out T x) // CS1956  
    {  
        x = default(T);  
        return 0;  
    }  
  
    // This is the "last" method and is the one that ends up being called  
    public virtual int Test(ref S x)  
    {  
        return 1;  
    }  
}  
  
interface IFace  
{  
    int Test(out int x);  
}  
  
class Derived : Base<int, int>, IFace  
{  
    static int Main()  
    {  
        IFace x = new Derived();  
        int y;  
        return x.Test(out y);  
    }  
}  
```
