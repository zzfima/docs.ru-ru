# <a name="compiler-error-cs0413"></a><span data-ttu-id="ecfa5-102">Ошибка компилятора CS0413</span><span class="sxs-lookup"><span data-stu-id="ecfa5-102">Compiler Error CS0413</span></span>
<span data-ttu-id="ecfa5-103">Параметр типа "параметр типа" не может использоваться с оператором "as", так как он не имеет ни ограничений типа класса, ни ограничения "class"</span><span class="sxs-lookup"><span data-stu-id="ecfa5-103">The type parameter 'type parameter' cannot be used with the 'as' operator because it does not have a class type constraint nor a 'class' constraint</span></span>  
  
 <span data-ttu-id="ecfa5-104">Эта ошибка возникает в том случае, если универсальный тип использует оператор [as](../../../csharp/language-reference/keywords/as.md), но при этом не имеет ограничений типа класса.</span><span class="sxs-lookup"><span data-stu-id="ecfa5-104">This error occurs if a generic type uses the [as](../../../csharp/language-reference/keywords/as.md) operator, but that generic type does not have a class type constraint.</span></span> <span data-ttu-id="ecfa5-105">Оператор `as` можно использовать только со ссылочными типами, поэтому необходимо определить ограничение для параметра типа, гарантирующее, что он не будет иметь тип значения.</span><span class="sxs-lookup"><span data-stu-id="ecfa5-105">The `as` operator is only allowed with reference types, so the type parameter must be constrained to guarantee that it is not a value type.</span></span> <span data-ttu-id="ecfa5-106">Чтобы избежать этой ошибки, используйте ограничение типа класса или ограничение ссылочного типа.</span><span class="sxs-lookup"><span data-stu-id="ecfa5-106">To avoid this error, use a class type constraint or a reference type constraint.</span></span>  
  
 <span data-ttu-id="ecfa5-107">Это связано с тем, что оператор `as` может возвращать значение `null`, недопустимое для типа значения, и параметр типа должен обрабатываться как тип значения, кроме случаев, когда задано ограничение типа класса или ограничение ссылочного типа.</span><span class="sxs-lookup"><span data-stu-id="ecfa5-107">This is because the `as` operator could return `null`, which is not a possible value for a value type, and the type parameter must be treated as a value type unless it is a class type constraint or a reference type constraint.</span></span>  
  
## <a name="example"></a><span data-ttu-id="ecfa5-108">Пример</span><span class="sxs-lookup"><span data-stu-id="ecfa5-108">Example</span></span>  
 <span data-ttu-id="ecfa5-109">В следующем примере возникает ошибка CS0413.</span><span class="sxs-lookup"><span data-stu-id="ecfa5-109">The following sample generates CS0413.</span></span>  
  
```csharp  
// CS0413.cs  
// compile with: /target:library  
class A {}  
class B : A {}  
  
class CMain  
{  
   A a = null;  
   public void G<T>()  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  // CS0413  
   }  
  
   // OK  
   public void H<T>() where T : A  
   {  
      a = new A();  
      System.Console.WriteLine (a as T);  
   }  
}  
```
