# <a name="compiler-error-cs1919"></a><span data-ttu-id="e89e9-102">Ошибка компилятора CS1919</span><span class="sxs-lookup"><span data-stu-id="e89e9-102">Compiler Error CS1919</span></span>

 <span data-ttu-id="e89e9-104">Оператор `new` создает объекты только в управляемой куче.</span><span class="sxs-lookup"><span data-stu-id="e89e9-104">The `new` operator creates objects only on the managed heap.</span></span> <span data-ttu-id="e89e9-105">Тем не менее объекты можно создавать в неуправляемой памяти косвенно, используя возможности взаимодействия языка для вызова собственных методов, возвращающих указатели.</span><span class="sxs-lookup"><span data-stu-id="e89e9-105">However, you can create objects in unmanaged memory indirectly by using the interoperability capabilities of the language to call native methods that return pointers.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="e89e9-106">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="e89e9-106">To correct this error</span></span>

## <a name="example"></a><span data-ttu-id="e89e9-110">Пример</span><span class="sxs-lookup"><span data-stu-id="e89e9-110">Example</span></span>

 <span data-ttu-id="e89e9-111">В следующем примере вызывается ошибка CS1919, поскольку тип указателя небезопасен:</span><span class="sxs-lookup"><span data-stu-id="e89e9-111">The following example generates CS1919 because a pointer type is unsafe:</span></span>

```csharp
unsafe public class C  
{  
    public static int Main()  
    {  
        var col1 = new int* { }; // CS1919

        return 1;  
    }  
}  
```
