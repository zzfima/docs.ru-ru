# <a name="compiler-error-cs1919"></a><span data-ttu-id="6369e-102">Ошибка компилятора CS1919</span><span class="sxs-lookup"><span data-stu-id="6369e-102">Compiler Error CS1919</span></span>
<span data-ttu-id="6369e-103">Небезопасный тип "имя типа" не может применяться при создании объекта.</span><span class="sxs-lookup"><span data-stu-id="6369e-103">Unsafe type 'type name' cannot be used in object creation.</span></span>  
  
 <span data-ttu-id="6369e-104">Оператор `new` создает объекты только в управляемой куче.</span><span class="sxs-lookup"><span data-stu-id="6369e-104">The `new` operator creates objects only on the managed heap.</span></span> <span data-ttu-id="6369e-105">Тем не менее объекты можно создавать в неуправляемой памяти косвенно, используя возможности взаимодействия языка для вызова собственных методов, возвращающих указатели.</span><span class="sxs-lookup"><span data-stu-id="6369e-105">However, you can create objects in unmanaged memory indirectly by using the interoperability capabilities of the language to call native methods that return pointers.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="6369e-106">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="6369e-106">To correct this error</span></span>

2.  <span data-ttu-id="6369e-109">Если необходимо создавать объекты в неуправляемой памяти, используйте метод Win32 или COM либо напишите собственную функцию на языке C или C++ и вызовите ее из C#.</span><span class="sxs-lookup"><span data-stu-id="6369e-109">If you must create objects in unmanaged memory, use a Win32 or COM method or else write your own function in C or C++ and call it from C#.</span></span>  
  
## <a name="example"></a><span data-ttu-id="6369e-110">Пример</span><span class="sxs-lookup"><span data-stu-id="6369e-110">Example</span></span>  
 <span data-ttu-id="6369e-111">В следующем примере вызывается ошибка CS1919, поскольку тип указателя небезопасен:</span><span class="sxs-lookup"><span data-stu-id="6369e-111">The following example generates CS1919 because a pointer type is unsafe:</span></span>  
  
```csharp

// Compile with: /unsafe  
unsafe public class C  
{  
    public static int Main()  
    {  
        var col1 = new int* { }; // CS1919  
        var col2 = new char* { }; // CS1919  
        return 1;  
    }  
}  
```
