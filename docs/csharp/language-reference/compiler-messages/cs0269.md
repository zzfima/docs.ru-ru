# <a name="compiler-error-cs0269"></a><span data-ttu-id="e2e19-102">Ошибка компилятора CS0269</span><span class="sxs-lookup"><span data-stu-id="e2e19-102">Compiler Error CS0269</span></span>
<span data-ttu-id="e2e19-103">Использование выходного параметра "parameter", которому не присвоено значение</span><span class="sxs-lookup"><span data-stu-id="e2e19-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="e2e19-104">Компилятору не удалось проверить, присвоено ли выходному параметру значение до его использования; значение параметра может быть не определено при присвоении.</span><span class="sxs-lookup"><span data-stu-id="e2e19-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="e2e19-105">Прежде чем обратиться к значению параметров `out`, его необходимо присвоить в вызываемом методе.</span><span class="sxs-lookup"><span data-stu-id="e2e19-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="e2e19-106">Если необходимо использовать значение переданной переменной, используйте параметр `ref`.</span><span class="sxs-lookup"><span data-stu-id="e2e19-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="e2e19-107">Дополнительные сведения см. в разделе [Передача параметров](../../programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="e2e19-107">For more information, see [Passing Parameters](../../programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="e2e19-108">Пример</span><span class="sxs-lookup"><span data-stu-id="e2e19-108">Example</span></span>  
 <span data-ttu-id="e2e19-109">Следующий пример приводит к возникновению ошибки CS0269:</span><span class="sxs-lookup"><span data-stu-id="e2e19-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="e2e19-110">Пример</span><span class="sxs-lookup"><span data-stu-id="e2e19-110">Example</span></span>  
 <span data-ttu-id="e2e19-111">Эта ошибка также возникает, если инициализация переменной выполняется в блоке "try", в котором компилятор не может проверить присвоение:</span><span class="sxs-lookup"><span data-stu-id="e2e19-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
