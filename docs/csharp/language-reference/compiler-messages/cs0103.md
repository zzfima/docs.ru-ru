# <a name="compiler-error-cs0103"></a><span data-ttu-id="f6c9f-102">Ошибка компилятора CS0103</span><span class="sxs-lookup"><span data-stu-id="f6c9f-102">Compiler Error CS0103</span></span>

<span data-ttu-id="f6c9f-103">Имя "identifier" не существует в текущем контексте</span><span class="sxs-lookup"><span data-stu-id="f6c9f-103">The name 'identifier' does not exist in the current context</span></span>

 <span data-ttu-id="f6c9f-104">Предпринята попытка использовать имя, которое не существует в классе, [пространстве имен](../keywords/namespace.md) или области.</span><span class="sxs-lookup"><span data-stu-id="f6c9f-104">An attempt was made to use a name that does not exist in the class, [namespace](../keywords/namespace.md), or scope.</span></span> <span data-ttu-id="f6c9f-105">Проверьте правильность написания имени и директивы using и ссылки на сборки, чтобы убедиться, что это имя доступно.</span><span class="sxs-lookup"><span data-stu-id="f6c9f-105">Check the spelling of the name and check your using directives and assembly references to make sure that the name that you are trying to use is available.</span></span>

 <span data-ttu-id="f6c9f-106">Эта ошибка часто возникает при объявлении переменной в цикле или в блоке `try` или `if`, если попытаться обратиться к ней из включающего блока кода или отдельного блока кода, как показано в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="f6c9f-106">This error frequently occurs if you declare a variable in a loop or a `try` or `if` block and then attempt to access it from an enclosing code block or a separate code block, as shown in the following example:</span></span>

```csharp
using System;

class MyClass1
{
    public static void Main()
    {
        try
        {
            // The following declaration is only available inside the try block.
            var conn = new MyClass1();
        }
        catch (Exception e)
        {  
            // The following expression causes error CS0103, because variable
            // conn only exists in the try block.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```

 <span data-ttu-id="f6c9f-107">Эта ошибка разрешена в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="f6c9f-107">The following example resolves the error:</span></span>

```csharp
using System;

class MyClass2
{
    public static void Main()
    {
        // To resolve the error in the example, the first step is to
        // move the declaration of conn out of the try block. The following
        // declaration is available throughout the Main method.
        MyClass2 conn = null;
        try
        {
            // Inside the try block, use the conn variable that you declared
            // previously.
            conn = new MyClass2();
        }
        catch (Exception e)
        {
            // The following expression no longer causes an error, because
            // the declaration of conn is in scope.
            if (conn != null)
                Console.WriteLine("{0}", e);
        }
    }
}
```
