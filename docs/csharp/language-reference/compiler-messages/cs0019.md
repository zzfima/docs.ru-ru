# <a name="compiler-error-cs0019"></a><span data-ttu-id="44244-102">Ошибка компилятора CS0019</span><span class="sxs-lookup"><span data-stu-id="44244-102">Compiler Error CS0019</span></span>
<span data-ttu-id="44244-103">Оператор "оператор" не может применяться к операнду типа "тип" и "тип"</span><span class="sxs-lookup"><span data-stu-id="44244-103">Operator 'operator' cannot be applied to operands of type 'type' and 'type'</span></span>  
  
 <span data-ttu-id="44244-104">Бинарный оператор применяется к типам данных, которые не поддерживают его.</span><span class="sxs-lookup"><span data-stu-id="44244-104">A binary operator is applied to data types that do not support it.</span></span> <span data-ttu-id="44244-105">Например, оператор [&#124;&#124;](../../../csharp/language-reference/operators/conditional-or-operator.md) нельзя применять к строкам, операторы [+](../../../csharp/language-reference/operators/addition-operator.md), [-](../../../csharp/language-reference/operators/subtraction-operator.md), [\<](../../../csharp/language-reference/operators/less-than-operator.md) или [>](../../../csharp/language-reference/operators/greater-than-operator.md) к переменным типа [bool](../../../csharp/language-reference/keywords/bool.md), а оператор [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) нельзя использовать с типом `struct`, если этот тип не перегружает оператор явно.</span><span class="sxs-lookup"><span data-stu-id="44244-105">For example, you cannot use the [&#124;&#124;](../../../csharp/language-reference/operators/conditional-or-operator.md) operator on strings, you cannot use [+](../../../csharp/language-reference/operators/addition-operator.md) , [-](../../../csharp/language-reference/operators/subtraction-operator.md) , [\<](../../../csharp/language-reference/operators/less-than-operator.md) , or [>](../../../csharp/language-reference/operators/greater-than-operator.md) operators on [bool](../../../csharp/language-reference/keywords/bool.md) variables, and you cannot use the [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) operator with a `struct` type unless the type explicitly overloads that operator.</span></span>  
  
 <span data-ttu-id="44244-106">Если эта ошибка возникает для типа класса, это значит, что класс не перегружает оператор.</span><span class="sxs-lookup"><span data-stu-id="44244-106">If you encounter this error with a class type, it is because the class does not overload the operator.</span></span> <span data-ttu-id="44244-107">Дополнительные сведения см. в разделе [Перегружаемые операторы](../../../csharp/programming-guide/statements-expressions-operators/overloadable-operators.md).</span><span class="sxs-lookup"><span data-stu-id="44244-107">For more information, see [Overloadable Operators](../../../csharp/programming-guide/statements-expressions-operators/overloadable-operators.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="44244-108">Пример</span><span class="sxs-lookup"><span data-stu-id="44244-108">Example</span></span>  
 <span data-ttu-id="44244-109">В следующем примере ошибка CS0019 возникает в двух местах, поскольку тип [bool](../../../csharp/language-reference/keywords/bool.md) в C# не допускает преобразование в [int](../../../csharp/language-reference/keywords/int.md). Кроме того, ошибка CS0019 возникает при попытке применить оператор вычитания к строке.</span><span class="sxs-lookup"><span data-stu-id="44244-109">In the following example, CS0019 is generated in two places because [bool](../../../csharp/language-reference/keywords/bool.md) in C# is not convertible to [int](../../../csharp/language-reference/keywords/int.md). CS0019 also is generated when the subtraction operator is applied to a string.</span></span> <span data-ttu-id="44244-110">Оператор сложения (+) можно использовать со строковыми операндами, поскольку он перегружен в классе `String`, где реализует объединение строк.</span><span class="sxs-lookup"><span data-stu-id="44244-110">The addition operator (+) can be used with string operands because that operator is overloaded by the `String` class to perform string concatenation.</span></span>  
  
```csharp  
static void Main()  
{  
    bool result = true;  
    if (result > 0) //CS0019  
    {  
        // Do something.  
    }  
  
    int i = 1;  
    // You cannot compare an integer and a boolean value.  
    if (i == true) //CS0019  
    {  
        //Do something...  
    }  
  
    // The following use of == causes no error. It is the comparison of  
    // an integer and a boolean value that causes the error in the   
    // previous if statement.  
    if (result == true)  
    {  
        //Do something...  
    }  
  
    string s = "Just try to subtract me.";  
    float f = 100 - s; // CS0019  
}  
```  
  
## <a name="example"></a><span data-ttu-id="44244-111">Пример</span><span class="sxs-lookup"><span data-stu-id="44244-111">Example</span></span>  
 <span data-ttu-id="44244-112">В следующем примере условная логика должна задаваться вне <xref:System.Diagnostics.ConditionalAttribute>.</span><span class="sxs-lookup"><span data-stu-id="44244-112">In the following example, conditional logic must be specified outside the <xref:System.Diagnostics.ConditionalAttribute>.</span></span> <span data-ttu-id="44244-113">В <xref:System.Diagnostics.ConditionalAttribute> можно передать только один предварительно определенный символ.</span><span class="sxs-lookup"><span data-stu-id="44244-113">You can pass only one predefined symbol to the <xref:System.Diagnostics.ConditionalAttribute>.</span></span>  
  
 <span data-ttu-id="44244-114">В следующем примере возникает ошибка CS0019.</span><span class="sxs-lookup"><span data-stu-id="44244-114">The following sample generates CS0019.</span></span>  
  
```csharp  
// CS0019_a.cs  
// compile with: /target:library  
using System.Diagnostics;  
public class MyClass  
{  
   [ConditionalAttribute("DEBUG" || "TRACE")]   // CS0019  
   public void TestMethod() {}  
  
   // OK  
   [ConditionalAttribute("DEBUG"), ConditionalAttribute("TRACE")]  
   public void TestMethod2() {}  
}  
```
