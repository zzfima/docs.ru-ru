## <a name="to-correct-this-error"></a><span data-ttu-id="4aa2e-105">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="4aa2e-105">To correct this error</span></span>  
  
1. <span data-ttu-id="4aa2e-106">Обратите внимание, что первый параметр определения метода расширения указывает, какой тип метод будет "расширять".</span><span class="sxs-lookup"><span data-stu-id="4aa2e-106">Remember that the first parameter of an extension method definition specifies which type the method will "extend".</span></span> <span data-ttu-id="4aa2e-107">Это не входной параметр.</span><span class="sxs-lookup"><span data-stu-id="4aa2e-107">It is not an input parameter.</span></span> <span data-ttu-id="4aa2e-108">Таким образом, в этом месте массив параметров не имеет смысла.</span><span class="sxs-lookup"><span data-stu-id="4aa2e-108">Therefore, it makes no sense to have a params array in this location.</span></span> <span data-ttu-id="4aa2e-109">Если нужно передать массив параметров, передайте его вторым параметром.</span><span class="sxs-lookup"><span data-stu-id="4aa2e-109">If you do have to pass in a params array, make it the second parameter.</span></span>
