# <a name="compiler-error-cs1931"></a><span data-ttu-id="69ae8-102">Ошибка компилятора CS1931</span><span class="sxs-lookup"><span data-stu-id="69ae8-102">Compiler Error CS1931</span></span>
<span data-ttu-id="69ae8-103">Переменная диапазона "переменная" конфликтует с предыдущим объявлением переменной "переменная".</span><span class="sxs-lookup"><span data-stu-id="69ae8-103">The range variable 'variable' conflicts with a previous declaration of 'variable'.</span></span>  
  
 <span data-ttu-id="69ae8-104">Объявление переменной диапазона, как и любое другие объявление, должно иметь идентификатор, который является уникальным в области объявления переменной.</span><span class="sxs-lookup"><span data-stu-id="69ae8-104">The declaration of a range variable, just like every other declaration, must have an identifier which is unique within the variable's declaration space.</span></span>

## <a name="example"></a><span data-ttu-id="69ae8-107">Пример</span><span class="sxs-lookup"><span data-stu-id="69ae8-107">Example</span></span>  
 <span data-ttu-id="69ae8-108">Следующий код создает ошибку CS1931, так как идентификатор `x` используется и как локальная переменная в `Main` , и как переменная диапазона в выражении запроса:</span><span class="sxs-lookup"><span data-stu-id="69ae8-108">The following code generates CS1931 because the identifier `x` is used both as a local variable in `Main` and as the range variable in the query expression:</span></span>  
  
```csharp  
// cs1931.cs  
class Test  
    {  
        static void Main()  
        {  
            int x = 1;  
            var y = from x in Enumerable.Range(1, 100) // CS1931  
                    select x;  
        }  
    }  
```
