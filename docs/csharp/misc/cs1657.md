# <a name="compiler-error-cs1657"></a>Ошибка компилятора CS1657
Невозможно передать "параметр" как аргумент с ключевым словом ref или out, потому что "причина"  
  
 Эта ошибка возникает при передаче переменной как [ref](../../csharp/language-reference/keywords/ref.md) или [out](../../csharp/language-reference/keywords/out-parameter-modifier.md) аргумента в контексте, в котором переменная доступна только для чтения. Контексты только для чтения включают переменные итерации [foreach](../../csharp/language-reference/keywords/foreach-in.md) , переменные [using](../../csharp/language-reference/keywords/using-statement.md) и переменные `fixed` . Чтобы устранить эту ошибку, не вызывайте функции, принимающие переменную `foreach`, `using` или `fixed` в качестве параметра `ref` или `out` в блоках `using` , операторах `foreach` и `fixed` .  
  
## <a name="example"></a>Пример  
 Следующий пример приводит к возникновению ошибки CS1657:  
  
```csharp  
// CS1657.cs  
using System;  
class C : IDisposable  
{  
    public int i;  
    public void Dispose() {}  
}  
  
class CMain  
{  
    static void f(ref C c)  
    {  
    }  
    static void Main()  
    {  
        using (C c = new C())  
        {  
            f(ref c);  // CS1657  
        }  
    }  
}  
```  
  
## <a name="example"></a>Пример  
 Следующий код иллюстрирует ту же проблему в операторе `fixed` :  
  
```csharp  
// CS1657b.cs  
// compile with: /unsafe  
unsafe class C  
{  
    static void F(ref int* p)  
    {  
    }  
  
    static void Main()  
    {  
        int[] a = new int[5];  
        fixed(int* p = a) F(ref p); // CS1657  
    }  
}  
```
