# <a name="compiler-error-cs0748"></a><span data-ttu-id="d14fa-102">Ошибка компилятора CS0748</span><span class="sxs-lookup"><span data-stu-id="d14fa-102">Compiler Error CS0748</span></span>
<span data-ttu-id="d14fa-103">Несогласованное использование лямбда-параметра; типы параметров должны быть либо все явными, либо все неявными.</span><span class="sxs-lookup"><span data-stu-id="d14fa-103">Inconsistent lambda parameter usage; all parameter types must either be explicit or implicit.</span></span>  
  
 <span data-ttu-id="d14fa-104">Если лямбда-выражение имеет несколько входных параметров, не разрешается, чтобы одни параметры использовали явную типизацию, а другие неявную.</span><span class="sxs-lookup"><span data-stu-id="d14fa-104">If a lambda expression has multiple input parameters, some parameters cannot use implicit typing while others use explicit typing.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="d14fa-105">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="d14fa-105">To correct this error</span></span>

## <a name="example"></a><span data-ttu-id="d14fa-107">Пример</span><span class="sxs-lookup"><span data-stu-id="d14fa-107">Example</span></span>  
 <span data-ttu-id="d14fa-108">В следующем коде возникает ошибка CS0748, поскольку в лямбда-выражении только параметру `alpha` присваивается явный тип:</span><span class="sxs-lookup"><span data-stu-id="d14fa-108">The following code generates CS0748 because, in the lambda expression, only `alpha` is given an explicit type:</span></span>

```csharp  
{  
    delegate double D(int x, int y);  
    D d = (int alpha, beta) => { return beta / alpha; }; // CS0748  
}  
```
