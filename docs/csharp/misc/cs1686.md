# <a name="compiler-error-cs1686"></a><span data-ttu-id="7149c-102">Ошибка компилятора CS1686</span><span class="sxs-lookup"><span data-stu-id="7149c-102">Compiler Error CS1686</span></span>
<span data-ttu-id="7149c-103">Адрес локальной "переменной" или ее членов не может быть получен или использован в анонимном методе или лямбда-выражении</span><span class="sxs-lookup"><span data-stu-id="7149c-103">Local 'variable' or its members cannot have their address taken and be used inside an anonymous method or lambda expression</span></span>  
  
 <span data-ttu-id="7149c-104">Эта ошибка появляется, если вы используете переменную и пытаетесь получить его адрес, и одно из этих действий выполняется внутри анонимного метода.</span><span class="sxs-lookup"><span data-stu-id="7149c-104">This error is generated when you use a variable, and attempt to take its address, and one of these actions is done inside an anonymous method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="7149c-105">Пример</span><span class="sxs-lookup"><span data-stu-id="7149c-105">Example</span></span>  
 <span data-ttu-id="7149c-106">Следующий пример приводит к возникновению ошибки CS1686.</span><span class="sxs-lookup"><span data-stu-id="7149c-106">The following sample generates CS1686.</span></span>  
  
```csharp  
// CS1686.cs  
// compile with: /unsafe /target:library  
class MyClass  
{  
   public unsafe delegate int * MyDelegate();  
  
   public unsafe int * Test()  
   {  
      int j = 0;  
      MyDelegate d = delegate { return &j; };   // CS1686  
      return &j;   // OK  
   }  
}  
```
