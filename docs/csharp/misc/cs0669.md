# <a name="compiler-error-cs0669"></a><span data-ttu-id="a12dd-102">Ошибка компилятора CS0669</span><span class="sxs-lookup"><span data-stu-id="a12dd-102">Compiler Error CS0669</span></span>

<span data-ttu-id="a12dd-103">Класс с атрибутом ComImport не может иметь определенного пользователем конструктора.</span><span class="sxs-lookup"><span data-stu-id="a12dd-103">A class with the ComImport attribute cannot have a user-defined constructor.</span></span>

<span data-ttu-id="a12dd-104">Уровень COM-взаимодействия в среде CLR предоставляет конструктор для классов [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) .</span><span class="sxs-lookup"><span data-stu-id="a12dd-104">The COM interop layer in the common language runtime supplies the constructor for [ComImport](xref:System.Runtime.InteropServices.ComImportAttribute) classes.</span></span> <span data-ttu-id="a12dd-105">Следовательно, COM-объект может использоваться в качестве управляемого объекта во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="a12dd-105">Consequently, a COM object can be used as a managed object in the runtime.</span></span>

<span data-ttu-id="a12dd-106">При компиляции следующего примера возникнет ошибка CS0669:</span><span class="sxs-lookup"><span data-stu-id="a12dd-106">The following sample generates CS0669:</span></span>

```csharp
// CS0669.cs
using System.Runtime.InteropServices;
[ComImport, Guid("00000000-0000-0000-0000-000000000001")]
class TestClass
{
   TestClass()   // CS0669, delete constructor to resolve
   {
   }

   public static void Main()
   {
   }
}
```
