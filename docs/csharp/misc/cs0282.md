# <a name="compiler-warning-level-3-cs0282"></a><span data-ttu-id="e8637-102">Предупреждение компилятора (уровень 3) CS0282</span><span class="sxs-lookup"><span data-stu-id="e8637-102">Compiler Warning (level 3) CS0282</span></span>
<span data-ttu-id="e8637-103">Не определен порядок полей в нескольких декларациях разделяемого класса или типа "тип" структуры.</span><span class="sxs-lookup"><span data-stu-id="e8637-103">There is no defined ordering between fields in multiple declarations of partial class or struct 'type'.</span></span> <span data-ttu-id="e8637-104">Для указания порядка все поля экземпляра должны быть в одном объявлении.</span><span class="sxs-lookup"><span data-stu-id="e8637-104">To specify an ordering, all instance fields must be in the same declaration.</span></span>

 <span data-ttu-id="e8637-106">Распространенной причиной этой ошибки является определение частичной `struct` в нескольких местах, причем часть переменных-членов находится в одном определении, а часть — в другом.</span><span class="sxs-lookup"><span data-stu-id="e8637-106">A common way to get this error is by having a partial `struct` defined in more than one place, with some of the member variables in one definition, and some in another.</span></span>
