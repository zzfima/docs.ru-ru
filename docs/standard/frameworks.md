<span data-ttu-id="798b0-104">При выборе целевой платформы для приложения или библиотеки вы указываете набор API-интерфейсов, которые вы хотите сделать доступными для приложения или библиотеки.</span><span class="sxs-lookup"><span data-stu-id="798b0-104">When you target a framework in an app or library, you're specifying the set of APIs that you'd like to make available to the app or library.</span></span> <span data-ttu-id="798b0-105">Целевая платформа указывается в файле проекта с помощью моникеров целевой платформы (TFM).</span><span class="sxs-lookup"><span data-stu-id="798b0-105">You specify the target framework in your project file using Target Framework Monikers (TFMs).</span></span>

<span data-ttu-id="798b0-109">Приложение или библиотека могут также работать в конкретной реализации .NET. <span data-ttu-id="798b0-111">Для некоторых целевых платформ (например, .NET Framework) API-интерфейсы определяются сборками, устанавливаемыми платформой в системе, в число которых могут входить API-интерфейсы платформы приложений (например, ASP.NET).</span><span class="sxs-lookup"><span data-stu-id="798b0-111">For some target frameworks (for example, the .NET Framework), the APIs are defined by the assemblies that the framework installs on a system and may include application framework APIs (for example, ASP.NET).</span></span>

<span data-ttu-id="798b0-112">Для целевых платформ на основе пакетов (например, .NET Standard и .NET Core) API-интерфейсы определяются пакетами в составе приложения или библиотеки.</span><span class="sxs-lookup"><span data-stu-id="798b0-112">For package-based target frameworks (for example, .NET Standard and .NET Core), the APIs are defined by the packages included in the app or library.</span></span> <span data-ttu-id="798b0-113">*Метапакет* — это пакет NuGet, не имеющий собственного содержимого, но имеющий список зависимостей (другие пакеты).</span><span class="sxs-lookup"><span data-stu-id="798b0-113">A *metapackage* is a NuGet package that has no content of its own but is a list of dependencies (other packages).</span></span> <span data-ttu-id="798b0-114">В этом случае целевая платформа на основе пакетов NuGet неявно задает метапакет, который ссылается на все пакеты, составляющие платформу.</span><span class="sxs-lookup"><span data-stu-id="798b0-114">A NuGet package-based target framework implicitly specifies a metapackage that references all the packages that together make up the framework.</span></span>

## <a name="how-to-specify-target-frameworks"></a><span data-ttu-id="798b0-194">Как указать целевые платформы</span><span class="sxs-lookup"><span data-stu-id="798b0-194">How to specify target frameworks</span></span>

<span data-ttu-id="798b0-195">Целевые платформы указываются в файле проекта.</span><span class="sxs-lookup"><span data-stu-id="798b0-195">Target frameworks are specified in your project file.</span></span> <span data-ttu-id="798b0-196">Если указана одна целевая платформа, используйте элемент **TargetFramework**.</span><span class="sxs-lookup"><span data-stu-id="798b0-196">When a single target framework is specified, use the **TargetFramework** element.</span></span> <span data-ttu-id="798b0-197">В следующем файле проекта консольного приложения показано, как указать целевую платформу .NET Core 2.2:</span><span class="sxs-lookup"><span data-stu-id="798b0-197">The following console app project file demonstrates how to target .NET Core 2.2:</span></span>

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp2.2</TargetFramework>
  </PropertyGroup>

</Project>
```

<span data-ttu-id="798b0-198">При указании нескольких целевых платформ можно условно ссылаться на сборки для каждой целевой платформы.</span><span class="sxs-lookup"><span data-stu-id="798b0-198">When you specify multiple target frameworks, you may conditionally reference assemblies for each target framework.</span></span> <span data-ttu-id="798b0-199">В коде можно условно компилировать эти сборки с использованием символов препроцессора с логикой *if-then-else*.</span><span class="sxs-lookup"><span data-stu-id="798b0-199">In your code, you can conditionally compile against those assemblies by using preprocessor symbols with *if-then-else* logic.</span></span>

<span data-ttu-id="798b0-200">Следующий файл проекта библиотеки предназначен для API-интерфейсов .NET Standard (`netstandard1.4`) и API-интерфейсов .NET Framework (`net40` и `net45`).</span><span class="sxs-lookup"><span data-stu-id="798b0-200">The following library project file targets APIs of .NET Standard (`netstandard1.4`) and APIs of the .NET Framework (`net40` and `net45`).</span></span> <span data-ttu-id="798b0-201">Используйте множественный элемент **TargetFrameworks** с несколькими целевыми платформами.</span><span class="sxs-lookup"><span data-stu-id="798b0-201">Use the plural **TargetFrameworks** element with multiple target frameworks.</span></span> <span data-ttu-id="798b0-202">Обратите внимание, каким образом атрибуты `Condition` включают пакеты, связанные с конкретной реализацией, при компиляции библиотеки для двух TFM .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="798b0-202">Note how the `Condition` attributes include implementation-specific packages when the library is compiled for the two .NET Framework TFMs:</span></span>

```xml
<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <TargetFrameworks>netstandard1.4;net40;net45</TargetFrameworks>
  </PropertyGroup>

  <!-- Conditionally obtain references for the .NET Framework 4.0 target -->
  <ItemGroup Condition=" '$(TargetFramework)' == 'net40' ">
    <Reference Include="System.Net" />
  </ItemGroup>

  <!-- Conditionally obtain references for the .NET Framework 4.5 target -->
  <ItemGroup Condition=" '$(TargetFramework)' == 'net45' ">
    <Reference Include="System.Net.Http" />
    <Reference Include="System.Threading.Tasks" />
  </ItemGroup>

</Project>
```

<span data-ttu-id="798b0-203">В библиотеке или приложении следует написать условный код компиляции для каждой целевой платформы.</span><span class="sxs-lookup"><span data-stu-id="798b0-203">Within your library or app, you write conditional code to compile for each target framework:</span></span>

```csharp
public class MyClass
{
    static void Main()
```
