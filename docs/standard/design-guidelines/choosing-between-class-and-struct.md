# <a name="choosing-between-class-and-struct"></a><span data-ttu-id="576ca-102">Выбор между классом и структурой</span><span class="sxs-lookup"><span data-stu-id="576ca-102">Choosing Between Class and Struct</span></span>
<span data-ttu-id="576ca-103">Одним из основных решений по проектированию каждого разработчика инфраструктуры является необходимость разрабатывать тип как класс (ссылочный тип) или как структура (тип значения).</span><span class="sxs-lookup"><span data-stu-id="576ca-103">One of the basic design decisions every framework designer faces is whether to design a type as a class (a reference type) or as a struct (a value type).</span></span> <span data-ttu-id="576ca-104">Чтобы сделать это, важно понимать различия в поведении ссылочных типов и типов значений.</span><span class="sxs-lookup"><span data-stu-id="576ca-104">Good understanding of the differences in the behavior of reference types and value types is crucial in making this choice.</span></span>

 <span data-ttu-id="576ca-105">Первое различие между ссылочными типами и типами значений, которое мы будем рассматривать, заключается в том, что ссылочные типы выделяются в куче и уничтожаются сборщиком мусора, тогда как типы значений выделяются либо в стеке, либо в виде встроенных строк, содержащих типы, и освобождаются, когда стек очищается или когда их вмещающий тип освобождается.</span><span class="sxs-lookup"><span data-stu-id="576ca-105">The first difference between reference types and value types we will consider is that reference types are allocated on the heap and garbage-collected, whereas value types are allocated either on the stack or inline in containing types and deallocated when the stack unwinds or when their containing type gets deallocated.</span></span> <span data-ttu-id="576ca-106">Таким образом, выделение и освобождение типов значений являются общими дешевле, чем выделение и освобождение ссылочных типов.</span><span class="sxs-lookup"><span data-stu-id="576ca-106">Therefore, allocations and deallocations of value types are in general cheaper than allocations and deallocations of reference types.</span></span>

 <span data-ttu-id="576ca-107">Затем массивы ссылочных типов выделяются вне строки, что означает, что элементы массива являются просто ссылками на экземпляры ссылочного типа, находящегося в куче.</span><span class="sxs-lookup"><span data-stu-id="576ca-107">Next, arrays of reference types are allocated out-of-line, meaning the array elements are just references to instances of the reference type residing on the heap.</span></span> <span data-ttu-id="576ca-108">Массивы типов значений выделяются встроенным, то есть элементы массива являются реальными экземплярами типа значения.</span><span class="sxs-lookup"><span data-stu-id="576ca-108">Value type arrays are allocated inline, meaning that the array elements are the actual instances of the value type.</span></span> <span data-ttu-id="576ca-109">Таким образом, выделение и освобождение массивов типов значений значительно дешевле, чем выделение и освобождение массивов ссылочного типа.</span><span class="sxs-lookup"><span data-stu-id="576ca-109">Therefore, allocations and deallocations of value type arrays are much cheaper than allocations and deallocations of reference type arrays.</span></span> <span data-ttu-id="576ca-110">Кроме того, в большинстве случаев массивы типов значений представляют гораздо более высокий уровень локализации ссылок.</span><span class="sxs-lookup"><span data-stu-id="576ca-110">In addition, in a majority of cases value type arrays exhibit much better locality of reference.</span></span>

 <span data-ttu-id="576ca-111">Следующее отличие связано с использованием памяти.</span><span class="sxs-lookup"><span data-stu-id="576ca-111">The next difference is related to memory usage.</span></span> <span data-ttu-id="576ca-112">Типы значений упаковываются при приведении к ссылочному типу или к одному из интерфейсов, которые они реализуют.</span><span class="sxs-lookup"><span data-stu-id="576ca-112">Value types get boxed when cast to a reference type or one of the interfaces they implement.</span></span> <span data-ttu-id="576ca-113">Они будут распакованы при приведении обратно к типу значения.</span><span class="sxs-lookup"><span data-stu-id="576ca-113">They get unboxed when cast back to the value type.</span></span> <span data-ttu-id="576ca-114">Так как поля представляют собой объекты, выделенные в куче и собираемые сборщиком мусора, слишком много упаковки и распаковки могут негативно повлиять на кучу, сборщик мусора и в конечном итоге производительность приложения.</span><span class="sxs-lookup"><span data-stu-id="576ca-114">Because boxes are objects that are allocated on the heap and are garbage-collected, too much boxing and unboxing can have a negative impact on the heap, the garbage collector, and ultimately the performance of the application.</span></span>  <span data-ttu-id="576ca-115">В отличие от этого, такая упаковка не происходит, так как ссылочные типы являются приведенными.</span><span class="sxs-lookup"><span data-stu-id="576ca-115">In contrast, no such boxing occurs as reference types are cast.</span></span> <span data-ttu-id="576ca-116">(Дополнительные сведения см. в разделе [Упаковка и распаковка](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span><span class="sxs-lookup"><span data-stu-id="576ca-116">(For more information, see [Boxing and Unboxing](../../csharp/programming-guide/types/boxing-and-unboxing.md)).</span></span>

 <span data-ttu-id="576ca-117">Затем назначения ссылочного типа копируют ссылку, тогда как назначения типа значения копируют все значение.</span><span class="sxs-lookup"><span data-stu-id="576ca-117">Next, reference type assignments copy the reference, whereas value type assignments copy the entire value.</span></span> <span data-ttu-id="576ca-118">Таким образом, присваивания больших ссылочных типов дешевле, чем присваивания типов больших значений.</span><span class="sxs-lookup"><span data-stu-id="576ca-118">Therefore, assignments of large reference types are cheaper than assignments of large value types.</span></span>

 <span data-ttu-id="576ca-119">Наконец, ссылочные типы передаются по ссылке, тогда как типы значений передаются по значению.</span><span class="sxs-lookup"><span data-stu-id="576ca-119">Finally, reference types are passed by reference, whereas value types are passed by value.</span></span> <span data-ttu-id="576ca-120">Изменения в экземпляр ссылочного типа влияют на все ссылки, указывающие на экземпляр.</span><span class="sxs-lookup"><span data-stu-id="576ca-120">Changes to an instance of a reference type affect all references pointing to the instance.</span></span> <span data-ttu-id="576ca-121">Экземпляры типа значения копируются, когда они передаются по значению.</span><span class="sxs-lookup"><span data-stu-id="576ca-121">Value type instances are copied when they are passed by value.</span></span> <span data-ttu-id="576ca-122">При изменении экземпляра типа значения это само по себе не влияет на его копии.</span><span class="sxs-lookup"><span data-stu-id="576ca-122">When an instance of a value type is changed, it of course does not affect any of its copies.</span></span> <span data-ttu-id="576ca-123">Поскольку копии не создаются явным образом пользователем, но неявно создаются при передаче аргументов или возвращении возвращаемых значений, типы значений, которые могут быть изменены, могут вызвать путаницу для многих пользователей.</span><span class="sxs-lookup"><span data-stu-id="576ca-123">Because the copies are not created explicitly by the user but are implicitly created when arguments are passed or return values are returned, value types that can be changed can be confusing to many users.</span></span> <span data-ttu-id="576ca-124">Поэтому типы значений должны быть неизменяемыми.</span><span class="sxs-lookup"><span data-stu-id="576ca-124">Therefore, value types should be immutable.</span></span>

 <span data-ttu-id="576ca-125">Как правило, большинство типов в платформе должны быть классами.</span><span class="sxs-lookup"><span data-stu-id="576ca-125">As a rule of thumb, the majority of types in a framework should be classes.</span></span> <span data-ttu-id="576ca-126">Однако существуют ситуации, в которых характеристики типа значения делают более подходящим использование структур.</span><span class="sxs-lookup"><span data-stu-id="576ca-126">There are, however, some situations in which the characteristics of a value type make it more appropriate to use structs.</span></span>

 <span data-ttu-id="576ca-127">✔️ Рассмотрите возможность определения структуры вместо класса, если экземпляры типа небольшие и часто используются кратковременно или обычно внедряются в другие объекты.</span><span class="sxs-lookup"><span data-stu-id="576ca-127">✔️ CONSIDER defining a struct instead of a class if instances of the type are small and commonly short-lived or are commonly embedded in other objects.</span></span>

 <span data-ttu-id="576ca-128">❌ избежать определения структуры, если тип имеет не все приведенные ниже характеристики.</span><span class="sxs-lookup"><span data-stu-id="576ca-128">❌ AVOID defining a struct unless the type has all of the following characteristics:</span></span>

- <span data-ttu-id="576ca-129">Он логически представляет одно значение, аналогично примитивным типам (`int`, `double`и т. д.).</span><span class="sxs-lookup"><span data-stu-id="576ca-129">It logically represents a single value, similar to primitive types (`int`, `double`, etc.).</span></span>

- <span data-ttu-id="576ca-130">Он имеет размер экземпляра менее 16 байт.</span><span class="sxs-lookup"><span data-stu-id="576ca-130">It has an instance size under 16 bytes.</span></span>

- <span data-ttu-id="576ca-131">Это неизменяемое.</span><span class="sxs-lookup"><span data-stu-id="576ca-131">It is immutable.</span></span>

- <span data-ttu-id="576ca-132">Он не должен быть часто в штучной упаковке.</span><span class="sxs-lookup"><span data-stu-id="576ca-132">It will not have to be boxed frequently.</span></span>

 <span data-ttu-id="576ca-133">Во всех остальных случаях типы следует определять как классы.</span><span class="sxs-lookup"><span data-stu-id="576ca-133">In all other cases, you should define your types as classes.</span></span>
