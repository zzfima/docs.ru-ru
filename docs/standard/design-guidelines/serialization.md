 <span data-ttu-id="bc580-163">**✓ Попробуйте** поддерживающий сериализацию во время выполнения, если к типам, которые будут использоваться при удаленном взаимодействии .NET.</span><span class="sxs-lookup"><span data-stu-id="bc580-163">**✓ CONSIDER** supporting Runtime Serialization if your types will be used with .NET Remoting.</span></span> <span data-ttu-id="bc580-164">Например <xref:System.AddIn?displayProperty=nameWithType> пространства имен использует удаленное взаимодействие .NET и обмениваются таким образом, все типы `System.AddIn` надстроек должны поддерживать сериализацию во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="bc580-164">For example, the <xref:System.AddIn?displayProperty=nameWithType> namespace uses .NET Remoting, and so all types exchanged between `System.AddIn` add-ins need to support Runtime Serialization.</span></span>  
  
 <span data-ttu-id="bc580-165">**✓ Попробуйте** реализации шаблона сериализуемые среды выполнения, если требуется полный контроль над процессом сериализации.</span><span class="sxs-lookup"><span data-stu-id="bc580-165">**✓ CONSIDER** implementing the Runtime Serializable Pattern if you want complete control over the serialization process.</span></span> <span data-ttu-id="bc580-166">Например, в том случае, когда необходимо преобразовать сериализуемые или десериализуемые данные.</span><span class="sxs-lookup"><span data-stu-id="bc580-166">For example, if you want to transform data as it gets serialized or deserialized.</span></span>

    protected Person(SerializationInfo info, StreamingContext context) {  
        ...
