# <a name="property-design"></a><span data-ttu-id="f79c2-102">Разработка свойств</span><span class="sxs-lookup"><span data-stu-id="f79c2-102">Property Design</span></span>
<span data-ttu-id="f79c2-103">Хотя свойства технически похожи на методы, они сильно отличаются в плане их сценариев использования.</span><span class="sxs-lookup"><span data-stu-id="f79c2-103">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="f79c2-104">Они должны отображаться как интеллектуальные поля.</span><span class="sxs-lookup"><span data-stu-id="f79c2-104">They should be seen as smart fields.</span></span> <span data-ttu-id="f79c2-105">Они имеют синтаксис для вызова полей и гибкость методов.</span><span class="sxs-lookup"><span data-stu-id="f79c2-105">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="f79c2-106">✔️ создать свойства только для получения, если вызывающий объект не может изменить значение свойства.</span><span class="sxs-lookup"><span data-stu-id="f79c2-106">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="f79c2-107">Помните, что если тип свойства является изменяемым ссылочным типом, значение свойства можно изменить, даже если это свойство доступно только для получения.</span><span class="sxs-lookup"><span data-stu-id="f79c2-107">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="f79c2-108">❌ не предоставляют свойства или свойства только для установки с методом задания, имеющим более широкие возможности доступа, чем метод получения.</span><span class="sxs-lookup"><span data-stu-id="f79c2-108">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="f79c2-109">Например, не используйте свойства с открытым методом задания и защищенным методом считывания.</span><span class="sxs-lookup"><span data-stu-id="f79c2-109">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="f79c2-110">Если метод получения свойства не может быть предоставлен, реализуйте в качестве метода функцию.</span><span class="sxs-lookup"><span data-stu-id="f79c2-110">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="f79c2-111">Рассмотрите возможность запуска имени метода с `Set` и выполните действия с именем свойства.</span><span class="sxs-lookup"><span data-stu-id="f79c2-111">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="f79c2-112">Например, <xref:System.AppDomain> имеет метод с именем `SetCachePath` вместо того, чтобы иметь свойство только для набора, именуемое `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="f79c2-112">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="f79c2-113">✔️ предоставляют разумные значения по умолчанию для всех свойств, гарантируя, что значения по умолчанию не приводят к появлению бреши в системе безопасности или неэффективному коду.</span><span class="sxs-lookup"><span data-stu-id="f79c2-113">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="f79c2-114">✔️ разрешить задавать свойства в любом порядке, даже если это приводит к временному недопустимому состоянию объекта.</span><span class="sxs-lookup"><span data-stu-id="f79c2-114">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="f79c2-115">Обычно два или более свойства взаимосвязаны с точкой, в которой некоторые значения одного свойства могут быть недопустимыми с учетом значений других свойств одного и того же объекта.</span><span class="sxs-lookup"><span data-stu-id="f79c2-115">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="f79c2-116">В таких случаях исключения, являющиеся результатом недопустимого состояния, должны быть отложены до тех пор, пока взаимосвязанные свойства фактически не будут использоваться объектом вместе.</span><span class="sxs-lookup"><span data-stu-id="f79c2-116">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="f79c2-117">✔️ сохранить предыдущее значение, если метод задания свойства создает исключение.</span><span class="sxs-lookup"><span data-stu-id="f79c2-117">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="f79c2-118">❌ избежать возникновения исключений из методов получения свойств.</span><span class="sxs-lookup"><span data-stu-id="f79c2-118">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="f79c2-119">Методы получения свойств должны быть простыми операциями и не должны иметь никаких предусловий.</span><span class="sxs-lookup"><span data-stu-id="f79c2-119">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="f79c2-120">Если метод получения может вызвать исключение, то, возможно, он будет переработано, чтобы быть методом.</span><span class="sxs-lookup"><span data-stu-id="f79c2-120">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="f79c2-121">Обратите внимание, что это правило не применяется к индексаторам, в которых мы предполагаем исключения в результате проверки аргументов.</span><span class="sxs-lookup"><span data-stu-id="f79c2-121">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="f79c2-122">Разработка индексированных свойств</span><span class="sxs-lookup"><span data-stu-id="f79c2-122">Indexed Property Design</span></span>
 <span data-ttu-id="f79c2-123">Индексированное свойство — это специальное свойство, которое может иметь параметры и может вызываться с помощью специального синтаксиса, аналогичного индексированию массива.</span><span class="sxs-lookup"><span data-stu-id="f79c2-123">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="f79c2-124">Индексированные свойства обычно называются индексаторами.</span><span class="sxs-lookup"><span data-stu-id="f79c2-124">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="f79c2-125">Индексаторы следует использовать только в интерфейсах API, которые предоставляют доступ к элементам в логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="f79c2-125">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="f79c2-126">Например, строка представляет собой набор символов, а индексатор для <xref:System.String?displayProperty=nameWithType> был добавлен для доступа к его символам.</span><span class="sxs-lookup"><span data-stu-id="f79c2-126">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="f79c2-127">✔️ Рассмотрите возможность использования индексаторов для предоставления доступа к данным, хранящимся во внутреннем массиве.</span><span class="sxs-lookup"><span data-stu-id="f79c2-127">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="f79c2-128">✔️ Рассмотрите возможность предоставления индексаторов для типов, представляющих коллекции элементов.</span><span class="sxs-lookup"><span data-stu-id="f79c2-128">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="f79c2-129">❌ Избегайте использования индексированных свойств более чем с одним параметром.</span><span class="sxs-lookup"><span data-stu-id="f79c2-129">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="f79c2-130">Если для проектирования требуется несколько параметров, следует подумать, действительно ли свойство представляет метод доступа к логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="f79c2-130">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="f79c2-131">Если это не так, используйте вместо них методы.</span><span class="sxs-lookup"><span data-stu-id="f79c2-131">If it does not, use methods instead.</span></span> <span data-ttu-id="f79c2-132">Попробуйте запустить имя метода с помощью `Get` или `Set`.</span><span class="sxs-lookup"><span data-stu-id="f79c2-132">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="f79c2-133">❌ избегать использования индексаторов с типами параметров, отличными от <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>или перечисления.</span><span class="sxs-lookup"><span data-stu-id="f79c2-133">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="f79c2-134">Если для разработки требуются другие типы параметров, настоятельно повторно оцените, действительно ли API представляет метод доступа к логической коллекции.</span><span class="sxs-lookup"><span data-stu-id="f79c2-134">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="f79c2-135">В противном случае используйте метод.</span><span class="sxs-lookup"><span data-stu-id="f79c2-135">If it does not, use a method.</span></span> <span data-ttu-id="f79c2-136">Попробуйте запустить имя метода с помощью `Get` или `Set`.</span><span class="sxs-lookup"><span data-stu-id="f79c2-136">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="f79c2-137">✔️ использовать имя `Item` для индексированных свойств, если нет очевидного лучшего имени (например, см. свойство <xref:System.String.Chars%2A> в `System.String`).</span><span class="sxs-lookup"><span data-stu-id="f79c2-137">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="f79c2-138">В C#индексаторы по умолчанию имеют именованный элемент.</span><span class="sxs-lookup"><span data-stu-id="f79c2-138">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="f79c2-139">Для настройки этого имени можно использовать <xref:System.Runtime.CompilerServices.IndexerNameAttribute>.</span><span class="sxs-lookup"><span data-stu-id="f79c2-139">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="f79c2-140">❌ не предоставляют и индексатор, и методы, которые являются семантически эквивалентными.</span><span class="sxs-lookup"><span data-stu-id="f79c2-140">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="f79c2-141">❌ не предоставляют более одного семейства перегруженных индексаторов в одном типе.</span><span class="sxs-lookup"><span data-stu-id="f79c2-141">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="f79c2-142">Это обеспечивается C# компилятором.</span><span class="sxs-lookup"><span data-stu-id="f79c2-142">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="f79c2-143">❌ не используют индексированные свойства, не являющиеся свойствами по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="f79c2-143">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="f79c2-144">Это обеспечивается C# компилятором.</span><span class="sxs-lookup"><span data-stu-id="f79c2-144">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="f79c2-145">События уведомления об изменении свойств</span><span class="sxs-lookup"><span data-stu-id="f79c2-145">Property Change Notification Events</span></span>
 <span data-ttu-id="f79c2-146">Иногда бывает полезно указать событие, уведомляющее пользователя об изменениях в значении свойства.</span><span class="sxs-lookup"><span data-stu-id="f79c2-146">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="f79c2-147">Например, `System.Windows.Forms.Control` вызывает событие `TextChanged` после изменения значения свойства `Text`.</span><span class="sxs-lookup"><span data-stu-id="f79c2-147">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="f79c2-148">✔️ Рассмотрите возможность вызова событий уведомлений об изменениях при изменении значений свойств в интерфейсах API высокого уровня (обычно в компонентах конструктора).</span><span class="sxs-lookup"><span data-stu-id="f79c2-148">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="f79c2-149">Если есть хороший сценарий, при котором пользователю известно, когда изменяется свойство объекта, объект должен создать событие уведомления об изменении для свойства.</span><span class="sxs-lookup"><span data-stu-id="f79c2-149">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="f79c2-150">Однако вряд ли стоит тратить издержки на создание таких событий для низкоуровневых API, таких как базовые типы или коллекции.</span><span class="sxs-lookup"><span data-stu-id="f79c2-150">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="f79c2-151">Например, <xref:System.Collections.Generic.List%601> не будет вызывать такие события при добавлении нового элемента в список и изменении свойства `Count`.</span><span class="sxs-lookup"><span data-stu-id="f79c2-151">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="f79c2-152">✔️ Рассмотрите возможность создания событий уведомления об изменениях, когда значение свойства изменяется через внешние силы.</span><span class="sxs-lookup"><span data-stu-id="f79c2-152">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="f79c2-153">Если значение свойства изменяется с помощью некоторой внешней силы (не путем вызова методов объекта), события вызываются разработчиком, что значение изменяется и изменилось.</span><span class="sxs-lookup"><span data-stu-id="f79c2-153">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="f79c2-154">Хорошим примером является свойство `Text` элемента управления "текстовое поле".</span><span class="sxs-lookup"><span data-stu-id="f79c2-154">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="f79c2-155">Когда пользователь вводит текст в `TextBox`, значение свойства изменяется автоматически.</span><span class="sxs-lookup"><span data-stu-id="f79c2-155">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
