<span data-ttu-id="f8a53-103">Для большинства объектов, создаваемых приложением, управление памятью осуществляется сборщиком мусора .NET.</span><span class="sxs-lookup"><span data-stu-id="f8a53-103">For the majority of the objects that your app creates, you can rely on .NET's garbage collector to handle memory management.</span></span> <span data-ttu-id="f8a53-104">Однако при создании объектов, которые включают неуправляемые ресурсы, после использования неуправляемых ресурсов в приложении необходимо освобождать их явно.</span><span class="sxs-lookup"><span data-stu-id="f8a53-104">However, when you create objects that include unmanaged resources, you must explicitly release those resources when you finish using them in your app.</span></span> <span data-ttu-id="f8a53-105">Основным типом неуправляемых ресурсов являются объекты, заключающие в себе ресурсы операционной системы, такие как файлы, окна, сетевые подключения и подключения к базам данным.</span><span class="sxs-lookup"><span data-stu-id="f8a53-105">The most common types of unmanaged resource are objects that wrap operating system resources, such as files, windows, network connections, or database connections.</span></span> <span data-ttu-id="f8a53-106">Хотя сборщик мусора может отслеживать время жизни управляемого объекта, инкапсулирующего неуправляемые ресурсы, он не имеет сведений о том, как освобождать такие ресурсы.</span><span class="sxs-lookup"><span data-stu-id="f8a53-106">Although the garbage collector is able to track the lifetime of an object that encapsulates an unmanaged resource, it doesn't know how to release and clean up the unmanaged resource.</span></span>  
  
 <span data-ttu-id="f8a53-107">Если ваши типы используют неуправляемые ресурсы, необходимо выполнить следующие действия:</span><span class="sxs-lookup"><span data-stu-id="f8a53-107">If your types use unmanaged resources, you should do the following:</span></span>  
  
-   <span data-ttu-id="f8a53-108">Реализуйте [шаблон удаления](../../../docs/standard/design-guidelines/dispose-pattern.md).</span><span class="sxs-lookup"><span data-stu-id="f8a53-108">Implement the [dispose pattern](../../../docs/standard/design-guidelines/dispose-pattern.md).</span></span> <span data-ttu-id="f8a53-109">Это требует реализации <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> для детерминированного освобождения неуправляемых ресурсов.</span><span class="sxs-lookup"><span data-stu-id="f8a53-109">This requires that you provide an <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> implementation to enable the deterministic release of  unmanaged resources.</span></span> <span data-ttu-id="f8a53-110">Объект-получатель типа вызывает метод <xref:System.IDisposable.Dispose%2A>, когда объект и используемые им ресурсы больше не нужны.</span><span class="sxs-lookup"><span data-stu-id="f8a53-110">A consumer of your type calls <xref:System.IDisposable.Dispose%2A> when the object (and the resources it uses) is no longer needed.</span></span> <span data-ttu-id="f8a53-111">Метод <xref:System.IDisposable.Dispose%2A> немедленно освобождает неуправляемые ресурсы.</span><span class="sxs-lookup"><span data-stu-id="f8a53-111">The <xref:System.IDisposable.Dispose%2A> method immediately releases the unmanaged resources.</span></span>  
  
-   <span data-ttu-id="f8a53-112">Предусмотрите освобождение неуправляемых ресурсов, в случае если метод <xref:System.IDisposable.Dispose%2A> не будет вызван объектом-получателем.</span><span class="sxs-lookup"><span data-stu-id="f8a53-112">Provide for your unmanaged resources to be released in the event that a consumer of your type forgets to call <xref:System.IDisposable.Dispose%2A>.</span></span> <span data-ttu-id="f8a53-113">Это можно сделать двумя способами.</span><span class="sxs-lookup"><span data-stu-id="f8a53-113">There are two ways to do this:</span></span>  
  
    -   <span data-ttu-id="f8a53-114">Используйте безопасный дескриптор в качестве оболочки для неуправляемого ресурса.</span><span class="sxs-lookup"><span data-stu-id="f8a53-114">Use a safe handle to wrap your unmanaged resource.</span></span> <span data-ttu-id="f8a53-115">Это рекомендуемая методика.</span><span class="sxs-lookup"><span data-stu-id="f8a53-115">This is the recommended technique.</span></span> <span data-ttu-id="f8a53-116">Безопасные дескрипторы являются производными от класса <xref:System.Runtime.InteropServices.SafeHandle?displayProperty=nameWithType>.          <span data-ttu-id="f8a53-119">—или—</span><span class="sxs-lookup"><span data-stu-id="f8a53-119">—or—</span></span>  
  
    -   <span data-ttu-id="f8a53-120">Переопределите метод <xref:System.Object.Finalize%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="f8a53-120">Override the <xref:System.Object.Finalize%2A?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="f8a53-121">Завершение включает недетерминированное освобождение неуправляемых ресурсов, когда объекту-получателю типа не удается вызвать метод <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> для их детерминированного удаления.</span><span class="sxs-lookup"><span data-stu-id="f8a53-121">Finalization enables the non-deterministic release of unmanaged resources when the consumer of a type fails to call <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> to dispose of them deterministically.</span></span> <span data-ttu-id="f8a53-122">Однако поскольку завершение объекта может быть сложной операцией с высокой вероятностью ошибок, рекомендуется использовать безопасный дескриптор вместо указания собственного метода завершения.</span><span class="sxs-lookup"><span data-stu-id="f8a53-122">However, because object finalization can be a complex and error-prone operation, we recommend that you use a safe handle instead of providing your own finalizer.</span></span>  
  
 <span data-ttu-id="f8a53-123">Объекты-получатели типа могут затем вызвать реализацию <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> непосредственно для освобождения памяти, используемой неуправляемыми ресурсами.</span><span class="sxs-lookup"><span data-stu-id="f8a53-123">Consumers of your type can then call your <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> implementation directly to free memory used by unmanaged resources.</span></span> <span data-ttu-id="f8a53-124">При правильной реализации метода <xref:System.IDisposable.Dispose%2A> метод <xref:System.Object.Finalize%2A> или переопределенная версия метода <xref:System.Object.Finalize%2A?displayProperty=nameWithType> становятся резервным средством очистки ресурсов в случае, если не вызывается метод <xref:System.IDisposable.Dispose%2A>.</span><span class="sxs-lookup"><span data-stu-id="f8a53-124">When you properly implement a <xref:System.IDisposable.Dispose%2A> method, either your safe handle's <xref:System.Object.Finalize%2A> method or your own override of the <xref:System.Object.Finalize%2A?displayProperty=nameWithType> method becomes a safeguard to clean up resources in the event that the <xref:System.IDisposable.Dispose%2A> method is not called.</span></span>
