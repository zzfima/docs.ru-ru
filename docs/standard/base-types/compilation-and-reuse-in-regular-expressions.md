# <a name="compilation-and-reuse-in-regular-expressions"></a><span data-ttu-id="c246f-102">Компиляция и многократное использование в регулярных выражениях</span><span class="sxs-lookup"><span data-stu-id="c246f-102">Compilation and Reuse in Regular Expressions</span></span>
<span data-ttu-id="c246f-103">Производительность приложений, активно использующих регулярные выражения, можно оптимизировать, если понимать, каким образом обработчик регулярных выражений компилирует выражения и как кэшируются регулярные выражения.</span><span class="sxs-lookup"><span data-stu-id="c246f-103">You can optimize the performance of applications that make extensive use of regular expressions by understanding how the regular expression engine compiles expressions and by understanding how regular expressions are cached.</span></span> <span data-ttu-id="c246f-104">В этом разделе описаны и компиляция, и кэширование.</span><span class="sxs-lookup"><span data-stu-id="c246f-104">This topic discusses both compilation and caching.</span></span>  
  
## <a name="compiled-regular-expressions"></a><span data-ttu-id="c246f-105">Скомпилированные регулярные выражения</span><span class="sxs-lookup"><span data-stu-id="c246f-105">Compiled Regular Expressions</span></span>  
 <span data-ttu-id="c246f-106">По умолчанию обработчик регулярных выражений компилирует регулярные выражения в последовательность внутренних инструкций (это коды высокого уровня, которые отличаются от языка MSIL).</span><span class="sxs-lookup"><span data-stu-id="c246f-106">By default, the regular expression engine compiles a regular expression to a sequence of internal instructions (these are high-level codes that are different from Microsoft intermediate language, or MSIL).</span></span> <span data-ttu-id="c246f-107">Когда обработчик выполняет регулярное выражение, он преобразовывает внутренние коды.</span><span class="sxs-lookup"><span data-stu-id="c246f-107">When the engine executes a regular expression, it interprets the internal codes.</span></span>  
  
 <span data-ttu-id="c246f-108">Если объект <xref:System.Text.RegularExpressions.Regex> создается с параметром <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType>, то регулярное выражение явно компилируется в язык MSIL вместо внутренних инструкций высокого уровня.</span><span class="sxs-lookup"><span data-stu-id="c246f-108">If a <xref:System.Text.RegularExpressions.Regex> object is constructed with the <xref:System.Text.RegularExpressions.RegexOptions.Compiled?displayProperty=nameWithType> option, it compiles the regular expression to explicit MSIL code instead of high-level regular expression internal instructions.</span></span> <span data-ttu-id="c246f-109">Это позволяет JIT-компилятору платформы .NET преобразовывать выражение в изначальный машинный код данного объекта для повышения производительности.</span><span class="sxs-lookup"><span data-stu-id="c246f-109">This allows .NET's just-in-time (JIT) compiler to convert the expression to native machine code for higher performance.</span></span>  <span data-ttu-id="c246f-110">Стоимость конструирования объекта <xref:System.Text.RegularExpressions.Regex> может быть выше, однако стоимость выполнения совпадений, скорее всего, будет значительно меньше.</span><span class="sxs-lookup"><span data-stu-id="c246f-110">The cost of constructing the <xref:System.Text.RegularExpressions.Regex> object may be higher, but the cost of performing matches with it is likely to be much smaller.</span></span>

 <span data-ttu-id="c246f-111">Также можно применить предварительно скомпилированные регулярные выражения.</span><span class="sxs-lookup"><span data-stu-id="c246f-111">An alternative is to use precompiled regular expressions.</span></span> <span data-ttu-id="c246f-112">Все выражения можно скомпилировать в библиотеку DLL с помощью метода <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A>, чтобы использовать их многократно.</span><span class="sxs-lookup"><span data-stu-id="c246f-112">You can compile all of your expressions into a reusable DLL by using the <xref:System.Text.RegularExpressions.Regex.CompileToAssembly%2A> method.</span></span> <span data-ttu-id="c246f-113">Этим вы избавитесь от компиляции во время выполнения, сохраняя скорость работы как при использовании скомпилированных регулярных выражений.</span><span class="sxs-lookup"><span data-stu-id="c246f-113">This avoids the need to compile at runtime while still benefiting from the speed of compiled regular expressions.</span></span>  
  
## <a name="the-regular-expressions-cache"></a><span data-ttu-id="c246f-114">Кэш регулярных выражений</span><span class="sxs-lookup"><span data-stu-id="c246f-114">The Regular Expressions Cache</span></span>  
 <span data-ttu-id="c246f-115">Для повышения производительности обработчик регулярных выражений поддерживает кэш скомпилированных регулярных выражений на уровне приложения.</span><span class="sxs-lookup"><span data-stu-id="c246f-115">To improve performance, the regular expression engine maintains an application-wide cache of compiled regular expressions.</span></span> <span data-ttu-id="c246f-116">Кэш хранит шаблоны регулярных выражений, которые используются только при вызове статических методов.</span><span class="sxs-lookup"><span data-stu-id="c246f-116">The cache stores regular expression patterns that are used only in static method calls.</span></span> <span data-ttu-id="c246f-117">(Шаблоны регулярных выражений, предоставляемые методами экземпляра, не кэшируются.) Это устраняет потребность в повторном разборе выражения в байт-код высокого уровня при каждом его использовании.</span><span class="sxs-lookup"><span data-stu-id="c246f-117">(Regular expression patterns supplied to instance methods are not cached.) This avoids the need to reparse an expression into high-level byte code each time it is used.</span></span>  
  
 <span data-ttu-id="c246f-118">Максимальное число кэшированных регулярных выражений определяется значением `static` (`Shared` в Visual Basic) свойства <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="c246f-118">The maximum number of cached regular expressions is determined by the value of the `static` (`Shared` in Visual Basic) <xref:System.Text.RegularExpressions.Regex.CacheSize%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="c246f-119">По умолчанию обработчик регулярных выражений кэширует до 15 скомпилированных регулярных выражений.</span><span class="sxs-lookup"><span data-stu-id="c246f-119">By default, the regular expression engine caches up to 15 compiled regular expressions.</span></span> <span data-ttu-id="c246f-120">Если число скомпилированных регулярных выражений превышает размер кэша, из него удаляется наиболее давнее по использованию регулярное выражение и кэшируется новое регулярное выражение.</span><span class="sxs-lookup"><span data-stu-id="c246f-120">If the number of compiled regular expressions exceeds the cache size, the least recently used regular expression is discarded and the new regular expression is cached.</span></span>  
  
 <span data-ttu-id="c246f-121">Приложение может повторно использовать регулярные выражения одним из указанных далее двух способов.</span><span class="sxs-lookup"><span data-stu-id="c246f-121">Your application can reuse regular expressions in one of the following two ways:</span></span>  
  
- <span data-ttu-id="c246f-122">Вы можете использовать статический метод объекта <xref:System.Text.RegularExpressions.Regex> для определения регулярного выражения.</span><span class="sxs-lookup"><span data-stu-id="c246f-122">By using a static method of the <xref:System.Text.RegularExpressions.Regex> object to define the regular expression.</span></span> <span data-ttu-id="c246f-123">Если используется шаблон регулярного выражения, который уже был определен при вызове другого статического метода, обработчик регулярных выражений попробует извлечь его из кэша.</span><span class="sxs-lookup"><span data-stu-id="c246f-123">If you're using a regular expression pattern that has already been defined by another static method call, the regular expression engine will try to retrieve it from the cache.</span></span> <span data-ttu-id="c246f-124">Если он недоступен в кэше, обработчик скомпилирует регулярное выражение и добавит его в кэш.</span><span class="sxs-lookup"><span data-stu-id="c246f-124">If it's not available in the cache, the engine will compile the regular expression and add it to the cache.</span></span>
  
- <span data-ttu-id="c246f-125">Вы можете многократно использовать существующий объект <xref:System.Text.RegularExpressions.Regex> везде, где потребуется созданный его шаблон регулярного выражения.</span><span class="sxs-lookup"><span data-stu-id="c246f-125">By reusing an existing <xref:System.Text.RegularExpressions.Regex> object as long as its regular expression pattern is needed.</span></span>  
  
 <span data-ttu-id="c246f-126">Создание и быстрое уничтожение многочисленных объектов <xref:System.Text.RegularExpressions.Regex> — это очень затратный процесс, так как высоки издержки на создание объектов и компиляцию регулярных выражений.</span><span class="sxs-lookup"><span data-stu-id="c246f-126">Because of the overhead of object instantiation and regular expression compilation, creating and rapidly destroying numerous <xref:System.Text.RegularExpressions.Regex> objects is a very expensive process.</span></span> <span data-ttu-id="c246f-127">Для приложений, использующих большое количество разных регулярных выражений, производительность можно повысить путем вызова статических методов `Regex` и, возможно, путем увеличения размера кэша для регулярных выражений.</span><span class="sxs-lookup"><span data-stu-id="c246f-127">For applications that use a large number of different regular expressions, you can optimize performance by using calls to static `Regex` methods and possibly by increasing the size of the regular expression cache.</span></span>
