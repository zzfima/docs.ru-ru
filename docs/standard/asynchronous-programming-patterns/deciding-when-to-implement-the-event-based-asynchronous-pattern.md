## <a name="general-principles"></a><span data-ttu-id="f302d-107">Общие принципы</span><span class="sxs-lookup"><span data-stu-id="f302d-107">General Principles</span></span>

<span data-ttu-id="f302d-108">Обычно для всех асинхронных функций лучше применять асинхронную модель на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-108">In general, you should expose asynchronous features using the Event-based Asynchronous Pattern whenever possible.</span></span> <span data-ttu-id="f302d-109">Но есть несколько требований, которым она не полностью соответствует.</span><span class="sxs-lookup"><span data-stu-id="f302d-109">However, there are some requirements that the event-based pattern cannot meet.</span></span> <span data-ttu-id="f302d-110">Если они применимы в вашей ситуации, в дополнение к модели на основе событий следует реализовать модель <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-110">In those cases, you may need to implement the <xref:System.IAsyncResult> pattern in addition to the event-based pattern.</span></span>

> [!NOTE]
> <span data-ttu-id="f302d-111">Модель <xref:System.IAsyncResult> очень редко реализуется отдельно, без модели на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-111">It is rare for the <xref:System.IAsyncResult> pattern to be implemented without the event-based pattern also being implemented.</span></span>

## <a name="guidelines"></a><span data-ttu-id="f302d-112">Рекомендации</span><span class="sxs-lookup"><span data-stu-id="f302d-112">Guidelines</span></span>

<span data-ttu-id="f302d-113">Ниже перечислены рекомендации по выбору асинхронной модели на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-113">The following list describes the guidelines for when you should implement Event-based Asynchronous Pattern:</span></span>

- <span data-ttu-id="f302d-114">Используйте модель на основе событий в качестве основного API для асинхронных возможностей класса.</span><span class="sxs-lookup"><span data-stu-id="f302d-114">Use the event-based pattern as the default API to expose asynchronous behavior for your class.</span></span>

- <span data-ttu-id="f302d-115">Не предоставляйте модель <xref:System.IAsyncResult>, если класс используется в основном в клиентском приложении, например Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="f302d-115">Do not expose the <xref:System.IAsyncResult> pattern when your class is primarily used in a client application, for example Windows Forms.</span></span>

- <span data-ttu-id="f302d-116">Предоставляйте модель <xref:System.IAsyncResult> только в тех случаях, когда это необходимо для конкретных требований.</span><span class="sxs-lookup"><span data-stu-id="f302d-116">Only expose the <xref:System.IAsyncResult> pattern when it is necessary for meeting your requirements.</span></span> <span data-ttu-id="f302d-117">Например, модель <xref:System.IAsyncResult> может потребоваться для совместимости с уже существующими API.</span><span class="sxs-lookup"><span data-stu-id="f302d-117">For example, compatibility with an existing API may require you to expose the <xref:System.IAsyncResult> pattern.</span></span>

- <span data-ttu-id="f302d-118">Не предоставляйте модель <xref:System.IAsyncResult> отдельно, без модели на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-118">Do not expose the <xref:System.IAsyncResult> pattern without also exposing the event-based pattern.</span></span>

- <span data-ttu-id="f302d-119">Если потребуется модель <xref:System.IAsyncResult>, предоставляйте ее только в качестве дополнительной возможности.</span><span class="sxs-lookup"><span data-stu-id="f302d-119">If you must expose the <xref:System.IAsyncResult> pattern, do so as an advanced option.</span></span> <span data-ttu-id="f302d-120">Например, если вы создаете прокси-объект, создайте модель на основе событий в качестве стандартного варианта, а модель <xref:System.IAsyncResult> предоставляйте опционально.</span><span class="sxs-lookup"><span data-stu-id="f302d-120">For example, if you generate a proxy object, generate the event-based pattern by default, with an option to generate the <xref:System.IAsyncResult> pattern.</span></span>

- <span data-ttu-id="f302d-121">Модели на основе событий следует основывать на вашей реализации модели <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-121">Build your event-based pattern implementation on your <xref:System.IAsyncResult> pattern implementation.</span></span>

- <span data-ttu-id="f302d-122">По возможности не размещайте модель на основе событий и модель <xref:System.IAsyncResult> в одном классе.</span><span class="sxs-lookup"><span data-stu-id="f302d-122">Avoid exposing both the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class.</span></span> <span data-ttu-id="f302d-123">Модель на основе событий должна размещаться в классах "более высокого уровня" относительно модели <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-123">Expose the event-based pattern on "higher level" classes and the <xref:System.IAsyncResult> pattern on "lower level" classes.</span></span> <span data-ttu-id="f302d-124">Сравните, например, модель на основе событий для компонента <xref:System.Net.WebClient> с моделью <xref:System.IAsyncResult> в классе <xref:System.Web.HttpRequest>.</span><span class="sxs-lookup"><span data-stu-id="f302d-124">For example, compare the event-based pattern on the <xref:System.Net.WebClient> component with the <xref:System.IAsyncResult> pattern on the <xref:System.Web.HttpRequest> class.</span></span>

  - <span data-ttu-id="f302d-125">Используйте один и тот же класс для предоставления модели на основе событий и модели <xref:System.IAsyncResult> только в том случае, если это необходимо для совместимости.</span><span class="sxs-lookup"><span data-stu-id="f302d-125">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class when compatibility requires it.</span></span> <span data-ttu-id="f302d-126">Например, если вы уже опубликовали API, в котором реализована модель <xref:System.IAsyncResult>, ее придется сохранить для обеспечения обратной совместимости<xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-126">For example, if you have already released an API that uses the <xref:System.IAsyncResult> pattern, you would need to retain the <xref:System.IAsyncResult> pattern for backward compatibility.</span></span>

  - <span data-ttu-id="f302d-127">Также допустимо размещать модель на основе событий и модель <xref:System.IAsyncResult> в одном классе в тех случаях, когда сложность объектной модели перевешивает преимущества от раздельной реализации.</span><span class="sxs-lookup"><span data-stu-id="f302d-127">Expose the event-based pattern and the <xref:System.IAsyncResult> pattern on the same class if the resulting object model complexity outweighs the benefit of separating the implementations.</span></span> <span data-ttu-id="f302d-128">Гораздо лучше предоставить обе модели в одном классе, чем вовсе не предоставлять модель на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-128">It is better to expose both patterns on a single class than to avoid exposing the event-based pattern.</span></span>

  - <span data-ttu-id="f302d-129">Если вам придется разместить модель на основе событий в том же классе, что и модель <xref:System.IAsyncResult>, используйте <xref:System.ComponentModel.EditorBrowsableAttribute> со значением <xref:System.ComponentModel.EditorBrowsableState.Advanced>, чтобы обозначить реализацию модели <xref:System.IAsyncResult> как дополнительную функцию.</span><span class="sxs-lookup"><span data-stu-id="f302d-129">If you must expose both the event-based pattern and <xref:System.IAsyncResult> pattern on a single class, use <xref:System.ComponentModel.EditorBrowsableAttribute> set to <xref:System.ComponentModel.EditorBrowsableState.Advanced> to mark the <xref:System.IAsyncResult> pattern implementation as an advanced feature.</span></span> <span data-ttu-id="f302d-130">В этом случае среды разработки, например Visual Studio IntelliSense, не будут отображать свойства и методы <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-130">This indicates to design environments, such as Visual Studio IntelliSense, not to display the <xref:System.IAsyncResult> properties and methods.</span></span> <span data-ttu-id="f302d-131">Эти свойства и методы будут функционировать как прежде, но не будут захламлять рабочее пространство API-интерфейса для разработчика, работающего в IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="f302d-131">These properties and methods are still fully usable, but the developer working through IntelliSense has a clearer view of the API.</span></span>

## <a name="criteria-for-exposing-the-iasyncresult-pattern-in-addition-to-the-event-based-pattern"></a><span data-ttu-id="f302d-132">Критерии для предоставления модели IAsyncResult в дополнение к модели на основе событий</span><span class="sxs-lookup"><span data-stu-id="f302d-132">Criteria for Exposing the IAsyncResult Pattern in Addition to the Event-based Pattern</span></span>

<span data-ttu-id="f302d-133">В описанных выше сценариях асинхронная модель на основе событий дает целый ряд преимуществ, но имеет и некоторые недостатки. Их важно учитывать, если у вас высокие требования к производительности.</span><span class="sxs-lookup"><span data-stu-id="f302d-133">While the Event-based Asynchronous Pattern has many benefits under the previously mentioned scenarios, it does have some drawbacks, which you should be aware of if performance is your most important requirement.</span></span>

<span data-ttu-id="f302d-134">Есть три сценария, для которых модель на основе событий подходит хуже, чем модель <xref:System.IAsyncResult>:</span><span class="sxs-lookup"><span data-stu-id="f302d-134">There are three scenarios that the event-based pattern does not address as well as the <xref:System.IAsyncResult> pattern:</span></span>

- <span data-ttu-id="f302d-135">ожидание с блокировкой для одного <xref:System.IAsyncResult>;</span><span class="sxs-lookup"><span data-stu-id="f302d-135">Blocking wait on one <xref:System.IAsyncResult></span></span>

- <span data-ttu-id="f302d-136">ожидание с блокировкой для нескольких объектов <xref:System.IAsyncResult>;</span><span class="sxs-lookup"><span data-stu-id="f302d-136">Blocking wait on many <xref:System.IAsyncResult> objects</span></span>

- <span data-ttu-id="f302d-137">опрос завершения для <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-137">Polling for completion on the <xref:System.IAsyncResult></span></span>

<span data-ttu-id="f302d-138">Все эти сценарии можно реализовать с помощью модели на основе событий, но такое решение будет более громоздким, чем использование модели <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-138">You can address these scenarios by using the event-based pattern, but doing so is more cumbersome than using the <xref:System.IAsyncResult> pattern.</span></span>

<span data-ttu-id="f302d-139">Разработчики часто используют модель <xref:System.IAsyncResult> для служб с высокими требованиями к производительности.</span><span class="sxs-lookup"><span data-stu-id="f302d-139">Developers often use the <xref:System.IAsyncResult> pattern for services that typically have very high performance requirements.</span></span> <span data-ttu-id="f302d-140">Например, сценарий с опросом завершения является одним из методов повышения производительности сервера.</span><span class="sxs-lookup"><span data-stu-id="f302d-140">For example, the polling for completion scenario is a high-performance server technique.</span></span>

<span data-ttu-id="f302d-141">Эффективность модели на основе событий ниже, чем у модели <xref:System.IAsyncResult>, поскольку она создает несколько объектов (особенно <xref:System.EventArgs>) и синхронизируется между потоками.</span><span class="sxs-lookup"><span data-stu-id="f302d-141">Additionally, the event-based pattern is less efficient than the <xref:System.IAsyncResult> pattern because it creates more objects, especially <xref:System.EventArgs>, and because it synchronizes across threads.</span></span>

<span data-ttu-id="f302d-142">Следующий список содержит рекомендации на случай, если вы решили использовать модель <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-142">The following list shows some recommendations to follow if you decide to use the <xref:System.IAsyncResult> pattern:</span></span>

- <span data-ttu-id="f302d-143">Предоставляйте модель <xref:System.IAsyncResult> только в том случае, когда строго необходима поддержка объектов <xref:System.Threading.WaitHandle> или <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-143">Only expose the <xref:System.IAsyncResult> pattern when you specifically require support for <xref:System.Threading.WaitHandle> or <xref:System.IAsyncResult> objects.</span></span>

- <span data-ttu-id="f302d-144">Предоставляйте модель <xref:System.IAsyncResult> только в том случае, если у вас уже есть API, который использует модель <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-144">Only expose the <xref:System.IAsyncResult> pattern when you have an existing API that uses the <xref:System.IAsyncResult> pattern.</span></span>

- <span data-ttu-id="f302d-145">Если у вас есть существующий API, основанный на модели <xref:System.IAsyncResult>, постарайтесь в следующем выпуске реализовать и модель на основе событий.</span><span class="sxs-lookup"><span data-stu-id="f302d-145">If you have an existing API based on the <xref:System.IAsyncResult> pattern, consider also exposing the event-based pattern in your next release.</span></span>

- <span data-ttu-id="f302d-146">Предоставляйте модель <xref:System.IAsyncResult> только в том случае, если у вас высокие требования к производительности, которые гарантированно недостижимы при применении модели на основе событий, но могут быть достижимы с помощью модели <xref:System.IAsyncResult>.</span><span class="sxs-lookup"><span data-stu-id="f302d-146">Only expose <xref:System.IAsyncResult> pattern if you have high performance requirements which you have verified cannot be met by the event-based pattern but can be met by the <xref:System.IAsyncResult> pattern.</span></span>
