# <a name="implement-resilient-applications"></a><span data-ttu-id="6c4e9-104">Реализация устойчивых приложений</span><span class="sxs-lookup"><span data-stu-id="6c4e9-104">Implement resilient applications</span></span>

<span data-ttu-id="6c4e9-105">*В приложениях на основе микрослужб и облачных приложениях должна быть предусмотрена обработка частичных сбоев, которые так или иначе будут происходить. Приложение следует проектировать так, чтобы оно было устойчиво к этим сбоям.*</span><span class="sxs-lookup"><span data-stu-id="6c4e9-105">*Your microservice and cloud-based applications must embrace the partial failures that will certainly occur eventually. You must design your application to be resilient to those partial failures.*</span></span>

<span data-ttu-id="6c4e9-106">Устойчивость — это возможность восстановления после сбоев и продолжение работы.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-106">Resiliency is the ability to recover from failures and continue to function.</span></span> <span data-ttu-id="6c4e9-107">Исходить следует из того, что избежать сбоев невозможно, и на них нужно реагировать так, чтобы избежать простоев или потери данных.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-107">It isn't about avoiding failures but accepting the fact that failures will happen and responding to them in a way that avoids downtime or data loss.</span></span> <span data-ttu-id="6c4e9-108">Цель устойчивости — вернуть приложение в полностью функционирующее состояние после сбоя.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-108">The goal of resiliency is to return the application to a fully functioning state after a failure.</span></span>

<span data-ttu-id="6c4e9-109">Разработать и развернуть приложение на основе микрослужб достаточно сложно.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-109">It's challenging enough to design and deploy a microservices-based application.</span></span> <span data-ttu-id="6c4e9-110">Но помимо этого, оно должно работать в среде, в которой неизбежны определенные неполадки.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-110">But you also need to keep your application running in an environment where some sort of failure is certain.</span></span> <span data-ttu-id="6c4e9-111">Поэтому приложение должно быть устойчивым.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-111">Therefore, your application should be resilient.</span></span> <span data-ttu-id="6c4e9-112">Оно должно уметь справляться с частичными сбоями, такими как отключения сети либо аварийные сбои узлов или виртуальных машин в облаке.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-112">It should be designed to cope with partial failures, like network outages or nodes or VMs crashing in the cloud.</span></span> <span data-ttu-id="6c4e9-113">Даже перенос микрослужб (контейнеров) в другой узел кластера может приводить к кратковременным сбоям в приложении.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-113">Even microservices (containers) being moved to a different node within a cluster can cause intermittent short failures within the application.</span></span>

<span data-ttu-id="6c4e9-114">Многие отдельные компоненты приложения также должны включать функции мониторинга работоспособности.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-114">The many individual components of your application should also incorporate health monitoring features.</span></span> <span data-ttu-id="6c4e9-115">Следуя рекомендациям этой главы, можно создать приложение, которое будет продолжать работать, несмотря на временные простои или небольшие неисправности, возникающие в сложных и облачных развертываниях.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-115">By following the guidelines in this chapter, you can create an application that can work smoothly in spite of transient downtime or the normal hiccups that occur in complex and cloud-based deployments.</span></span>

>[!IMPORTANT]
> <span data-ttu-id="6c4e9-116">Для реализации устойчивости с помощью [типизированных клиентов](./use-httpclientfactory-to-implement-resilient-http-requests.md) в приложении eShopOnContainer использовалась [библиотека Polly](http://www.thepollyproject.org/) вплоть до выпуска 3.0.0.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-116">eShopOnContainer had been using the [Polly library](http://www.thepollyproject.org/) to implement resiliency using [Typed Clients](./use-httpclientfactory-to-implement-resilient-http-requests.md) up until the release 3.0.0.</span></span>
>
> <span data-ttu-id="6c4e9-117">Начиная с выпуска 3.0.0, устойчивость вызовов HTTP реализуется с помощью [сетки Linkerd](https://linkerd.io/), которая обеспечивает прозрачную и настраиваемую обработку повторных попыток в кластере Kubernetes, что избавляет от необходимости обрабатывать эти события в коде.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-117">Starting with release 3.0.0, the HTTP calls resiliency is implemented using a [Linkerd mesh](https://linkerd.io/), that handles retries in a transparent and configurable fashion, within a Kubernetes cluster, without having to handle those concerns in the code.</span></span>
>
> <span data-ttu-id="6c4e9-118">Библиотека Polly все еще используется для повышения устойчивости при подключении к базам данных, а именно при запуске служб.</span><span class="sxs-lookup"><span data-stu-id="6c4e9-118">The Polly library is still used to add resilience to database connections, specially while starting up the services.</span></span>
