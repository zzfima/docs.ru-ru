## <a name="how-stream-upgrades-work"></a><span data-ttu-id="319b8-110">Принципы работы обновления потока</span><span class="sxs-lookup"><span data-stu-id="319b8-110">How Stream Upgrades Work</span></span>

1. <span data-ttu-id="319b8-112">Обновления потока *инициатора* начинает процесс: во время выполнения она может инициировать запрос на обновление транспортного уровня канала к другой стороне связи.</span><span class="sxs-lookup"><span data-stu-id="319b8-112">An upgrade stream *Initiator* begins the process: at run-time it can initiate a request to the other end of its connection to upgrade the channel transport layer.</span></span>  
  
2. <span data-ttu-id="319b8-113">Обновления потока *Акцептор* выполняет обновление: во время выполнения он получает запрос об обновлении от другого компьютера и по возможности принимает обновление.</span><span class="sxs-lookup"><span data-stu-id="319b8-113">An upgrade stream *Acceptor* carries out the upgrade: at run-time it receives the upgrade request from the other machine, and if possible, accepts the upgrade.</span></span>  
  
3. <span data-ttu-id="319b8-114">Обновление *поставщика* создает *инициатора* на стороне клиента и *Акцептор* на сервере.</span><span class="sxs-lookup"><span data-stu-id="319b8-114">An upgrade *Provider* creates the *Initiator* on the client and the *Acceptor* on the server.</span></span>  
  
4. <span data-ttu-id="319b8-115">Обновления потока *элемента привязки* добавляется в привязки службы и клиента, а также создает поставщик во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="319b8-115">A stream upgrade *Binding Element* is added to the bindings on the service and the client, and creates the provider at runtime.</span></span>
