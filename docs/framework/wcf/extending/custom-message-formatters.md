<span data-ttu-id="a0215-103">Содержимое сообщения зачастую представлено в XML-виде, который обычно является неудобным форматом для приложений.</span><span class="sxs-lookup"><span data-stu-id="a0215-103">The content in a message is often in the form of XML, which is usually not a convenient format for an application.</span></span> <span data-ttu-id="a0215-104">Приложения управляют объектами, получая и устанавливая их свойства.</span><span class="sxs-lookup"><span data-stu-id="a0215-104">Applications manipulate objects, getting and setting their properties.</span></span> <span data-ttu-id="a0215-105">Windows Communication Foundation (WCF) использует *контракт данных* для преобразования <xref:System.ServiceModel.Channels.Message> объекта в объект, легко обрабатываемый приложением.</span><span class="sxs-lookup"><span data-stu-id="a0215-105">Windows Communication Foundation (WCF) uses the *Data Contract* to convert a <xref:System.ServiceModel.Channels.Message> object into an object easily handled by an application.</span></span> <span data-ttu-id="a0215-106">Эти действия называются сериализацией и десериализацией.</span><span class="sxs-lookup"><span data-stu-id="a0215-106">These processes are called serialization and deserialization.</span></span> <span data-ttu-id="a0215-107">Обратите внимание, что данные термины также используются для описания сериализации и десериализации, выполненной транспортным слоем, в формат сообщений для передачи по линиям связи, что является несвязанным процессом.</span><span class="sxs-lookup"><span data-stu-id="a0215-107">Note that these same terms are used to describe the serialization and deserialization done by the transport layer to and from the message wire format, which is an unrelated process.</span></span>
