## <a name="the-import-process"></a><span data-ttu-id="3f5ab-105">Процесс импорта</span><span class="sxs-lookup"><span data-stu-id="3f5ab-105">The Import Process</span></span>
 <span data-ttu-id="3f5ab-106">Процесс импорта схемы начинается с создания объекта <xref:System.Xml.Schema.XmlSchemaSet>, что приводит к созданию объекта <xref:System.CodeDom.CodeCompileUnit>.</span><span class="sxs-lookup"><span data-stu-id="3f5ab-106">The schema import process starts with an <xref:System.Xml.Schema.XmlSchemaSet> and produces a <xref:System.CodeDom.CodeCompileUnit>.</span></span>

 <span data-ttu-id="3f5ab-109">Класс `CodeCompileUnit` является частью документной объектной модели кода (CodeDOM) платформы [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)], представляющей код [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] абстрактным способом.</span><span class="sxs-lookup"><span data-stu-id="3f5ab-109">The `CodeCompileUnit` is part of the [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)]’s Code Document Object Model (CodeDOM) that represents [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] code in an abstract way.</span></span> <span data-ttu-id="3f5ab-110">Чтобы создать фактический код из `CodeCompileUnit`, используйте подкласс класса <xref:System.CodeDom.Compiler.CodeDomProvider>, например класс <xref:Microsoft.CSharp.CSharpCodeProvider> или класс <xref:Microsoft.VisualBasic.VBCodeProvider>.</span><span class="sxs-lookup"><span data-stu-id="3f5ab-110">To generate the actual code from a `CodeCompileUnit`, use a subclass of the <xref:System.CodeDom.Compiler.CodeDomProvider> class, such as the <xref:Microsoft.CSharp.CSharpCodeProvider> or <xref:Microsoft.VisualBasic.VBCodeProvider> class.</span></span>
