## <a name="findcriteria"></a><span data-ttu-id="dadea-111">FindCriteria</span><span class="sxs-lookup"><span data-stu-id="dadea-111">FindCriteria</span></span>  
 <span data-ttu-id="dadea-112">Параметр <xref:System.ServiceModel.Discovery.FindCriteria> имеет несколько свойств, которые могут быть сгруппированы в критерии поиска (с указанием искомых служб) и критерии прекращения поиска (время выполнения поиска).</span><span class="sxs-lookup"><span data-stu-id="dadea-112"><xref:System.ServiceModel.Discovery.FindCriteria> has several properties, which can be grouped into search criteria, which specify what services you are looking for, and find termination criteria (how long the search should last).</span></span> <span data-ttu-id="dadea-113">Параметр <xref:System.ServiceModel.Discovery.FindCriteria> может содержать несколько критериев поиска.</span><span class="sxs-lookup"><span data-stu-id="dadea-113">A <xref:System.ServiceModel.Discovery.FindCriteria> can contain multiple search criteria.</span></span> <span data-ttu-id="dadea-114">По умолчанию служба должна соответствовать всем компонентам, в противном случае она не будет считаться службой, которая соответствует критериям.</span><span class="sxs-lookup"><span data-stu-id="dadea-114">By default, the service has to match all of the components otherwise it does not consider itself a matching service.</span></span> <span data-ttu-id="dadea-115">Если необходимо найти службы, которые соответствуют только части критериев, для службы можно реализовать пользовательскую логику поиска или использовать несколько критериев.</span><span class="sxs-lookup"><span data-stu-id="dadea-115">If you want to find services that only match some of the criteria, you can implement custom find logic on the service or you can use multiple queries.</span></span>  
  
 <span data-ttu-id="dadea-116">К критериям поиска относятся следующие.</span><span class="sxs-lookup"><span data-stu-id="dadea-116">Search criteria include:</span></span>

-   <span data-ttu-id="dadea-121"><xref:System.ServiceModel.Discovery.Configuration.ScopeElement> - необязательный.</span><span class="sxs-lookup"><span data-stu-id="dadea-121"><xref:System.ServiceModel.Discovery.Configuration.ScopeElement> - Optional.</span></span> <span data-ttu-id="dadea-122">Области представляют собой абсолютные идентификаторы URI, которые используются для категоризации отдельных конечных точек служб.</span><span class="sxs-lookup"><span data-stu-id="dadea-122">Scopes are absolute URIs that are used to categorize individual service endpoints.</span></span> <span data-ttu-id="dadea-123">Их можно использовать в случаях, когда несколько конечных точек используется для предоставления одного контракта и необходим способ поиска подмножества конечных точек.</span><span class="sxs-lookup"><span data-stu-id="dadea-123">You may want to use this in scenarios where multiple endpoints expose the same contract and you want a way to search for a subset of the endpoints.</span></span> <span data-ttu-id="dadea-124">Если указано более одной области, будет получен ответ только от конечных точек службы, соответствующих всем областям.</span><span class="sxs-lookup"><span data-stu-id="dadea-124">If more than one scope is specified, only service endpoints matching ALL scopes reply.</span></span>

    -   <span data-ttu-id="dadea-133"><xref:System.ServiceModel.Discovery.FindCriteria.ScopeMatchByNone?displayProperty=nameWithType> выполняет сопоставление только тех служб, которые не указывают области.</span><span class="sxs-lookup"><span data-stu-id="dadea-133"><xref:System.ServiceModel.Discovery.FindCriteria.ScopeMatchByNone?displayProperty=nameWithType> matches only those services that do not specify a scope.</span></span>  
  
     <span data-ttu-id="dadea-134">Если правило сопоставления областей не указано, используется <xref:System.ServiceModel.Discovery.FindCriteria.ScopeMatchByPrefix>.</span><span class="sxs-lookup"><span data-stu-id="dadea-134">If a scope-matching rule is not specified, <xref:System.ServiceModel.Discovery.FindCriteria.ScopeMatchByPrefix> is used.</span></span>

## <a name="findresponse"></a><span data-ttu-id="dadea-140">FindResponse</span><span class="sxs-lookup"><span data-stu-id="dadea-140">FindResponse</span></span>  
 <span data-ttu-id="dadea-141"><xref:System.ServiceModel.Discovery.FindResponse> имеет свойство коллекции <xref:System.ServiceModel.Discovery.FindResponse.Endpoints%2A>, в котором содержатся все ответы, отправленные соответствующими службами в сети.</span><span class="sxs-lookup"><span data-stu-id="dadea-141"><xref:System.ServiceModel.Discovery.FindResponse> has an <xref:System.ServiceModel.Discovery.FindResponse.Endpoints%2A> collection property that contains any replies sent by matching services on the network.</span></span> <span data-ttu-id="dadea-142">Если ни одна служба не отправила ответ, коллекция пустая.</span><span class="sxs-lookup"><span data-stu-id="dadea-142">If no services replied, the collection is empty.</span></span> <span data-ttu-id="dadea-143">Если ответила одна или несколько служб, все ответы сохраняются в объекте <xref:System.ServiceModel.Discovery.EndpointDiscoveryMetadata>, в котором содержится адрес, контракт и дополнительные сведения о службе.</span><span class="sxs-lookup"><span data-stu-id="dadea-143">If one or more services replied, each reply is stored in an <xref:System.ServiceModel.Discovery.EndpointDiscoveryMetadata> object, which contains the address, contract, and some additional information about the service.</span></span>  
  
 <span data-ttu-id="dadea-144">В следующем примере описывается выполнение операции поиска в коде.</span><span class="sxs-lookup"><span data-stu-id="dadea-144">The following example shows how to perform a find operation in code.</span></span>  
  
```  
// Create DiscoveryClient  
DiscoveryClient discoveryClient = new DiscoveryClient(new UdpDiscoveryEndpoint());  
  
// Create FindCriteria  
FindCriteria findCriteria = new FindCriteria(typeof(IPrinterService));  
findCriteria.Scopes.Add(new Uri("http://www.contoso.com/building1/floor1"));  
findCriteria.Duration = TimeSpan.FromSeconds(10);   
  
// Find ICalculatorService endpoints              
FindResponse findResponse = discoveryClient.Find(findCriteria);  
  
Console.WriteLine("Found {0} ICalculatorService endpoint(s).", findResponse.Endpoints.Count)  
```
