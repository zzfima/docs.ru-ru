# <a name="streaming-message-transfer"></a><span data-ttu-id="d7c05-102">Потоковая передача сообщений</span><span class="sxs-lookup"><span data-stu-id="d7c05-102">Streaming Message Transfer</span></span>
<span data-ttu-id="d7c05-103">В транспортных протоколах [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)] поддерживаются два режима передачи сообщений.</span><span class="sxs-lookup"><span data-stu-id="d7c05-103">[!INCLUDE[indigo1](../../../../includes/indigo1-md.md)] transports support two modes for transferring messages:</span></span>

-   <span data-ttu-id="d7c05-106">При потоковой передаче сообщение передается в виде потока.</span><span class="sxs-lookup"><span data-stu-id="d7c05-106">Streamed transfers expose the message as a stream.</span></span> <span data-ttu-id="d7c05-107">Получатель начинает обработку сообщения до того, как оно будет полностью доставлено.</span><span class="sxs-lookup"><span data-stu-id="d7c05-107">The receiver starts processing the message before it is completely delivered.</span></span>  
  
-   <span data-ttu-id="d7c05-108">Потоковая передача позволяет улучшить возможности масштабирования службы за счет исключения необходимости использовать большие буферы памяти.</span><span class="sxs-lookup"><span data-stu-id="d7c05-108">Streamed transfers can improve the scalability of a service by eliminating the requirement for large memory buffers.</span></span> <span data-ttu-id="d7c05-109">Степень улучшения масштабируемости при изменении режима передачи зависит от размера передаваемых сообщений.</span><span class="sxs-lookup"><span data-stu-id="d7c05-109">Whether changing the transfer mode improves scalability depends on the size of the messages being transferred.</span></span> <span data-ttu-id="d7c05-110">Для сообщений большого размера предпочтительнее использовать потоковую передачу.</span><span class="sxs-lookup"><span data-stu-id="d7c05-110">Large message sizes favor using streamed transfers.</span></span>  
  
 <span data-ttu-id="d7c05-111">По умолчанию транспорты HTTP, TCP/IP и именованных каналов используют буферизованную передачу.</span><span class="sxs-lookup"><span data-stu-id="d7c05-111">By default, the HTTP, TCP/IP, and named pipe transports use buffered transfers.</span></span> <span data-ttu-id="d7c05-112">Этот документ описывает изменение режима передачи с буферизованной на потоковую для этих транспортов, а также возможные последствия.</span><span class="sxs-lookup"><span data-stu-id="d7c05-112">This document describes how to switch these transports from a buffered to streamed transfer mode and the consequences of doing so.</span></span>

 <span data-ttu-id="d7c05-118">Привязки <xref:System.ServiceModel.BasicHttpBinding>, <xref:System.ServiceModel.NetTcpBinding> и <xref:System.ServiceModel.NetNamedPipeBinding> предоставляют свойство <xref:System.ServiceModel.TransferMode>.</span><span class="sxs-lookup"><span data-stu-id="d7c05-118">The <xref:System.ServiceModel.BasicHttpBinding>, <xref:System.ServiceModel.NetTcpBinding>, and <xref:System.ServiceModel.NetNamedPipeBinding> bindings expose the <xref:System.ServiceModel.TransferMode> property.</span></span> <span data-ttu-id="d7c05-119">Для других транспортов необходимо создать пользовательскую привязку для задания режима передачи.</span><span class="sxs-lookup"><span data-stu-id="d7c05-119">For other transports, you must create a custom binding to set the transfer mode.</span></span>

## <a name="enabling-asynchronous-streaming"></a><span data-ttu-id="d7c05-126">Включение асинхронной потоковой передачи</span><span class="sxs-lookup"><span data-stu-id="d7c05-126">Enabling Asynchronous Streaming</span></span>

 <span data-ttu-id="d7c05-128">Эта версия WCF также добавляет возможность истинной асинхронной потоковой передачи на стороне отправки.</span><span class="sxs-lookup"><span data-stu-id="d7c05-128">This version of WCF also adde the capability of true asynchronous streaming on the send side.</span></span> <span data-ttu-id="d7c05-129">Это повышает масштабируемость службы в сценариях, когда обрабатываются потоковые сообщения для нескольких клиентов. ## <a name="restrictions-on-streamed-transfers"></a><span data-ttu-id="d7c05-132">Ограничения потоковой передачи</span><span class="sxs-lookup"><span data-stu-id="d7c05-132">Restrictions on Streamed Transfers</span></span>

 <span data-ttu-id="d7c05-134">Операции, происходящие в потоковом транспорте, могут иметь контракт не более чем с одним входным или выходным параметром.</span><span class="sxs-lookup"><span data-stu-id="d7c05-134">Operations that occur across a streamed transport can have a contract with at most one input or output parameter.</span></span> <span data-ttu-id="d7c05-135">Такой параметр соответствует всему телу сообщения и должен принадлежать к типу <xref:System.ServiceModel.Channels.Message>, наследуемому от <xref:System.IO.Stream>, или реализации <xref:System.Xml.Serialization.IXmlSerializable>.</span><span class="sxs-lookup"><span data-stu-id="d7c05-135">That parameter corresponds to the entire body of the message and must be a <xref:System.ServiceModel.Channels.Message>, a derived type of <xref:System.IO.Stream>, or an <xref:System.Xml.Serialization.IXmlSerializable> implementation.</span></span> <span data-ttu-id="d7c05-136">Наличие возвращаемого значения операции эквивалентно наличию выходного параметра.</span><span class="sxs-lookup"><span data-stu-id="d7c05-136">Having a return value for an operation is equivalent to having an output parameter.</span></span>  
  
 <span data-ttu-id="d7c05-137">Некоторые функции [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)], такие как надежный обмен сообщениями, транзакции и безопасность уровня сообщений SOAP, используют буферизацию сообщений при передаче.</span><span class="sxs-lookup"><span data-stu-id="d7c05-137">Some [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] features, such as reliable messaging, transactions, and SOAP message-level security, rely on buffering messages for transmissions.</span></span> <span data-ttu-id="d7c05-138">Использование этих возможностей может уменьшить или свести до минимума повышение производительности, достигаемое за счет потоковой передачи.</span><span class="sxs-lookup"><span data-stu-id="d7c05-138">Using these features may reduce or eliminate the performance benefits gained by using streaming.</span></span> <span data-ttu-id="d7c05-139">Для защиты потокового транспорта следует использовать либо безопасность только на транспортном уровне, либо безопасность на транспортном уровне и безопасность сообщений только с проверкой подлинности.</span><span class="sxs-lookup"><span data-stu-id="d7c05-139">To secure a streamed transport, use transport-level security only or use transport-level security plus authentication-only message security.</span></span>
