- <span data-ttu-id="73b77-113">[Надежные сеансы](../../../docs/framework/wcf/feature-details/reliable-sessions.md), обеспечивающие надежную передачу без сильного повышения задержки.</span><span class="sxs-lookup"><span data-stu-id="73b77-113">[Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md), which offers reliable transfer without the cost of high latency.</span></span>  
  
- <span data-ttu-id="73b77-114">[Очереди в WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md), обеспечивающие надежную передачу и разделение между источником и назначением.</span><span class="sxs-lookup"><span data-stu-id="73b77-114">[Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md), which offers both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="73b77-115">Надежные сеансы</span><span class="sxs-lookup"><span data-stu-id="73b77-115">Reliable Sessions</span></span>  
 <span data-ttu-id="73b77-116">Надежные сеансы обеспечивают надежную сквозную передачу сообщений между источником и назначением с помощью протокола WS-Reliable Messaging независимо от количества и типа посредников между конечными точками обмена сообщениями (источником и назначением),</span><span class="sxs-lookup"><span data-stu-id="73b77-116">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-Reliable Messaging protocol, regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="73b77-117">В него входят любые транспортные посредники, не использующие протокол SOAP (например, прокси-серверы HTTP), и посредники, использующие этот протокол (например, мосты и маршрутизаторы на базе SOAP), необходимые для продвижения сообщений между конечными точками.</span><span class="sxs-lookup"><span data-stu-id="73b77-117">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="73b77-118">Надежные сеансы используют окно передачи в памяти для маскировки сбоев на уровне сообщений SOAP и повторной установки соединений при транспортных сбоях.</span><span class="sxs-lookup"><span data-stu-id="73b77-118">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="73b77-119">Надежные сеансы обеспечивают надежную передачу сообщений с низкой задержкой.</span><span class="sxs-lookup"><span data-stu-id="73b77-119">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="73b77-120">Они обеспечивают доставку сообщений протокола SOAP через любые прокси или маршрутизаторы подобно тому, как протокол TCP обеспечивает доставку пакетов через IP-мосты.</span><span class="sxs-lookup"><span data-stu-id="73b77-120">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="73b77-121">Дополнительные сведения о надежных сеансах см. в разделе [надежные сеансы](../../../docs/framework/wcf/feature-details/reliable-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="73b77-121">For more information about reliable sessions, see [Reliable Sessions](../../../docs/framework/wcf/feature-details/reliable-sessions.md).</span></span>  
  
### <a name="queues"></a><span data-ttu-id="73b77-122">Очереди</span><span class="sxs-lookup"><span data-stu-id="73b77-122">Queues</span></span>  
 <span data-ttu-id="73b77-123">Очереди в WCF обеспечивают надежную передачу сообщений и отделение источника от назначения ценой высокой задержки.</span><span class="sxs-lookup"><span data-stu-id="73b77-123">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="73b77-124">WCF в очереди обмена данными построена на основе очереди сообщений (MSMQ).</span><span class="sxs-lookup"><span data-stu-id="73b77-124">WCF queued communication is built on top of Message Queuing (MSMQ).</span></span>  
  
 <span data-ttu-id="73b77-125">MSMQ поставляется как дополнительный компонент с Windows.</span><span class="sxs-lookup"><span data-stu-id="73b77-125">MSMQ ships as an optional component with Windows.</span></span> <span data-ttu-id="73b77-126">Служба MSMQ работает как служба Windows.</span><span class="sxs-lookup"><span data-stu-id="73b77-126">The MSMQ service runs as a Windows Service.</span></span> <span data-ttu-id="73b77-127">MSMQ помещает передаваемые сообщения в очередь передачи от имени источника и доставляет их в целевую очередь.</span><span class="sxs-lookup"><span data-stu-id="73b77-127">It captures messages for transmission in a transmission queue on behalf of the source and delivers it to a target queue.</span></span> <span data-ttu-id="73b77-128">Целевая очередь принимает сообщения от имени назначения для последующей доставки при получении запроса от назначения.</span><span class="sxs-lookup"><span data-stu-id="73b77-128">The target queue accepts messages on behalf of the destination for later delivery whenever the destination requests messages.</span></span> <span data-ttu-id="73b77-129">Диспетчеры MSMQ реализуют протокол надежной передачи сообщений, благодаря которому сообщения не теряются при передаче.</span><span class="sxs-lookup"><span data-stu-id="73b77-129">The MSMQ managers implement a reliable message-transfer protocol so that messages are not lost in transmission.</span></span> <span data-ttu-id="73b77-130">Этот протокол может быть собственным или основанным на SOAP (протокол SRMP).</span><span class="sxs-lookup"><span data-stu-id="73b77-130">The protocol can be native or a SOAP-based protocol called SOAP Reliable Messaging Protocol (SRMP).</span></span>  
  
 <span data-ttu-id="73b77-131">Разделение в сочетании с надежностью передачи сообщений между очередями обеспечивает надежную передачу данных между слабо связанными приложениями.</span><span class="sxs-lookup"><span data-stu-id="73b77-131">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="73b77-132">В отличие от технологии надежных сеансов, не требуется, чтобы источник и назначение выполнялись одновременно.</span><span class="sxs-lookup"><span data-stu-id="73b77-132">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="73b77-133">Таким образом, возможны сценарии, в которых очереди фактически используются в качестве механизма выравнивания нагрузки при несоответствии скорости создания сообщений источником и их обработки в пункте назначения.</span><span class="sxs-lookup"><span data-stu-id="73b77-133">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when the source's rate of message production and the destination's rate of the message consumption do not match.</span></span> <span data-ttu-id="73b77-134">Дополнительные сведения об очередях см. в разделе [очереди в WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span><span class="sxs-lookup"><span data-stu-id="73b77-134">For more information about queues, see [Queues in WCF](../../../docs/framework/wcf/feature-details/queues-in-wcf.md).</span></span>
