# <a name="putinstancewmi-function"></a>Функция Путинстанцевми

Создает или обновляет экземпляр существующего класса. Экземпляр записывается в репозиторий WMI.

[!INCLUDE[internalonly-unmanaged](../../../../includes/internalonly-unmanaged.md)]

## <a name="syntax"></a>Синтаксис

```cpp
HRESULT PutInstanceWmi (
   [in] IWbemClassObject*    pInst,
   [in] long                 lFlags,
   [in] IWbemContext*        pCtx,
   [out] IWbemCallResult**   ppCallResult
);
```

## <a name="parameters"></a>Параметры

`pInst`\
окне Указатель на экземпляр, который необходимо записать.

`lFlags`\
окне Сочетание флагов, влияющих на поведение этой функции. Следующие значения определены в файле заголовка *вбемкли. h* , или их можно определить как константы в коде:

|Константа  |значения  |Описание  |
|---------|---------|---------|
| `WBEM_FLAG_USE_AMENDED_QUALIFIERS` | 0x20000 | Если задано, Инструментарий WMI не сохраняет никаких квалификаторов с **измененной** разновидностью. <br> Если не задано, предполагается, что этот объект не локализован, а все квалификаторы хранятся в этом экземпляре. |
| `WBEM_FLAG_CREATE_OR_UPDATE` | 0 | Создайте экземпляр, если он не существует, или перезапишите его, если он уже существует. |
| `WBEM_FLAG_UPDATE_ONLY` | 1 | Обновите экземпляр. Для успешного вызова должен существовать экземпляр. |
| `WBEM_FLAG_CREATE_ONLY` | 2 | Создайте экземпляр. Вызов завершается ошибкой, если экземпляр уже существует. |
| `WBEM_FLAG_RETURN_IMMEDIATELY` | 0x10 | Флаг вызывает вызов семисинчронаус. |

`pCtx`\
окне Как правило, это значение `null`. В противном случае он является указателем на экземпляр [ивбемконтекст](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemcontext) , который может использоваться поставщиком, который предоставляет запрошенные классы.

`ppCallResult`\
заполняет Если `null`, этот параметр не используется. Если `lFlags` содержит `WBEM_FLAG_RETURN_IMMEDIATELY`, функция немедленно возвращает значение `WBEM_S_NO_ERROR`. Параметр `ppCallResult` получает указатель на новый объект [IWbemCallResult](/windows/desktop/api/wbemcli/nn-wbemcli-iwbemcallresult).

## <a name="return-value"></a>Возвращаемое значение

Следующие значения, возвращаемые этой функцией, определены в файле заголовка *вбемкли. h* , или их можно определить как константы в коде:

|Константа  |значения  |Описание  |
|---------|---------|---------|
| `WBEM_E_ACCESS_DENIED` | 0x80041003 | У пользователя нет разрешения на обновление экземпляра указанного класса. |
| `WBEM_E_FAILED` | 0x80041001 | Произошла неопределенная ошибка. |
| `WBEM_E_INVALID_CLASS` | 0x80041010 | Недопустимый класс, поддерживающий данный экземпляр. |
| `WBEM_E_ILLEGAL_NULL` | 0x80041028 | `null` был указан для свойства, которое не может быть `null`, например, которое помечено квалификатором **индексирования** или **NOT_NULL** . |
| `WBEM_E_INVALID_OBJECT` | 0x8004100f | Указан недопустимый экземпляр. (Например, вызов `PutInstanceWmi` с классом возвращает это значение.) |
| `WBEM_E_INVALID_PARAMETER` | 0x80041008 | Недопустимый параметр. |
| `WBEM_E_ALREADY_EXISTS` | 0x80041019 | Указан флаг `WBEM_FLAG_CREATE_ONLY`, но экземпляр уже существует. |
| `WBEM_E_NOT_FOUND` | 0x80041002 | `WBEM_FLAG_UPDATE_ONLY` был указан в `lFlags`, но экземпляр не существует. |
| `WBEM_E_OUT_OF_MEMORY` | 0x80041006 | Недостаточно памяти для завершения операции. |
| `WBEM_E_SHUTTING_DOWN` | 0x80041033 | Вероятно, Инструментарий WMI остановлен и перезапущен. Снова вызовите [коннектсервервми](connectserverwmi.md) . |
| `WBEM_E_TRANSPORT_FAILURE` | 0x80041015 | Не удалось создать ссылку на удаленный вызов процедур (RPC) между текущим процессом и WMI. |
| `WBEM_S_NO_ERROR` | 0 | Вызов функции выполнен успешно. |

## <a name="remarks"></a>Заметки

Эта функция заключает вызов метода [IWbemServices::P утинстанце](/windows/desktop/api/wbemcli/nf-wbemcli-iwbemservices-putinstance) .

Функция `PutInstanceWmi` поддерживает создание экземпляров и обновление только экземпляров существующих классов.  В зависимости от того, как задан параметр `pCtx`, обновляются либо некоторые, либо все свойства экземпляра.

Когда экземпляр, на который указывает `pInst`, принадлежит подклассу, управление Windows вызывает всех поставщиков, ответственных за классы, от которых наследуется подкласс. Все эти поставщики должны быть выполнены для того, чтобы исходный запрос на `PutInstanceWmi` был выполнен. Поставщик, поддерживающий самый верхний класс в иерархии, вызывается первым. Порядок вызова продолжается с подклассом самого верхнего класса и продолжается сверху вниз до тех пор, пока Управление Windows не достигнет поставщика для класса, которому принадлежит экземпляр, на который указывает `pInst`.
Система управления Windows не вызывает поставщики для любого из дочерних классов экземпляра.

Когда приложение должно обновить экземпляр, принадлежащий иерархии классов, параметр `pInst` должен указывать на экземпляр, содержащий свойства, которые необходимо изменить. То есть рассмотрим целевой экземпляр, принадлежащий **ClassB**. Экземпляр **ClassB** является производным от **класса**a, а **класс** a определяет свойство **prop**. Если приложению требуется внести изменение в значение **prop** a в экземпляре **ClassB** , оно должно задать `pInst` для этого экземпляра, а не экземпляра **класса**a.

Вызов `PutInstanceWmi` в экземпляре абстрактного класса не допускается.

Если вызов функции завершается неудачно, можно получить дополнительные сведения об ошибке, вызвав функцию [жетерроринфо](geterrorinfo.md) .
