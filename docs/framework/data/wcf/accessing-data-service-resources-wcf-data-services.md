## <a name="addressing-resources"></a><span data-ttu-id="62b9e-107">Обращение к ресурсам</span><span class="sxs-lookup"><span data-stu-id="62b9e-107">Addressing Resources</span></span>  
 <span data-ttu-id="62b9e-108">В службах [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)] обращение к данным, предоставленным моделью данных, осуществляется с использованием URI.</span><span class="sxs-lookup"><span data-stu-id="62b9e-108">In [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)], you address any data exposed by the data model by using a URI.</span></span> <span data-ttu-id="62b9e-109">Например следующий URI возвращает канал, который представляет набор сущностей клиентов, который содержит записи для всех экземпляров типа сущности Customer:</span><span class="sxs-lookup"><span data-stu-id="62b9e-109">For example, the following URI returns a feed that is the Customers entity set, which contains entries for all instances of the Customer entity type:</span></span>

```  
  
 <span data-ttu-id="62b9e-110">Сущности имеют специальные свойства, называемые ключами сущностей.</span><span class="sxs-lookup"><span data-stu-id="62b9e-110">Entities have special properties called entity keys.</span></span> <span data-ttu-id="62b9e-111">Ключ сущности используется для уникальной идентификации одной сущности в наборе сущностей.</span><span class="sxs-lookup"><span data-stu-id="62b9e-111">An entity key is used to uniquely identify a single entity in an entity set.</span></span> <span data-ttu-id="62b9e-112">Это позволяет адресовать конкретный экземпляр типа сущности в наборе сущностей.</span><span class="sxs-lookup"><span data-stu-id="62b9e-112">This enables you to address a specific instance of an entity type in the entity set.</span></span> <span data-ttu-id="62b9e-113">Например, следующий URI возвращает запись для определенного экземпляра типа сущности Customer, имеющего значение ключа `ALFKI`:</span><span class="sxs-lookup"><span data-stu-id="62b9e-113">For example, the following URI returns an entry for a specific instance of the Customer entity type that has a key value of `ALFKI`:</span></span>

```  
  
 <span data-ttu-id="62b9e-114">К свойствам примитивного типа и сложным свойствам экземпляра сущности можно обращаться по отдельности.</span><span class="sxs-lookup"><span data-stu-id="62b9e-114">Primitive and complex properties of an entity instance can also be individually addressed.</span></span> <span data-ttu-id="62b9e-115">Например, следующий URI возвращает XML-элемент, содержащий значение свойства `ContactName` для конкретной сущности Customer:</span><span class="sxs-lookup"><span data-stu-id="62b9e-115">For example, the following URI returns an XML element that contains the `ContactName` property value for a specific Customer:</span></span>

```  
  
 <span data-ttu-id="62b9e-116">При включении конечной точки `$value` в предыдущий URI в ответном сообщении возвращается только значение свойства примитивного типа.</span><span class="sxs-lookup"><span data-stu-id="62b9e-116">When you include the `$value` endpoint in the previous URI, only the value of the primitive property is returned in the response message.</span></span> <span data-ttu-id="62b9e-117">В следующем примере возвращается только строка «Maria Anders» без XML-элемента:</span><span class="sxs-lookup"><span data-stu-id="62b9e-117">The following example returns only the string "Maria Anders" without the XML element:</span></span>

```  
  
 <span data-ttu-id="62b9e-118">Связи между сущностями определяются в модели данных с помощью ассоциаций.</span><span class="sxs-lookup"><span data-stu-id="62b9e-118">Relationships between entities are defined in the data model by associations.</span></span> <span data-ttu-id="62b9e-119">Эти ассоциации позволяют адресовать набор связанных сущностей с помощью свойств навигации экземпляра сущности.</span><span class="sxs-lookup"><span data-stu-id="62b9e-119">These associations enable you to address related entities by using navigation properties of an entity instance.</span></span> <span data-ttu-id="62b9e-120">Свойство навигации возвращает либо одну связанную сущность в случае связи «многие к одному», либо набор связанных сущностей в случае связи «один ко многим».</span><span class="sxs-lookup"><span data-stu-id="62b9e-120">A navigation property can return either a single related entity, in the case of a many-to-one relationship, or a set of related entities, in the case of a one-to-many relationship.</span></span> <span data-ttu-id="62b9e-121">Например, следующий URI возвращает канал, который представляет набор всех заказов, связанных с конкретной сущностью Customer:</span><span class="sxs-lookup"><span data-stu-id="62b9e-121">For example, the following URI returns a feed that is the set of all the Orders that are related to a specific Customer:</span></span>

```  
  
 <span data-ttu-id="62b9e-122">Связи, которые обычно являются двунаправленными, представлены парой свойств навигации.</span><span class="sxs-lookup"><span data-stu-id="62b9e-122">Relationships, which are usually bi-directional, are represented by a pair of navigation properties.</span></span> <span data-ttu-id="62b9e-123">В отличие от связей, описанных в предыдущем примере, следующий URI возвращает ссылку на сущность Customer, к которой принадлежит определенная сущность Order:</span><span class="sxs-lookup"><span data-stu-id="62b9e-123">As the reverse of the relationship shown in the previous example, the following URI returns a reference to the Customer entity to which a specific Order entity belongs:</span></span>

```  
  
 [!INCLUDE[ssODataShort](../../../../includes/ssodatashort-md.md)]<span data-ttu-id="62b9e-124"> позволяют также адресовать ресурсы на основе результатов выражений запросов.</span><span class="sxs-lookup"><span data-stu-id="62b9e-124"> also enables you to address resources based on the results of query expressions.</span></span> <span data-ttu-id="62b9e-125">Это дает возможность фильтровать наборы ресурсов на основе вычисленного выражения.</span><span class="sxs-lookup"><span data-stu-id="62b9e-125">This makes it possible to filter sets of resources based on an evaluated expression.</span></span> <span data-ttu-id="62b9e-126">Например, следующий URI фильтрует для возврата только заказы конкретного клиента, доставленные с 22 сентября 1997 г.</span><span class="sxs-lookup"><span data-stu-id="62b9e-126">For example, the following URI filters the resources to return only the Orders for the specified Customer that have shipped since September 22, 1997:</span></span>

 <span data-ttu-id="62b9e-127">Дополнительные сведения см. в разделе [OData: соглашения URI](https://go.microsoft.com/fwlink/?LinkId=185564).</span><span class="sxs-lookup"><span data-stu-id="62b9e-127">For more information, see [OData: URI Conventions](https://go.microsoft.com/fwlink/?LinkId=185564).</span></span>
