# <a name="namespaces-entity-sql"></a><span data-ttu-id="f3125-102">Пространства имен (Entity SQL)</span><span class="sxs-lookup"><span data-stu-id="f3125-102">Namespaces (Entity SQL)</span></span>
<span data-ttu-id="f3125-103">В [!INCLUDE[esql](../../../../../../includes/esql-md.md)] были введены пространства имен, чтобы избежать возникновения конфликтов с именами глобальных идентификаторов, например именами типов, наборов сущностей, функций и т. д.</span><span class="sxs-lookup"><span data-stu-id="f3125-103">[!INCLUDE[esql](../../../../../../includes/esql-md.md)] introduces namespaces to avoid name conflicts for global identifiers such as type names, entity sets, functions, and so on.</span></span> <span data-ttu-id="f3125-104">Поддержка пространства имен в [!INCLUDE[esql](../../../../../../includes/esql-md.md)] аналогична поддержке пространства имен в .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="f3125-104">The namespace support in [!INCLUDE[esql](../../../../../../includes/esql-md.md)] is similar to the namespace support in the .NET Framework.</span></span>  
  
 [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="f3125-105">предоставляет две формы предложения USING: полные пространства имен (для пространства имен предоставляется короткий псевдоним) и неполные пространства имен, как показано в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="f3125-105">provides two forms of the USING clause: qualified namespaces (where a shorter alias is provided for the namespace), and unqualified namespaces, as illustrated in the following example:</span></span>  
  
 `USING System.Data;`  
  
 `USING tsql = System.Data;`  
  
## <a name="name-resolution-rules"></a><span data-ttu-id="f3125-106">Правила разрешения имен</span><span class="sxs-lookup"><span data-stu-id="f3125-106">Name Resolution Rules</span></span>  
 <span data-ttu-id="f3125-107">Если идентификатор не может быть разрешен в локальных областях, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] пытается определить имя в глобальных областях (пространства имен).</span><span class="sxs-lookup"><span data-stu-id="f3125-107">If an identifier cannot be resolved in the local scopes, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to locate the name in the global scopes (the namespaces).</span></span> [!INCLUDE[esql](../../../../../../includes/esql-md.md)] <span data-ttu-id="f3125-108">вначале пытается сопоставить идентификатор (префикс) с одним из полных пространств имен.</span><span class="sxs-lookup"><span data-stu-id="f3125-108">first tries to match the identifier (prefix) with one of the qualified namespaces.</span></span> <span data-ttu-id="f3125-109">Если найдено совпадение, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] пытается разрешить остальную часть идентификатора в указанном пространстве имен.</span><span class="sxs-lookup"><span data-stu-id="f3125-109">If there is a match, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to resolve the rest of the identifier in the specified namespace.</span></span> <span data-ttu-id="f3125-110">Если совпадение не найдено, вызывается исключение.</span><span class="sxs-lookup"><span data-stu-id="f3125-110">If no match is found, an exception is thrown.</span></span>  
  
 <span data-ttu-id="f3125-111">Затем [!INCLUDE[esql](../../../../../../includes/esql-md.md)] пытается выполнить поиск идентификатора в всех неполных пространствах имен (указанных в прологе).</span><span class="sxs-lookup"><span data-stu-id="f3125-111">Next, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] tries to search all unqualified namespaces (specified in the prolog) for the identifier.</span></span> <span data-ttu-id="f3125-112">Если идентификатор удается найти только в одном пространстве имен, возвращается расположение.</span><span class="sxs-lookup"><span data-stu-id="f3125-112">If the identifier can be located in exactly one namespace, that location is returned.</span></span> <span data-ttu-id="f3125-113">Если для идентификатора обнаружены совпадения в нескольких пространствах имен, вызывается исключение.</span><span class="sxs-lookup"><span data-stu-id="f3125-113">If more than one namespace has a match for that identifier, an exception is thrown.</span></span> <span data-ttu-id="f3125-114">Если для идентификатора не может быть определено пространство имен, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] передает имя в следующую наружную область (объект <xref:System.Data.Common.DbCommand> или <xref:System.Data.Common.DbConnection>), как показано в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="f3125-114">If no namespace can be identified for the identifier, [!INCLUDE[esql](../../../../../../includes/esql-md.md)] passes the name onto the next outward scope (the <xref:System.Data.Common.DbCommand> or <xref:System.Data.Common.DbConnection> object), as illustrated in the following example:</span></span>  
  
```sql  
SELECT TREAT(p AS NamespaceName.Employee)  
FROM ContainerName.Person AS p  
WHERE p IS OF (NamespaceName.Employee)  
```
