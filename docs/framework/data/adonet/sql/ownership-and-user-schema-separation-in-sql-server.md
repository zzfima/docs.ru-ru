<span data-ttu-id="ce75c-103">Основным принципом безопасности SQL Server является то, что владельцы объектов имеют неотзываемые разрешения на их администрирование.</span><span class="sxs-lookup"><span data-stu-id="ce75c-103">A core concept of SQL Server security is that owners of objects have irrevocable permissions to administer them.</span></span> <span data-ttu-id="ce75c-104">Нельзя удалять права доступа у владельцев объектов. ## <a name="user-schema-separation"></a><span data-ttu-id="ce75c-105">Отделение пользователей от схем</span><span class="sxs-lookup"><span data-stu-id="ce75c-105">User-Schema Separation</span></span>  
 <span data-ttu-id="ce75c-106">Отделение пользователей от схем обеспечивает дополнительную гибкость в управлении разрешениями объектов базы данных.</span><span class="sxs-lookup"><span data-stu-id="ce75c-106">User-schema separation allows for more flexibility in managing database object permissions.</span></span> <span data-ttu-id="ce75c-107">Объект *схемы* представляет собой именованный контейнер для объектов базы данных, который позволяет выполнять группирование по отдельным пространствам имен.</span><span class="sxs-lookup"><span data-stu-id="ce75c-107">A *schema* is a named container for database objects, which allows you to group objects into separate namespaces.</span></span> <span data-ttu-id="ce75c-108">Например, образец базы данных AdventureWorks содержит схемы для Production, Sales и HumanResources.</span><span class="sxs-lookup"><span data-stu-id="ce75c-108">For example, the AdventureWorks sample database contains schemas for Production, Sales, and HumanResources.</span></span>  
  
 <span data-ttu-id="ce75c-109">Четырехкомпонентный синтаксис ссылок на объекты указывает имя схемы.</span><span class="sxs-lookup"><span data-stu-id="ce75c-109">The four-part naming syntax for referring to objects specifies the schema name.</span></span>  
  
```  
Server.Database.DatabaseSchema.DatabaseObject
```  

### <a name="schema-owners-and-permissions"></a><span data-ttu-id="ce75c-110">Владельцы схемы и разрешения</span><span class="sxs-lookup"><span data-stu-id="ce75c-110">Schema Owners and Permissions</span></span>  
 <span data-ttu-id="ce75c-111">Владельцем схемы может быть любой участник базы данных, при этом одному участнику может принадлежать несколько схем.</span><span class="sxs-lookup"><span data-stu-id="ce75c-111">Schemas can be owned by any database principal, and a single principal can own multiple schemas.</span></span> <span data-ttu-id="ce75c-112">К схеме можно применить правила безопасности, которые наследуются всеми объектами схемы.</span><span class="sxs-lookup"><span data-stu-id="ce75c-112">You can apply security rules to a schema, which are inherited by all objects in the schema.</span></span> <span data-ttu-id="ce75c-113">После установки разрешений доступа для схемы они автоматически применяются по мере добавления к схеме новых объектов.</span><span class="sxs-lookup"><span data-stu-id="ce75c-113">Once you set up access permissions for a schema, those permissions are automatically applied as new objects are added to the schema.</span></span> <span data-ttu-id="ce75c-114">Пользователь может назначить новую схему, а несколько пользователей базы данных могут совместно использовать одну одну схему.</span><span class="sxs-lookup"><span data-stu-id="ce75c-114">Users can be assigned a default schema, and multiple database users can share the same schema.</span></span>  
  
 <span data-ttu-id="ce75c-115">По умолчанию, если разработчик создает объект в схеме, он принадлежит участнику безопасности, являющемуся владельцем схемы, а не разработчику.</span><span class="sxs-lookup"><span data-stu-id="ce75c-115">By default, when developers create objects in a schema, the objects are owned by the security principal that owns the schema, not the developer.</span></span> <span data-ttu-id="ce75c-116">Владение объектом можно передать с помощью инструкции Transact-SQL ALTER AUTHORIZATION.</span><span class="sxs-lookup"><span data-stu-id="ce75c-116">Object ownership can be transferred with ALTER AUTHORIZATION Transact-SQL statement.</span></span> <span data-ttu-id="ce75c-117">Схема может также содержать объекты, принадлежащие другим пользователям и иметь более детализированные разрешения, чем назначенные схеме, хотя это не рекомендуется из-за увеличения сложности управления разрешениями.</span><span class="sxs-lookup"><span data-stu-id="ce75c-117">A schema can also contain objects that are owned by different users and have more granular permissions than those assigned to the schema, although this is not recommended because it adds complexity to managing permissions.</span></span> <span data-ttu-id="ce75c-118">Объекты можно перемещать из одной схемы в другую, а принадлежность схемы передавать от одного участника другому.</span><span class="sxs-lookup"><span data-stu-id="ce75c-118">Objects can be moved between schemas, and schema ownership can be transferred between principals.</span></span> <span data-ttu-id="ce75c-119">Пользователей базы данных можно удалять, не влияя этим на схемы.</span><span class="sxs-lookup"><span data-stu-id="ce75c-119">Database users can be dropped without affecting schemas.</span></span>  
  
### <a name="built-in-schemas"></a><span data-ttu-id="ce75c-120">Встроенные схемы</span><span class="sxs-lookup"><span data-stu-id="ce75c-120">Built-In Schemas</span></span>  
 <span data-ttu-id="ce75c-121">SQL Server поставляется с десятью предварительно определенными схемами, имеющими одинаковое имя со встроенными пользователями и ролями базы данных.</span><span class="sxs-lookup"><span data-stu-id="ce75c-121">SQL Server ships with ten pre-defined schemas that have the same names as the built-in database users and roles.</span></span> <span data-ttu-id="ce75c-122">Они существуют главным образом ради обратной совместимости.</span><span class="sxs-lookup"><span data-stu-id="ce75c-122">These exist mainly for backward compatibility.</span></span> <span data-ttu-id="ce75c-123">Схемы, имеющие одинаковые имена с предопределенными ролями базы данных, можно удалить, если они не нужны.</span><span class="sxs-lookup"><span data-stu-id="ce75c-123">You can drop the schemas that have the same names as the fixed database roles if you do not need them.</span></span> <span data-ttu-id="ce75c-124">Можно удалить следующие схемы.</span><span class="sxs-lookup"><span data-stu-id="ce75c-124">You cannot drop the following schemas:</span></span>

> [!NOTE]

#### <a name="the-dbo-schema"></a><span data-ttu-id="ce75c-128">Схема dbo</span><span class="sxs-lookup"><span data-stu-id="ce75c-128">The dbo Schema</span></span>

 <span data-ttu-id="ce75c-132">Пользователи, которым назначена схема `dbo`, не наследуют разрешения пользовательской учетной записи `dbo`.</span><span class="sxs-lookup"><span data-stu-id="ce75c-132">Users who are assigned the `dbo` schema do not inherit the permissions of the `dbo` user account.</span></span> <span data-ttu-id="ce75c-133">Пользователи не наследуют разрешения схемы, их наследуют объекты базы данных, содержащиеся в схеме.</span><span class="sxs-lookup"><span data-stu-id="ce75c-133">No permissions are inherited from a schema by users; schema permissions are inherited by the database objects contained in the schema.</span></span>
