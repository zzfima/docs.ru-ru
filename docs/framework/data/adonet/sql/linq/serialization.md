## <a name="overview"></a><span data-ttu-id="ef254-109">Обзор</span><span class="sxs-lookup"><span data-stu-id="ef254-109">Overview</span></span>  
 <span data-ttu-id="ef254-110">Код, сгенерированный [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] предоставляет возможности отложенной загрузки по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="ef254-110">The code generated by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] provides deferred loading capabilities by default.</span></span> <span data-ttu-id="ef254-111">Отложенная загрузка - очень удобное средство среднего уровня для прозрачной загрузки данных по требованию.</span><span class="sxs-lookup"><span data-stu-id="ef254-111">Deferred loading is very convenient on the mid-tier for transparent loading of data on demand.</span></span> <span data-ttu-id="ef254-112">Однако при сериализации с этим средством возникают проблемы, поскольку сериализатор инициирует отложенную загрузку независимо от намерений пользователя.</span><span class="sxs-lookup"><span data-stu-id="ef254-112">However, it is problematic for serialization, because the serializer triggers deferred loading whether deferred loading is intended or not.</span></span> <span data-ttu-id="ef254-113">В действительности, при сериализации объекта также сериализуется его транзитивное замыкание для всех исходящих ссылок с отложенной загрузкой.</span><span class="sxs-lookup"><span data-stu-id="ef254-113">In effect, when an object is serialized, its transitive closure under all outbound defer-loaded references is serialized.</span></span>  
  
 <span data-ttu-id="ef254-114">[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] Возможность сериализации устраняет эту проблему, в первую очередь, благодаря двум описанным далее механизмам:</span><span class="sxs-lookup"><span data-stu-id="ef254-114">The [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] serialization feature addresses this problem, primarily through two mechanisms:</span></span>  
  
-   <span data-ttu-id="ef254-115">Режим <xref:System.Data.Linq.DataContext> для выключения отложенной загрузки (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>).</span><span class="sxs-lookup"><span data-stu-id="ef254-115">A <xref:System.Data.Linq.DataContext> mode for turning off deferred loading (<xref:System.Data.Linq.DataContext.ObjectTrackingEnabled%2A>).</span></span> <span data-ttu-id="ef254-116">Для получения дополнительной информации см. <xref:System.Data.Linq.DataContext>.</span><span class="sxs-lookup"><span data-stu-id="ef254-116">For more information, see <xref:System.Data.Linq.DataContext>.</span></span>  
  
-   <span data-ttu-id="ef254-117">Переключение создания кода на создания атрибутов <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> и <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> в создаваемых классах.</span><span class="sxs-lookup"><span data-stu-id="ef254-117">A code-generation switch to generate <xref:System.Runtime.Serialization.DataContractAttribute?displayProperty=nameWithType> and <xref:System.Runtime.Serialization.DataMemberAttribute?displayProperty=nameWithType> attributes on generated entities.</span></span> <span data-ttu-id="ef254-118">Главной темой этого раздела является именно этот аспект, включая поведение классов с отложенной загрузкой при сериализации.</span><span class="sxs-lookup"><span data-stu-id="ef254-118">This aspect, including the behavior of defer-loading classes under serialization, is the major subject of this topic.</span></span>  
  
### <a name="definitions"></a><span data-ttu-id="ef254-119">Определения</span><span class="sxs-lookup"><span data-stu-id="ef254-119">Definitions</span></span>  
  
-   <span data-ttu-id="ef254-120">*Сериализатор DataContract*: по умолчанию сериализатор, используемый компонентом Windows Communication Framework (WCF), .NET Framework 3.0 или более поздней версии.</span><span class="sxs-lookup"><span data-stu-id="ef254-120">*DataContract serializer*: Default serializer used by the Windows Communication Framework (WCF) component of the .NET Framework 3.0 or later versions.</span></span>  
  
-   <span data-ttu-id="ef254-121">*Однонаправленная сериализация*: сериализованная версия класса, который содержит только однонаправленное свойство ассоциации (Чтобы избежать зацикливания).</span><span class="sxs-lookup"><span data-stu-id="ef254-121">*Unidirectional serialization*: The serialized version of a class that contains only a one-way association property (to avoid a cycle).</span></span> <span data-ttu-id="ef254-122">По соглашению, для сериализации помечается свойство родительской стороны отношения «первичный ключ - внешний ключ».</span><span class="sxs-lookup"><span data-stu-id="ef254-122">By convention, the property on the parent side of a primary-foreign key relationship is marked for serialization.</span></span> <span data-ttu-id="ef254-123">Другая сторона, участвующая в двунаправленном отношении, не сериализуется.</span><span class="sxs-lookup"><span data-stu-id="ef254-123">The other side in a bidirectional association is not serialized.</span></span>  
  
     <span data-ttu-id="ef254-124">Однонаправленная сериализация является единственным типом сериализации, поддерживаемым технологией [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)].</span><span class="sxs-lookup"><span data-stu-id="ef254-124">Unidirectional serialization is the only type of serialization supported by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)].</span></span>
