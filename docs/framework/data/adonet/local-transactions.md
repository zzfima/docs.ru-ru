 <span data-ttu-id="ea1e3-111">Для выполнения локальных транзакций каждый поставщик данных платформы [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] имеет свой собственный объект `Transaction`.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-111">Each of the [!INCLUDE[dnprdnshort](../../../../includes/dnprdnshort-md.md)] data providers has its own `Transaction` object for performing local transactions.</span></span> <span data-ttu-id="ea1e3-112">Если требуется выполнить транзакцию в базе данных SQL Server, выбирается транзакция <xref:System.Data.SqlClient>.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-112">If you require a transaction to be performed in a SQL Server database, select a <xref:System.Data.SqlClient> transaction.</span></span> <span data-ttu-id="ea1e3-113">Для транзакции Oracle используйте поставщик <xref:System.Data.OracleClient>.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-113">For an Oracle transaction, use the <xref:System.Data.OracleClient> provider.</span></span> <span data-ttu-id="ea1e3-114">Кроме того, <xref:System.Data.Common.DbTransaction> класс, который доступен для написания независимого от поставщика кода, который требуется транзакций.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-114">In addition, there is a <xref:System.Data.Common.DbTransaction> class that is available for writing provider-independent code that requires transactions.</span></span>

## <a name="performing-a-transaction-using-a-single-connection"></a><span data-ttu-id="ea1e3-117">Выполнение транзакций с использованием одного соединения</span><span class="sxs-lookup"><span data-stu-id="ea1e3-117">Performing a Transaction Using a Single Connection</span></span>  
 <span data-ttu-id="ea1e3-118">В [!INCLUDE[vstecado](../../../../includes/vstecado-md.md)] транзакции управляются объектом `Connection`.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-118">In [!INCLUDE[vstecado](../../../../includes/vstecado-md.md)], you control transactions with the `Connection` object.</span></span> <span data-ttu-id="ea1e3-119">Инициировать транзакцию можно с помощью метода `BeginTransaction`.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-119">You can initiate a local transaction with the `BeginTransaction` method.</span></span> <span data-ttu-id="ea1e3-120">После начала транзакции при помощи свойства `Transaction` объекта `Command` к ней можно прикрепить команду.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-120">Once you have begun a transaction, you can enlist a command in that transaction with the `Transaction` property of a `Command` object.</span></span> <span data-ttu-id="ea1e3-121">Затем в зависимости от успеха или ошибки компонентов транзакции можно зафиксировать или откатить изменения, сделанные в источнике данных.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-121">You can then commit or roll back modifications made at the data source based on the success or failure of the components of the transaction.</span></span>

 <span data-ttu-id="ea1e3-123">Область действия транзакции ограничена соединением.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-123">The scope of the transaction is limited to the connection.</span></span> <span data-ttu-id="ea1e3-124">В следующем примере выполняется явная транзакция, состоящая из двух отдельных команд в блоке `try`.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-124">The following example performs an explicit transaction that consists of two separate commands in the `try` block.</span></span> <span data-ttu-id="ea1e3-125">Команды выполняют инструкции INSERT для таблицы Production.ScrapReason в образце базы данных AdventureWorks в SQL Server, которые будут зафиксированы при отсутствии исключений.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-125">The commands execute INSERT statements against the Production.ScrapReason table in the AdventureWorks SQL Server sample database, which are committed if no exceptions are thrown.</span></span> <span data-ttu-id="ea1e3-126">При возникновении исключения код в блоке `catch` произведет откат транзакции.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-126">The code in the `catch` block rolls back the transaction if an exception is thrown.</span></span> <span data-ttu-id="ea1e3-127">При отмене транзакции или обрыве соединения до выполнения транзакции она откатывается автоматически.</span><span class="sxs-lookup"><span data-stu-id="ea1e3-127">If the transaction is aborted or the connection is closed before the transaction has completed, it is automatically rolled back.</span></span>
