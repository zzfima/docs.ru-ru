# <a name="entity-key"></a><span data-ttu-id="2b0f2-102">ключ сущности</span><span class="sxs-lookup"><span data-stu-id="2b0f2-102">entity key</span></span>
<span data-ttu-id="2b0f2-103">*Ключ сущности* — это [свойство](property.md) или набор свойств [типа сущности](entity-type.md) , которые используются для определения удостоверения.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-103">An *entity key* is a [property](property.md) or a set of properties of an [entity type](entity-type.md) that are used to determine identity.</span></span> <span data-ttu-id="2b0f2-104">Свойства, составляющие ключ сущности, выбираются во время разработки.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-104">The properties that make up an entity key are chosen at design time.</span></span> <span data-ttu-id="2b0f2-105">Значения свойств ключа сущности должны уникальным образом идентифицировать экземпляр типа сущности в [наборе сущностей](entity-set.md) во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-105">The values of entity key properties must uniquely identify an entity type instance within an [entity set](entity-set.md) at run time.</span></span> <span data-ttu-id="2b0f2-106">Свойства, составляющие ключ сущности, должны гарантировать уникальность экземпляра набора сущностей.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-106">The properties that make up an entity key should be chosen to guarantee uniqueness of instances in an entity set.</span></span>  
  
 <span data-ttu-id="2b0f2-107">Ниже приводятся требования к набору свойств, входящих в ключ сущности.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-107">The following are the requirements for a set of properties to be an entity key:</span></span>  
  
- <span data-ttu-id="2b0f2-108">Два ключа сущностей не могут быть идентичными в одном наборе сущностей.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-108">No two entity keys within an entity set can be identical.</span></span> <span data-ttu-id="2b0f2-109">То есть для любых двух сущностей внутри набора сущностей значения для всех свойств, составляющих ключ, не могут быть одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-109">That is, for any two entities within an entity set, the values for all of the properties that constitute a key cannot be the same.</span></span> <span data-ttu-id="2b0f2-110">Тем не менее, некоторые (но не все) значения, составляющие ключ сущности, могут быть одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-110">However, some (but not all) of the values that make up an entity key can be the same.</span></span>  
  
- <span data-ttu-id="2b0f2-111">Ключ сущности должен состоять из набора не допускающих значения NULL, неизменяемых [свойств типа-примитивов](entity-data-model-primitive-data-types.md).</span><span class="sxs-lookup"><span data-stu-id="2b0f2-111">An entity key must consist of a set of non-nullable, immutable, [primitive type properties](entity-data-model-primitive-data-types.md).</span></span>  
  
- <span data-ttu-id="2b0f2-112">Свойства, составляющие ключ сущности для данного типа сущности, не могут быть изменены.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-112">The properties that make up an entity key for a given entity type cannot change.</span></span> <span data-ttu-id="2b0f2-113">Для одного типа сущности возможен только один ключ сущности; суррогатные ключи не поддерживаются.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-113">You cannot allow more than one possible entity key for a given entity type; surrogate keys are not supported.</span></span>  
  
- <span data-ttu-id="2b0f2-114">Если сущность участвует в иерархии наследования, корневая сущность должна содержать все свойства, составляющие ключ сущности, а ключ сущности должен быть определен в типе корневой сущности.</span><span class="sxs-lookup"><span data-stu-id="2b0f2-114">When an entity is involved in an inheritance hierarchy, the root entity must contain all the properties that make up the entity key, and the entity key must be defined on the root entity type.</span></span> <span data-ttu-id="2b0f2-115">Дополнительные сведения см. в разделе [EDM: наследование](entity-data-model-inheritance.md).</span><span class="sxs-lookup"><span data-stu-id="2b0f2-115">For more information, see [Entity Data Model: Inheritance](entity-data-model-inheritance.md).</span></span>
