## <a name="windows-forms-in-com-client-applications"></a><span data-ttu-id="c14bd-129">Windows Forms в клиентских приложениях COM</span><span class="sxs-lookup"><span data-stu-id="c14bd-129">Windows Forms in COM client applications</span></span>  
 <span data-ttu-id="c14bd-130">При открытии формы Windows Forms из клиентского приложения COM, такого как приложение Visual Basic 6.0 или приложение MFC, форма может вести себя не так, как ожидается.</span><span class="sxs-lookup"><span data-stu-id="c14bd-130">When you open a Windows Form from a COM client application, such as a Visual Basic 6.0 application or an MFC application, the form may behave unexpectedly.</span></span> <span data-ttu-id="c14bd-131">Например, при нажатии клавиши TAB фокус не переходит от одного элемента управления к другому.</span><span class="sxs-lookup"><span data-stu-id="c14bd-131">For example, when you press the TAB key, the focus does not change from one control to another control.</span></span> <span data-ttu-id="c14bd-132">При нажатии клавиши ВВОД, если фокус установлен на кнопке, событие кнопки <xref:System.Windows.Forms.Control.Click> не возникает.</span><span class="sxs-lookup"><span data-stu-id="c14bd-132">When you press the ENTER key while a command button has focus, the button's <xref:System.Windows.Forms.Control.Click> event is not raised.</span></span> <span data-ttu-id="c14bd-133">Также может наблюдаться и другое неожиданное поведение при нажатии клавиш или действиях мышью.</span><span class="sxs-lookup"><span data-stu-id="c14bd-133">You may also experience unexpected behavior for keystrokes or mouse activity.</span></span>

 <span data-ttu-id="c14bd-136">Цикл обработки сообщений приложения является внутренним циклом программы, который получает сообщения из очереди сообщений потока, преобразует их и отправляет приложению для обработки.</span><span class="sxs-lookup"><span data-stu-id="c14bd-136">An application's message loop is an internal program loop that retrieves messages from a thread's message queue, translates them, and then sends them to the application to be handled.</span></span> <span data-ttu-id="c14bd-137">Архитектура цикла обработки сообщений для формы Windows Forms отличается от архитектуры, используемой в более ранних приложениях, таких как приложения Visual Basic 6.0 и приложения MFC.</span><span class="sxs-lookup"><span data-stu-id="c14bd-137">The message loop for a Windows Form does not have the same architecture as message loops that earlier applications, such as Visual Basic 6.0 applications and MFC applications, provide.</span></span> <span data-ttu-id="c14bd-138">Сообщения окна, которые отправляются в цикл обработки сообщений, могут обрабатываться не так, как ожидает форма Windows Forms.</span><span class="sxs-lookup"><span data-stu-id="c14bd-138">The window messages that are posted to the message loop may be handled differently than the Windows Form expects.</span></span> <span data-ttu-id="c14bd-139">Таким образом, может наблюдаться неожиданное поведение.</span><span class="sxs-lookup"><span data-stu-id="c14bd-139">Therefore, unexpected behavior may occur.</span></span> <span data-ttu-id="c14bd-140">Могут не работать некоторые сочетания клавиш и некоторые действия мыши, а также могут не возникать некоторые события.</span><span class="sxs-lookup"><span data-stu-id="c14bd-140">Some keystroke combinations may not work, some mouse activity may not work, or some events may not be raised as expected.</span></span>
