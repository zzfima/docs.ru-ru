# <a name="how-to-override-metadata-for-a-dependency-property"></a><span data-ttu-id="6574b-102">Практическое руководство. Переопределение метаданных для свойств зависимостей</span><span class="sxs-lookup"><span data-stu-id="6574b-102">How to: Override Metadata for a Dependency Property</span></span>
<span data-ttu-id="6574b-103">В этом примере показано, как переопределить метаданные свойства зависимостей по умолчанию, которое поступает из наследуемого класса, путем вызова <xref:System.Windows.DependencyProperty.OverrideMetadata%2A> с указанием конкретного типа метаданных.</span><span class="sxs-lookup"><span data-stu-id="6574b-103">This example shows how to override default dependency property metadata that comes from an inherited class, by calling the <xref:System.Windows.DependencyProperty.OverrideMetadata%2A> method and providing type-specific metadata.</span></span>  
  
## <a name="example"></a><span data-ttu-id="6574b-104">Пример</span><span class="sxs-lookup"><span data-stu-id="6574b-104">Example</span></span>  
 <span data-ttu-id="6574b-105">Определив его <xref:System.Windows.PropertyMetadata>, класс может определить поведение свойства зависимости, например, значение и свойство системы обратные вызовы по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="6574b-105">By defining its <xref:System.Windows.PropertyMetadata>, a class can define the dependency property's behaviors, such as its default value and property system callbacks.</span></span> <span data-ttu-id="6574b-106">Многие классы свойств зависимостей уже имеют метаданные по умолчанию, заданные во время процесса их регистрации.</span><span class="sxs-lookup"><span data-stu-id="6574b-106">Many dependency property classes already have default metadata established as part of their registration process.</span></span> <span data-ttu-id="6574b-107">К ним относятся свойства зависимостей, которые являются частью [!INCLUDE[TLA2#tla_api](../../../../includes/tla2sharptla-api-md.md)] [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span><span class="sxs-lookup"><span data-stu-id="6574b-107">This includes the dependency properties that are part of the [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] [!INCLUDE[TLA2#tla_api](../../../../includes/tla2sharptla-api-md.md)].</span></span> <span data-ttu-id="6574b-108">Класс, который наследует свойство зависимости при наследовании класса, может переопределить исходные метаданные, так что характеристики свойства, которые могут изменяться посредством метаданных, будут соответствовать любым особым требованиям подкласса.</span><span class="sxs-lookup"><span data-stu-id="6574b-108">A class that inherits the dependency property through its class inheritance can override the original metadata so that the characteristics of the property that can be altered through metadata will match any subclass-specific requirements.</span></span>  
  
 <span data-ttu-id="6574b-109">Переопределение метаданных в свойстве зависимости должно быть выполнено до того, как свойство начнет использоваться системой свойств (то есть до того момента, когда будут созданы определенные экземпляры объектов, регистрирующие свойство).</span><span class="sxs-lookup"><span data-stu-id="6574b-109">Overriding metadata on a dependency property must be done prior data-ttu-id="6574b-110">Вызовы <xref:System.Windows.DependencyProperty.OverrideMetadata%2A> должны быть выполнены в статических конструкторах типа, предоставляющего себя в качестве `forType` параметр <xref:System.Windows.DependencyProperty.OverrideMetadata%2A>.</span><span class="sxs-lookup"><span data-stu-id="6574b-110">Calls to <xref:System.Windows.DependencyProperty.OverrideMetadata%2A> must be performed within the static constructors of the type that provides itself as the `forType` parameter of <xref:System.Windows.DependencyProperty.OverrideMetadata%2A>.</span></span> <span data-ttu-id="6574b-111">При попытке изменить метаданные существующих экземпляров типа-владельца не возникнет исключения, но это приведет к несогласованному поведению системы свойств.</span><span class="sxs-lookup"><span data-stu-id="6574b-111">If you attempt to change metadata once instances of the owner type exist, this will not raise exceptions, but will result in inconsistent behaviors in the property system.</span></span> <span data-ttu-id="6574b-112">Кроме того, метаданные могут переопределяться один раз для каждого типа.</span><span class="sxs-lookup"><span data-stu-id="6574b-112">Also, metadata can only be overridden once per type.</span></span> <span data-ttu-id="6574b-113">Последующие попытки переопределить метаданные для того же типа вызовут исключение.</span><span class="sxs-lookup"><span data-stu-id="6574b-113">Subsequent attempts to override metadata on the same type will raise an exception.</span></span>
