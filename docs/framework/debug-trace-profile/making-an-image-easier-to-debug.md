<span data-ttu-id="d9375-104">При компиляции неуправляемого кода можно настроить исполняемый образ для отладки, задав параметры командной строки или интегрированной среды разработки.</span><span class="sxs-lookup"><span data-stu-id="d9375-104">When compiling unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="d9375-105">Например, параметр командной строки /**Zi** в Visual C++ служит для создания файлов с отладочными символами (с расширением PDB).</span><span class="sxs-lookup"><span data-stu-id="d9375-105">For example, you can use the /**Zi** command-line option in Visual C++ to ask it to emit debug symbol files (file extension .pdb).</span></span> <span data-ttu-id="d9375-106">Аналогичным образом параметр командной строки /**Od** отключает оптимизацию компилятора.</span><span class="sxs-lookup"><span data-stu-id="d9375-106">Similarly, the /**Od** command-line option tells the compiler to disable optimization.</span></span> <span data-ttu-id="d9375-107">Результирующий код работает медленно, но это упрощает отладку, необходимости.</span><span class="sxs-lookup"><span data-stu-id="d9375-107">The resulting code runs more slowly, but it's easier to debug, should this be necessary.</span></span>

<span data-ttu-id="d9375-108">При компиляции .NET Framework управляемый код, компиляторы, такие как Visual C++, Visual Basic и C# компилируют исходную программу в промежуточный язык Майкрософт (MSIL).</span><span class="sxs-lookup"><span data-stu-id="d9375-108">When compiling .NET Framework managed code, compilers such as Visual C++, Visual Basic, and C# compile their source program into Microsoft intermediate language (MSIL).</span></span> <span data-ttu-id="d9375-109">Код MSIL будет JIT-компиляции, перед выполнением в машинный код.</span><span class="sxs-lookup"><span data-stu-id="d9375-109">MSIL is then JIT-compiled, just before execution, into native machine code.</span></span> <span data-ttu-id="d9375-110">Как и в случае с неуправляемым кодом, можно настроить исполняемый образ для отладки, задав параметры командной строки или интегрированной среды разработки.</span><span class="sxs-lookup"><span data-stu-id="d9375-110">As with unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="d9375-111">Также можно настроить для отладки в так же JIT-компиляцию.</span><span class="sxs-lookup"><span data-stu-id="d9375-111">You can also configure the JIT compilation for debugging in much the same way.</span></span>

<span data-ttu-id="d9375-112">Данная JIT-конфигурация имеет два аспекта.</span><span class="sxs-lookup"><span data-stu-id="d9375-112">This JIT configuration has two aspects:</span></span>

- <span data-ttu-id="d9375-113">Вы можете запросить JIT-компилятор для создания сведений об отслеживании.</span><span class="sxs-lookup"><span data-stu-id="d9375-113">You can request the JIT compiler to generate tracking information.</span></span> <span data-ttu-id="d9375-114">После этого отладчик сможет сопоставлять цепочки команд MSIL с соответствующими командами машинного кода, а также отслеживать место хранения локальных переменных и аргументов функций.</span><span class="sxs-lookup"><span data-stu-id="d9375-114">This makes it possible for the debugger to match up a chain of MSIL with its machine code counterpart, and to track where local variables and function arguments are stored.</span></span> <span data-ttu-id="d9375-115">Начиная с .NET Framework версии 2.0 JIT-компилятор всегда создает сведения об отслеживании, поэтому нет необходимости для запроса.</span><span class="sxs-lookup"><span data-stu-id="d9375-115">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information, so there is no need to request it.</span></span>

- <span data-ttu-id="d9375-116">Вы можете запросить JIT-компилятор, можно отключить оптимизацию получившегося машинного кода.</span><span class="sxs-lookup"><span data-stu-id="d9375-116">You can request the JIT compiler to not optimize the resulting machine code.</span></span>

<span data-ttu-id="d9375-117">Как правило, компилятор, создающим код MSIL задает эти параметры компилятора JIT соответствующим образом, в зависимости от интегрированной среды разработки параметры командной строки или указать, например, /**Od**.</span><span class="sxs-lookup"><span data-stu-id="d9375-117">Normally, the compiler that generates the MSIL sets these JIT compiler options appropriately, based upon the IDE switches or command-line options you specify, for example, /**Od**.</span></span>

<span data-ttu-id="d9375-118">В некоторых случаях может возникнуть необходимость изменить поведение JIT-компилятора, чтобы создаваемый машинный код было проще отлаживать.</span><span class="sxs-lookup"><span data-stu-id="d9375-118">In some cases, you might want to change the behavior of the JIT compiler so that the machine code it generates is easier to debug.</span></span> <span data-ttu-id="d9375-119">Например, для окончательной сборки или оптимизации управления может потребоваться создать сведения о JIT-отслеживании.</span><span class="sxs-lookup"><span data-stu-id="d9375-119">For example, you might want to generate JIT tracking information for a retail build or control optimization.</span></span> <span data-ttu-id="d9375-120">Это можно сделать с помощью файла инициализации (INI).</span><span class="sxs-lookup"><span data-stu-id="d9375-120">You can do so with an initialization (.ini) file.</span></span>

<span data-ttu-id="d9375-121">Например, если сборки, необходимо выполнить отладку называется *MyApp.exe*, то можно создать текстовый файл с именем *MyApp.ini*, в той же папке, что *MyApp.exe*, который содержит Эти три строки:</span><span class="sxs-lookup"><span data-stu-id="d9375-121">For example, if the assembly you want to debug is called *MyApp.exe*, then you can create a text file named *MyApp.ini*, in the same folder as *MyApp.exe*, which contains these three lines:</span></span>
