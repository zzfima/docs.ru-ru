# <a name="get-started-with-f-in-visual-studio-code"></a>Начало работы с F# в Visual Studio Code

Можно написать F# в [Visual Studio Code](https://code.visualstudio.com) с [подключаемый модуль Ionide](https://marketplace.visualstudio.com/items?itemName=Ionide.Ionide-fsharp) для получения кросс платформенных, облегченный интегрированная среда разработки (IDE) удобной с IntelliSense и базовый код операции рефакторинга. Посетите [Ionide.io](http://ionide.io) Дополнительные сведения о подключаемом модуле.

Чтобы начать, убедитесь, что у вас есть [ F# и правильно установлен подключаемый модуль Ionide](install-fsharp.md#install-f-with-visual-studio-code).

## <a name="creating-your-first-project-with-ionide"></a>Создание первого проекта с помощью Ionide

Чтобы создать новую F# проекта, откройте Visual Studio Code в новую папку (можно назвать ее любым).

Затем откройте палитру команд (**представление > палитру команд**) и введите следующую команду:

```
> F# new project
```

Это реализуется с помощью [FORGE](https://github.com/fsharp-editing/Forge) проекта.

> [!NOTE]
> Если вы не видите параметры шаблона, обновите шаблоны, выполнив следующую команду в палитре команд: `>F#: Refresh Project Templates`.

Выберите "F#: Новый проект», нажав **ввод**. Это осуществляется переход к следующему шагу, можно выбрать шаблон проекта.

Выбрать `classlib` шаблона и нажмите клавишу **ввод**.

Затем выберите каталог для создания проекта. Если оставить его пустым, используется текущий каталог.

Наконец имя проекта на последнем шаге. F#использует [регистре Pascal](http://c2.com/cgi/wiki?PascalCase) для имен проектов. В этой статье используется `ClassLibraryDemo` как имя. Как только вы ввели имя, которое будет для вашего проекта, попадания **ввод**.

Если вы выполнили ранее, вы должны получить Visual Studio код рабочую область в левой части окна отображаются со следующими:

1. F# Проектом, под `ClassLibraryDemo` папки.
2. Правильной структуры каталогов для добавления пакетов с помощью [ `Paket` ](https://fsprojects.github.io/Paket/).
3. Кросс платформенные сборки скрипт с [ `FAKE` ](https://fsharp.github.io/FAKE/).
4. `paket.exe` Исполняемый файл, который можно получить пакеты и разрешать зависимости для вас.
5. Объект `.gitignore` файла, если вы хотите добавить этот проект в системе управления версиями на основе Git.

## <a name="moving-your-script-code-into-the-implementation-file"></a>Перемещение кода скрипта в файле реализации

Предыдущих разделах этой статьи показано распространенных первым шагом в письменной F# кода: написание функцию начальной и выполнение его в интерактивном режиме с FSI. Этот процесс известен как разработка на основе REPL, где [REPL](https://en.wikipedia.org/wiki/Read%E2%80%93eval%E2%80%93print_loop) расшифровывается как «Read-Evaluate-Print Loop». Это отличный способ поэкспериментировать с функциональными возможностями, пока не получится, что-то работа.

Следующий шаг в разработки на основе REPL — Чтобы переместить рабочий код в F# файл реализации. Он может быть скомпилирован с F# компилятора в сборку, которая может быть выполнена.

Для начала работы откройте `ClassLibraryDemo.fs`.  Вы заметите, что некоторый код уже существует. Продолжить и удалить определение класса, но не забудьте оставить [ `namespace` ](../language-reference/namespaces.md) объявление вверху.

Создайте новый [ `module` ](../language-reference/modules.md) вызывается `PigLatin` и скопируйте `toPigLatin` функции в нее таким образом:

[!code-fsharp[ToPigLatin](../../../samples/snippets/fsharp/getting-started/pig-latin.fs#L1-L14)]

Затем откройте `Script.fsx` файл еще раз и удалить весь `toPigLatin` работать, но не забудьте сохранить следующие две строки в файле:

```fsharp
#load "ClassLibraryDemo.fs"
open ClassLibraryDemo
```
Выберите обе строки текста и нажмите клавиши Alt + ВВОД, чтобы выполнить эти строки в FSI. Это будет загружать содержимое библиотеки Pig Latin в процесс FSI и `open` `ClassLibraryDemo` пространства имен таким образом, чтобы у вас есть доступ к функциям.

Затем в окне «FSI» вызовите функцию с `PigLatin` модуль, который вы задали ранее:

```
> PigLatin.toPigLatin "banana";;
val it : string = "ananabay"
> PigLatin.toPigLatin "apple";;
val it : string = "appleyay"
```

Готово! Вы получите те же результаты, как раньше, но теперь загружаются из F# файл реализации. Основное отличие здесь заключается в, F# исходные файлы компилируются в сборки, которые могут быть выполнены в любом месте, не только в FSI.

## <a name="troubleshooting"></a>Устранение неполадок

Ниже приведены несколько способов устранения некоторых проблем, которые вы можете столкнуться.

1. Чтобы получить возможности редактирования кода среды Ionide, ваш F# файлы должны быть сохранены на диске, а также внутри папки, который открыт в рабочей области Visual Studio Code.
2. Если были внесены изменения в систему или установить необходимые компоненты Ionide с помощью Visual Studio Code откройте, перезапустите Visual Studio Code.
3. Убедитесь, что можно использовать F# компилятора и F# интерактивные сообщения от командной строки без полного пути. Это можно сделать, введя `fsc` в командную строку для F# компилятора, и `fsi` или `fsharpi` для визуального F# средства в Windows и Mono в Mac/Linux, соответственно.
4. Если у вас есть недопустимые символы в каталогах проекта, Ionide может не работать.  Переименование каталогов проекта, если это так.
5. Если ни одна из команд Ionide работаете, проверьте ваши [сочетания клавиш Visual Studio Code](https://code.visualstudio.com/docs/customization/keybindings#_customizing-shortcuts) для просмотра, если вы переопределение случайно.
6. Если Ionide разбивается на вашем компьютере, и ни один из указанных выше исправил вашу проблему, попробуйте удалить `ionide-fsharp` каталог на компьютере и переустановить suite подключаемого модуля.

Ionide — это проект с открытым кодом создаваемый и используемый средством членами F# сообщества. Пожалуйста, сообщить о проблемах и вы можете внести свой вклад в [Ionide VSCode: Репозиторий FSharp GitHub](https://github.com/ionide/ionide-vscode-fsharp).

Если у вас проблема к отчету, выполните [инструкции по получению журналы для использования при описании проблемы](https://github.com/ionide/ionide-vscode-fsharp#how-to-get-logs-for-debugging--issue-reporting).
