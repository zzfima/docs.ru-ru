# <a name="type-parameter-with-a-structure-constraint-cannot-be-used-as-a-constraint"></a><span data-ttu-id="c83a0-102">Параметр типа с ограничением "Structure" нельзя использовать в качестве ограничения.</span><span class="sxs-lookup"><span data-stu-id="c83a0-102">Type parameter with a 'Structure' constraint cannot be used as a constraint</span></span>
<span data-ttu-id="c83a0-103">Параметр типа с ограничением `Structure` используется в качестве ограничения для другого параметра типа.</span><span class="sxs-lookup"><span data-stu-id="c83a0-103">A type parameter with a `Structure` constraint is used as the constraint for another type parameter.</span></span>  
  
 <span data-ttu-id="c83a0-104">Ограничение `Structure` требует, чтобы аргумент типа, переданный в его параметр типа, был типом значения.</span><span class="sxs-lookup"><span data-stu-id="c83a0-104">The `Structure` constraint requires that the type argument passed to its type parameter be a value type.</span></span> <span data-ttu-id="c83a0-105">Однако тип значения не может быть реализован или унаследован, поэтому нет смысла использовать его в качестве ограничения, для которого потребуется другой параметр типа для реализации или наследования от него.</span><span class="sxs-lookup"><span data-stu-id="c83a0-105">However, a value type cannot be implemented or inherited, so it is not meaningful to use it as a constraint, which would require the other type parameter to implement it or inherit from it.</span></span>  
  
 <span data-ttu-id="c83a0-106">Единственное осмысленное толкование этой ситуации заключается в том, что оба аргумента типа должны быть точно того же типа.</span><span class="sxs-lookup"><span data-stu-id="c83a0-106">The only meaningful interpretation of this situation is that both type arguments must be of the exact same type.</span></span> <span data-ttu-id="c83a0-107">Если это так, то универсальному типу требуется только один параметр типа.</span><span class="sxs-lookup"><span data-stu-id="c83a0-107">If that is the case, your generic type needs only one type parameter.</span></span>  
  
 <span data-ttu-id="c83a0-108">Эту ошибку может вызвать следующая инструкция:</span><span class="sxs-lookup"><span data-stu-id="c83a0-108">The following statement can generate this error.</span></span>  
  
 `Class c1(Of t1 As Structure, t2 As t1)`  
  
 <span data-ttu-id="c83a0-109">Тип, передаваемый `t2` , не может реализовывать или наследовать тип, передаваемый в `t1`, поскольку тип, передаваемый в `t1` , должен быть типом значения.</span><span class="sxs-lookup"><span data-stu-id="c83a0-109">The type passed to `t2` cannot implement or inherit the type passed to `t1`, because the type passed to `t1` must be a value type.</span></span>  
  
 <span data-ttu-id="c83a0-110">**Идентификатор ошибки:** BC32114</span><span class="sxs-lookup"><span data-stu-id="c83a0-110">**Error ID:** BC32114</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="c83a0-111">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="c83a0-111">To correct this error</span></span>  
  
-   <span data-ttu-id="c83a0-112">Удалите параметр типа, ограниченный до `Structure` , из списка ограничений в другом параметре типа.</span><span class="sxs-lookup"><span data-stu-id="c83a0-112">Remove the type parameter constrained to `Structure` from the constraint list on the other type parameter.</span></span>  
  
-   <span data-ttu-id="c83a0-113">Если оба параметра типа требуют одинаковый тип значения, определите универсальный тип с помощью только одного параметра типа.</span><span class="sxs-lookup"><span data-stu-id="c83a0-113">If both type parameters require the same value type, define the generic type with only one type parameter.</span></span>
