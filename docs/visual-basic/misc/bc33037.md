 <span data-ttu-id="4d1f3-107">Если передается аргумент [ByRef](../../visual-basic/language-reference/modifiers/byref.md), Visual Basic иногда копирует его значение в локальную переменную в процедуре вместо передачи ссылки.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-107">If you pass the argument [ByRef](../../visual-basic/language-reference/modifiers/byref.md), Visual Basic sometimes copies the argument value into a local variable in the procedure instead of passing a reference.</span></span> <span data-ttu-id="4d1f3-108">В этом случае когда процедура возвращает результат, Visual Basic должен скопировать значение локальной переменной обратно в аргументе в вызывающем коде.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-108">In such a case, when the procedure returns, Visual Basic must then copy the local variable value back into the argument in the calling code.</span></span>  
  
 <span data-ttu-id="4d1f3-109">Если значение аргумента `ByRef` копируется в процедуру, а аргумент и параметр имеют один и тот же тип, то преобразование не требуется.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-109">If a `ByRef` argument value is copied into the procedure and the argument and parameter are of the same type, no conversion is necessary.</span></span> <span data-ttu-id="4d1f3-110">Но если типы различны, Visual Basic необходимо преобразовать в обоих направлениях.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-110">But if the types are different, Visual Basic must convert in both directions.</span></span> <span data-ttu-id="4d1f3-111">Если один из типов является типом соответствующего класса или структуры, Visual Basic необходимо преобразовать в и от другого типа.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-111">If one of the types is your class or structure type, Visual Basic must convert it both to and from the other type.</span></span> <span data-ttu-id="4d1f3-112">Это означает, что необходимо определить операторы преобразования в обоих направлениях.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-112">This means you must define conversion operators in both directions.</span></span>

-   <span data-ttu-id="4d1f3-115">По возможности используйте аргумент вызова того же типа, что и параметр процедуры, поэтому Visual Basic не нужно выполнять никаких преобразований.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-115">If possible, use a calling argument of the same type as the procedure parameter, so Visual Basic does not need to do any conversion.</span></span>

-   <span data-ttu-id="4d1f3-117">Если требуется возвращать значение в аргумент вызова, определите оператор обратного преобразования, Visual Basic можно преобразовать обратно в тип аргумента вызова.</span><span class="sxs-lookup"><span data-stu-id="4d1f3-117">If you need to return a value into the calling argument, define the reverse conversion operator so Visual Basic can convert back to the calling argument type.</span></span>
