# <a name="cannot-implement-39ltinterfacename1gtltmembernamegt39-because-its-implementation-could-conflict-with-the-implementation-for-39ltinterfacename2gtltmembernamegt39-for-some-type-arguments"></a><span data-ttu-id="74431-102">Не удается реализовать &#39; &lt;имя_интерфейса1&gt;.&lt; имя пользователя&gt;&#39; поскольку может вызвать конфликт с реализацией для &#39;&lt; имя_интерфейса2&gt;.&lt; имя пользователя&gt;&#39; для некоторых аргументов типа.</span><span class="sxs-lookup"><span data-stu-id="74431-102">Cannot implement &#39;&lt;interfacename1&gt;.&lt;membername&gt;&#39; because its implementation could conflict with the implementation for &#39;&lt;interfacename2&gt;.&lt;membername&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="74431-103">Класс реализует более одного универсального интерфейса, один из которых наследуется от другого, и две реализации члена интерфейса могут конфликтовать из-за определенных значений аргументов типа.</span><span class="sxs-lookup"><span data-stu-id="74431-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="74431-104">Эта ошибка может быть вызвана указанными ниже операторами.</span><span class="sxs-lookup"><span data-stu-id="74431-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="74431-105">Так как `iFace2` наследуется от `iFace1` с помощью собственного параметра типа (`u`), `testClass` будет реализовывать две версии `iFace1.testSub` с одинаковыми сигнатурами, если одинаковые аргументы типа будет переданы `y` и `z`.</span><span class="sxs-lookup"><span data-stu-id="74431-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="74431-106">Это вызовет неоднозначность в том, к какой версии получать доступ.</span><span class="sxs-lookup"><span data-stu-id="74431-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="74431-107">**Идентификатор ошибки:** BC32125</span><span class="sxs-lookup"><span data-stu-id="74431-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="74431-108">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="74431-108">To correct this error</span></span>  
  
-   <span data-ttu-id="74431-109">Измените структуру наследования интерфейсов так, чтобы `iFace1` нельзя было реализовать с двумя разными аргументами типа.</span><span class="sxs-lookup"><span data-stu-id="74431-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="74431-110">-или-</span><span class="sxs-lookup"><span data-stu-id="74431-110">-or-</span></span>  
  
-   <span data-ttu-id="74431-111">Удалите из инструкции `Implements` один из интерфейсов, приводящих к появлению конфликта.</span><span class="sxs-lookup"><span data-stu-id="74431-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
