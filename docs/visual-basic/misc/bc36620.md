# <a name="and-expected"></a><span data-ttu-id="8b039-102">Требуется And</span><span class="sxs-lookup"><span data-stu-id="8b039-102">'And' expected</span></span>
<span data-ttu-id="8b039-103">Для объединения двух или более операторов `Equals` в предложении `Join` или `Group Join` используется оператор сравнения, отличный от `And`.</span><span class="sxs-lookup"><span data-stu-id="8b039-103">A comparison operator other than `And` has been used to combine two or more `Equals` operators in a `Join` or `Group Join` clause.</span></span> <span data-ttu-id="8b039-104">Только оператор `And` может объединять несколько операторов `Equals` в предложении `Join` или `Group Join` .</span><span class="sxs-lookup"><span data-stu-id="8b039-104">Only the `And` operator is allowed to combine multiple `Equals` operators in a `Join` or `Group Join` clause.</span></span>  
  
 <span data-ttu-id="8b039-105">**Идентификатор ошибки:** BC36620</span><span class="sxs-lookup"><span data-stu-id="8b039-105">**Error ID:** BC36620</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="8b039-106">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="8b039-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="8b039-107">Реструктурируйте предложения `Equals` , чтобы проводить сравнения только с помощью оператора `And` .</span><span class="sxs-lookup"><span data-stu-id="8b039-107">Restructure the `Equals` clauses to make comparisons by using only the `And` operator.</span></span> <span data-ttu-id="8b039-108">Ниже представлен пример такого кода.</span><span class="sxs-lookup"><span data-stu-id="8b039-108">The following is an example:</span></span>

                    Join pet In pets _  
                    On pet.Owner Equals pers And _  
                       pet.Name = pers.PetName_  
                    Select pers, pet
