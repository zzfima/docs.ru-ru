# <a name="class-classname-must-declare-a-sub-new-because-its-base-class-classname2-does-not-have-an-accessible-sub-new-that-can-be-called-with-no-arguments"></a><span data-ttu-id="f8b40-102">Класс\<имя_класса > 'должен объявить «Sub New», так как его базовый класс\<имя_класса2 >' не имеет доступного «Sub New», который может быть вызван без аргументов</span><span class="sxs-lookup"><span data-stu-id="f8b40-102">Class '\<classname>' must declare a 'Sub New' because its base class '\<classname2>' does not have an accessible 'Sub New' that can be called with no arguments</span></span>
<span data-ttu-id="f8b40-103">Производный класс не объявляет конструктор, и Visual Basic не может создать его, поскольку отсутствует конструктор базового класса, который он может вызвать.</span><span class="sxs-lookup"><span data-stu-id="f8b40-103">A derived class does not declare a constructor, and Visual Basic cannot generate one because there is no base class constructor it can call.</span></span>  
  
 <span data-ttu-id="f8b40-104">Если производный класс не объявляет конструктор, Visual Basic пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="f8b40-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="f8b40-105">Если в базовом классе нет доступного конструктора, который может вызываться без аргументов, или если имеется более одного, Visual Basic не может создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="f8b40-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="f8b40-106">**Идентификатор ошибки:** BC30387</span><span class="sxs-lookup"><span data-stu-id="f8b40-106">**Error ID:** BC30387</span></span>
