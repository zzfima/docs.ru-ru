# <a name="type-parameter-lttypeparameternamegt-for-ltgenericprocedurenamegt-cannot-be-inferred"></a><span data-ttu-id="c78a9-102">Параметр типа "&lt;имя_параметра_типа&gt;«for»&lt;имя_универсальной_процедуры&gt;" не могут быть получены</span><span class="sxs-lookup"><span data-stu-id="c78a9-102">Type parameter '&lt;typeparametername&gt;' for '&lt;genericprocedurename&gt;' cannot be inferred</span></span>
<span data-ttu-id="c78a9-103">Универсальная процедура вызывается без указания списка аргументов типов, и для одного из аргументов типа не удается вывести тип.</span><span class="sxs-lookup"><span data-stu-id="c78a9-103">A generic procedure is called without supplying a type argument list, and type inference fails for one of the type arguments.</span></span>  
  
 <span data-ttu-id="c78a9-104">Как правило, при вызове универсальной процедуры указывается аргумент типа для каждого параметра типа, определяемого этой процедурой.</span><span class="sxs-lookup"><span data-stu-id="c78a9-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="c78a9-105">Тем не менее, существует возможность полностью опустить список аргументов типов.</span><span class="sxs-lookup"><span data-stu-id="c78a9-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="c78a9-106">При этом компилятор пытается определить тип каждого аргумента типа из контекста вызова.</span><span class="sxs-lookup"><span data-stu-id="c78a9-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="c78a9-107">Дополнительные сведения см. в разделе "Определение типа" статьи [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span><span class="sxs-lookup"><span data-stu-id="c78a9-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>  
  
 <span data-ttu-id="c78a9-108">Одной из возможных причин сбоя определения типа является несоответствие ранга параметра типа и вызываемого типа.</span><span class="sxs-lookup"><span data-stu-id="c78a9-108">One possible cause of type inference failure is a mismatch of rank between a type parameter and the calling type.</span></span> <span data-ttu-id="c78a9-109">Это проиллюстрировано в следующем коде.</span><span class="sxs-lookup"><span data-stu-id="c78a9-109">The following code illustrates this.</span></span>  
  
```  
Public Sub displayLargest(Of t As IComparable)(ByVal arg() As t)  
    ' Insert code to find and display the largest element of arg().  
End Sub  
Public Sub callGenericSub()  
    Dim testValue As Integer  
    findLargest(testValue)  
    Dim testMatrix(,) As Integer  
    findLargest(testMatrix)  
End Sub  
```  
  
 <span data-ttu-id="c78a9-110">В приведенном выше коде оба вызова `findLargest` вызывают эту ошибку, поскольку параметр типа `t` запрашивает одномерный массив, тогда как аргументы типа, которые компилятор выводит из вызовов, являются скаляром (`testValue`) и двумерным массивом (`testMatrix`).</span><span class="sxs-lookup"><span data-stu-id="c78a9-110">In the preceding code, the two calls to `findLargest` both produce this error, because the type parameter `t` calls for a one-dimensional array, whereas the type arguments the compiler infers from the calls are a scalar (`testValue`) and a two-dimensional array (`testMatrix`).</span></span>  
  
 <span data-ttu-id="c78a9-111">**Идентификатор ошибки:** BC32050</span><span class="sxs-lookup"><span data-stu-id="c78a9-111">**Error ID:** BC32050</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="c78a9-112">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="c78a9-112">To correct this error</span></span>  
  
-   <span data-ttu-id="c78a9-113">Убедитесь, что типы нормальных аргументов таковы, что определение типа согласуется с параметрами типов, объявленными для универсальной процедуры.</span><span class="sxs-lookup"><span data-stu-id="c78a9-113">Make sure the types of the normal arguments are such that type inference is consistent with the type parameters declared for the generic procedure.</span></span>  
  
     <span data-ttu-id="c78a9-114">- или -</span><span class="sxs-lookup"><span data-stu-id="c78a9-114">-or-</span></span>  
  
-   <span data-ttu-id="c78a9-115">Вызовите универсальную процедуру с полным списком аргументов типов, чтобы определение типа не требовалось.</span><span class="sxs-lookup"><span data-stu-id="c78a9-115">Call the generic procedure with a complete type argument list, so that type inference is not necessary.</span></span>
