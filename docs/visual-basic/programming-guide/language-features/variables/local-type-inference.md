## <a name="type-inference-vs-late-binding"></a><span data-ttu-id="6bc14-113">Определение типа или. Позднее связывание</span><span class="sxs-lookup"><span data-stu-id="6bc14-113">Type Inference vs. Late Binding</span></span>  
 <span data-ttu-id="6bc14-114">Код, который использует выведение типа напоминает код, который полагается на динамическое связывание.</span><span class="sxs-lookup"><span data-stu-id="6bc14-114">Code that uses type inference resembles code that relies on late binding.</span></span> <span data-ttu-id="6bc14-115">Тем не менее, выведение типа строго типизирует переменной не оставляйте его как `Object`.</span><span class="sxs-lookup"><span data-stu-id="6bc14-115">However, type inference strongly types the variable instead of leaving it as `Object`.</span></span> <span data-ttu-id="6bc14-116">Компилятор использует инициализатор переменной для определения типа переменной во время компиляции для создания кода с ранним связыванием.</span><span class="sxs-lookup"><span data-stu-id="6bc14-116">The compiler uses a variable's initializer to determine the variable's type at compile time to produce early-bound code.</span></span> <span data-ttu-id="6bc14-117">В предыдущем примере `num2`, например `num1`, типизируется как `Integer`.</span><span class="sxs-lookup"><span data-stu-id="6bc14-117">In the previous example, `num2`, like `num1`, is typed as an `Integer`.</span></span>  
  
 <span data-ttu-id="6bc14-118">Поведение переменных с ранним связыванием отличается от переменных с поздним связыванием, для которых тип известен только во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="6bc14-118">The behavior of early-bound variables differs from that of late-bound variables, for which the type is known only at run time.</span></span> <span data-ttu-id="6bc14-119">Знание типа заранее позволяет компилятору определить проблемы до выполнения, точно выделить память и выполнять другие виды оптимизации.</span><span class="sxs-lookup"><span data-stu-id="6bc14-119">Knowing the type early enables the compiler to identify problems before execution, allocate memory precisely, and perform other optimizations.</span></span> <span data-ttu-id="6bc14-120">Раннее связывание позволяет Visual Basic интегрированной среде разработки (IDE) для предоставления IntelliSense справки о членах класса объекта.</span><span class="sxs-lookup"><span data-stu-id="6bc14-120">Early binding also enables the Visual Basic integrated development environment (IDE) to provide IntelliSense Help about the members of an object.</span></span> <span data-ttu-id="6bc14-121">Раннее связывание также является предпочтительным для производительности.</span><span class="sxs-lookup"><span data-stu-id="6bc14-121">Early binding is also preferred for performance.</span></span> <span data-ttu-id="6bc14-122">Это обусловлено тем, все данные, хранящиеся в переменной с поздним связыванием, которые должны быть упакованы как тип `Object`, и доступ к членам типа во время выполнения делает программу медленнее.</span><span class="sxs-lookup"><span data-stu-id="6bc14-122">This is because all data stored in a late-bound variable must be wrapped as type `Object`, and accessing members of the type at run time makes the program slower.</span></span>
